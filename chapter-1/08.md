# 8. 대사모델링 전체 워크플로 개관

**왜 이걸 배우나요?** §1~7에서 대사·GEM·CBM·분류·역사·응용이라는 개념의 조각들을 하나씩 배웠습니다. 이번 절은 이 조각들을 실제 "작업 순서"로 이어 붙입니다 — 즉 "지금까지 배운 개념들이 실제 연구 현장에서는 어떤 순서로 등장하는가"를 보여주는 이 장의 마지막 큰 그림입니다. 이 워크플로는 동시에 이 책 2장부터 9장까지의 목차이기도 합니다 — 각 단계가 곧 하나 또는 여러 장의 주제입니다.

![게놈 서열에서 네트워크 재구축과 제약 기반 모델을 거쳐 표현형을 예측하는 GEM 워크플로](../.gitbook/assets/fig2_gem_workflow.png)

*그림 8.1. GEM의 입력–재구축–수학적 모델–표현형 예측 흐름. 아래의 7단계 목록은 이 개념도를 품질관리와 맥락 특이화까지 확장한 것입니다.*

```
1단계. 대사 네트워크 재구축(Reconstruction)
        게놈 주석 + 데이터베이스(KEGG, MetaCyc) + 문헌 → 초안 네트워크
        [상세: Chapter 5]
              │
              ▼
2단계. 모델 구조화(Formalization)
        화학량론 행렬 S 구성, GPR 규칙 부여, 구획·수송 반응,
        생물량 목적함수(Biomass Objective Function) 정의
        [상세: Chapter 2, Chapter 3]
              │
              ▼
3단계. 제약 설정(Constraining)
        열역학적/용량적/환경적 bounds 설정 (배지 조성, 산소 조건 등)
        [상세: Chapter 2, Chapter 3]
              │
              ▼
4단계. 시뮬레이션(Simulation)
        FBA/pFBA/FVA를 통한 통량 분포 예측, 유전자 결실 스크리닝
        [상세: Chapter 4]
              │
              ▼
5단계. 검증과 정제(Validation & QC)
        실험 데이터(성장률, 필수성, 분비 프로파일)와 비교, MEMOTE 품질 평가
        [상세: Chapter 5]
              │
              ▼
6단계. 조건 특이화(Context-Specific Integration) — 선택적
        전사체/단백체 데이터를 통합하여 특정 세포·조직·조건에 맞는 모델 도출
        [상세: Chapter 6]
              │
              ▼
7단계. 응용(Application)
        균주 설계, 약물 표적 발굴, 질병 메커니즘 규명, AI 결합 예측 등
        [상세: Chapter 7, Chapter 8, Chapter 9]
```

각 단계를 아주 간단히 한 문장씩 풀어보면 다음과 같습니다.

- **1단계(재구축)**는 "이 생명체가 가진 대사 능력의 목록을 만드는" 단계입니다. 게놈을 읽고, 어떤 유전자가 어떤 효소를 만드는지 찾아, 그 효소가 촉매하는 반응들을 데이터베이스와 문헌에서 확인합니다.
- **2단계(구조화)**는 이렇게 모은 반응 목록을 컴퓨터가 계산할 수 있는 수학적 형태 — 화학량론 행렬, GPR 규칙, 생물량 반응 — 로 바꾸는 단계입니다.
- **3단계(제약 설정)**는 "이 세포가 어떤 환경에 있는가"를 반영하는 단계입니다. 배지에 포도당이 얼마나 있는지, 산소가 있는지 없는지 등을 각 반응의 상한·하한으로 표현합니다.
- **4단계(시뮬레이션)**는 이렇게 완성된 수학 문제를 실제로 풀어서, "이 조건에서 세포는 어떤 속도로 자라고, 각 반응은 얼마나 활발히 일어나는가"를 계산하는 단계입니다.
- **5단계(검증)**는 계산 결과를 실제 실험 데이터와 비교해서 모델이 믿을 만한지 확인하는 단계입니다.
- **6단계(조건 특이화)**는 선택적 단계로, "일반적인 세포"가 아니라 "이 환자의 이 조직 세포"처럼 더 구체적인 상황에 맞게 모델을 조정하는 단계입니다.
- **7단계(응용)**은 완성된 모델을 실제 문제 — 균주 설계, 신약 표적 탐색 등 — 에 사용하는 단계입니다.

이 워크플로는 선형적이라기보다 **반복적(Iterative)**입니다. 5단계에서 모델-실험 불일치가 발견되면 1~2단계로 되돌아가 네트워크를 정제하는 과정이 반복되며, 이는 §6.3에서 살펴본 96단계 프로토콜의 핵심 철학이기도 합니다.

**`e_coli_core`로 7단계를 미리 따라가 보기.** 아직 어떤 계산도 하지 않았지만, 이 워크플로가 §9(실습)에서 만날 `e_coli_core` 모델에 어떻게 적용되었을지 미리 상상해 보는 것은 앞으로의 여러 장을 읽어나가는 데 좋은 나침반이 됩니다.

| 단계 | `e_coli_core`에서는 이렇게 일어났을 것입니다 | 확인은 어느 장에서? |
|:---|:---|:---|
| 1. 재구축 | *E. coli* K-12의 중심 탄소 대사 유전자·문헌을 모아 95개 반응의 초안을 구성 | [Chapter 5](../chapter-5/README.md) |
| 2. 구조화 | 72개 대사물 × 95개 반응의 $$\mathbf{S}$$ 행렬, GPR, `BIOMASS_Ecoli_core_w_GAM` 정의 | [Chapter 2](../chapter-2/README.md), [Chapter 3](../chapter-3/README.md) |
| 3. 제약 설정 | 포도당 흡수 상한, 호기/혐기 조건별 산소 bounds 설정 | [Chapter 2](../chapter-2/README.md), [Chapter 3](../chapter-3/README.md) |
| 4. 시뮬레이션 | FBA로 표준 호기 조건 성장률 μ ≈ 0.874 h⁻¹ 계산 | [Chapter 4](../chapter-4/README.md) |
| 5. 검증 | 예측된 성장률·분비물을 실험적 배양 데이터와 대조 | [Chapter 5](../chapter-5/README.md) |
| 6. 조건 특이화 | (`core` 모델 자체에는 보통 생략; 완전 모델에서 조직·조건별로 수행) | [Chapter 6](../chapter-6/README.md) |
| 7. 응용 | 특정 유전자 결손 시 성장률 변화 스크리닝 등 교육적 활용 | [Chapter 4](../chapter-4/README.md), [Chapter 8](../chapter-8/README.md) |

이 표의 4단계에 등장하는 μ ≈ 0.874 h⁻¹라는 숫자를 기억해 두십시오. [Chapter 4](../chapter-4/README.md)에서 여러분이 직접 FBA를 실행하면 정확히 이 값을 눈으로 확인하게 됩니다.

{% hint style="warning" %}
❓ **흔한 오해:** 이 7단계를 보고 "한 번 모델을 완성하면 그걸로 끝"이라고 생각하기 쉽습니다. 그러나 실제 GEM 개발은 5단계(검증)에서 실험과 어긋나는 예측이 발견될 때마다 1~2단계로 돌아가 네트워크를 고치는 일이 반복되는, 마치 나선형 계단을 오르는 과정에 가깝습니다. §6.3에서 iJR904 → iAF1260 → iJO1366 → iML1515로 이어지는 *E. coli* 모델 계보를 다시 떠올려 보십시오 — 각 모델은 이전 모델의 "검증 결과 발견된 한계"를 딛고 재구축을 반복한 결과물입니다.
{% endhint %}
