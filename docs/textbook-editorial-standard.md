# 교재 편집·인용 기준

이 문서는 본문을 학부 고학년·대학원 입문 수준의 전공 교재로 유지하기 위한 내부 편집 기준이다. 새로운 원고와 기존 원고의 개정에 동일하게 적용한다.

## 1. 서술 수준과 문체

- 절은 학술적 문제, 정의 또는 분석 범위로 시작한다. “왜 이걸 배우나요?”, “아주 쉬운 비유”, 독자에게 답을 유도하는 수사적 질문은 사용하지 않는다.
- 일상적 비유는 정확한 정의를 대체하지 않는 경우에만 제한적으로 사용한다. 비유를 사용하면 성립하지 않는 지점도 함께 밝힌다.
- 독자를 직접 지칭하는 “여러분”, 학습을 독려하는 “기억하십시오”, 감탄형 표현과 이모지는 사용하지 않는다.
- 핵심 주장은 정의 → 가정 → 수학적 표현 또는 실험적 근거 → 해석 범위와 한계 순으로 전개한다.
- 한국어 용어를 우선하되 처음 등장할 때 영어 원어와 약어를 병기한다. 이후에는 하나의 표기를 일관되게 사용한다.
- 문장 종결 어미는 글의 유형에 따라 일관되게 쓴다. **이론 본문(각 장의 README와 번호 절)은 학술 다체(~한다/이다)**로, **실습·튜토리얼(각 장의 lab, 10장, 11장)은 친절한 습니다체(~합니다/입니다)**로 통일한다. 한 파일 안에서 두 어미를 섞지 않는다.
- 절 소제목은 서술형 명제로 쓴다. "왜 …하는가?", "…할까요?"와 같은 수사적 질문형 제목은 사용하지 않는다.

## 2. 개념·수식·수치

- 반응, 대사물, 플럭스처럼 모델에서 특별한 의미를 갖는 용어는 생화학적 대상과 계산 객체를 구분하여 정의한다.
- 수식을 제시할 때 기호의 정의, 차원 또는 단위, 적용 가정을 함께 쓴다. 플럭스는 양이 아니라 단위 시간·단위 생물량당 반응 진행률임을 명시한다.
- 정상 상태, 최적성, 반응 방향성처럼 조건에 의존하는 진술은 보편적 사실로 서술하지 않는다.
- 반응·대사물·유전자 수는 생물 종의 고정 특성이 아니라 특정 모델 릴리스의 속성으로 기록한다. 모델명, 버전 또는 논문, 집계 기준을 함께 제시한다.
- 계산 결과에는 모델 ID와 버전, 소프트웨어와 솔버, 배지·경계조건, 목적함수, 허용오차를 가능한 범위에서 기록한다.
- 예측, 실험 관찰, 저자의 해석을 구분한다. 단일 연구나 단일 조건의 결과를 일반 법칙으로 확장하지 않는다.

## 3. 그림과 표

- 모든 주요 그림에는 번호와 자립적인 캡션을 붙인다. 캡션만 읽어도 대상, 축·기호, 조건, 핵심 해석을 파악할 수 있어야 한다.
- 자체 제작 모식도도 개념적 근거가 된 원 논문이나 표준 문헌을 “저자 작성; 개념 근거:” 형식으로 인용한다.
- 논문의 그림을 변형하거나 재작성한 경우 “~에서 수정·재구성”이라고 표시하고 원출처의 DOI 또는 영구 링크와 라이선스를 확인한다.
- 계산 그림에는 생성 스크립트와 계산 조건을 밝힌다. 모식도와 실제 모델 계산 결과를 캡션에서 명시적으로 구분한다.
- 외부 그림을 직접 복제하기보다, 필요한 관계를 재현 가능한 코드나 벡터 도식으로 새로 그리고 원개념을 인용하는 방식을 우선한다.

## 4. 인용과 참고문헌

- 정량값, 역사적 최초성, 성능 비교, 임상·산업적 효과에는 일차 문헌 또는 공식 데이터베이스를 인용한다.
- 방법론 도식은 그 방법을 처음 제안한 논문과, 현재 구현을 설명하는 공식 문서를 구분해 인용한다.
- 웹 자료는 DOI, PubMed/PMC, 공식 프로젝트 저장소, 모델 데이터베이스 등 장기적으로 추적 가능한 주소를 우선한다.
- 출처가 확인되지 않은 이미지와 수치, 논문 결론보다 강한 표현은 삭제하거나 검증 가능한 수준으로 낮춘다.

## 5. 학습 요소

- 장 도입부의 학습목표는 측정 가능한 동사(정의한다, 유도한다, 비교한다, 계산한다, 비판적으로 평가한다)로 작성한다.
- 본문 확인문제는 개념 전개를 중단하지 않도록 절 말미나 장 요약에 배치한다. 정답을 질문 바로 뒤에 노출하지 않는다.
- “흔한 오해” 블록은 “해석상의 주의”로 바꾸고, 오해를 유발하는 조건과 올바른 적용 범위를 구체적으로 적는다.
- 연습문제는 정의 확인, 손 계산, 결과 해석, 가정 비판의 네 수준을 균형 있게 포함한다.
