# 4. Context manager로 배지와 목적함수 바꾸기

## 4.0 값을 바꾸고 나중에 되돌리면 안 되는 이유

`Model`은 가변(mutable) 객체입니다. 파이썬에서 `model.medium["EX_glc__D_e"] = 5`처럼 값을 바꾸면, 그 변경은 다음 셀에서도, 심지어 다른 함수 안에서도 계속 남아 있습니다. 이것이 왜 위험한지 일상 비유로 설명하면: 여러 사람이 함께 쓰는 사무실 프린터의 기본 설정(용지 크기, 흑백/컬러)을 내가 임시로 바꿔 놓고 원상복구를 잊으면, 다음 사람이 의도치 않게 내가 바꾼 설정으로 인쇄하게 되는 것과 같습니다.

FBA 실험에서 이 문제는 더 치명적입니다. glucose 조건을 5로 바꿔 실험한 뒤 복원을 깜빡하면, 다음 셀의 "wild-type 기준 실험"이 사실은 glucose 5인 조건에서 계산된 것이 되어 버립니다. `with model:` 구문(context manager)은 이 문제를 구조적으로 막아 줍니다. 블록 안에서 일어난 모든 bound·목적함수 변경을 COBRApy가 내부적으로 기록해 두었다가, 블록을 벗어나는 순간 자동으로 되돌립니다. 파이썬의 `with` 구문 자체는 `__enter__`/`__exit__` 메서드를 가진 어떤 객체에도 적용되는 일반적인 문법이며, 파일을 열고 자동으로 닫는 `with open(...) as f:`와 같은 원리입니다.

## 4.1 Glucose 제한과 무산소 조건

이 절은 §2.2에서 확인한 exchange 부호 규약을 실제로 활용합니다. `model.medium`에 새 딕셔너리를 대입하면 COBRApy가 내부적으로 각 exchange 반응의 lower bound를 계산해 갱신합니다.

> **선행 셀 필요: §1.1의 `model`, `results`**

```python
baseline_medium = model.medium.copy()   # 나중에 "원래대로 돌아왔는지" 비교할 기준 스냅샷.

glucose_limited_medium = baseline_medium.copy()
glucose_limited_medium["EX_glc__D_e"] = 5.0   # 섭취 용량 10 -> 5로 제한(lower bound: -10 -> -5).
with model:
    model.medium = glucose_limited_medium     # 이 블록 안에서만 유효한 임시 변경.
    glucose_5_growth = model.slim_optimize(error_value=np.nan)
# 여기(with 블록 밖)로 나오는 순간 model.medium은 자동으로 baseline_medium과 같아진다.

anaerobic_medium = baseline_medium.copy()
anaerobic_medium["EX_o2_e"] = 0.0             # 산소 섭취를 완전히 차단(무산소 조건).
with model:
    model.medium = anaerobic_medium
    anaerobic_growth = model.slim_optimize(error_value=np.nan)

assert np.isfinite(glucose_5_growth)
assert np.isfinite(anaerobic_growth)
assert model.medium == baseline_medium  # 두 실험 뒤 원상 복구

glucose_5_growth = float(glucose_5_growth)
anaerobic_growth = float(anaerobic_growth)
results["glucose_5_growth"] = glucose_5_growth
results["anaerobic_growth"] = anaerobic_growth

print("glucose uptake capacity 5:", f"{glucose_5_growth:.9f}")
print("oxygen uptake capacity 0:", f"{anaerobic_growth:.9f}")

assert abs(glucose_5_growth - 0.415597775) < 1e-6
assert abs(anaerobic_growth - 0.211662950) < 1e-6
```

```text
# 기대 출력:
glucose uptake capacity 5: 0.415597775
oxygen uptake capacity 0: 0.211662950
```

두 숫자는 각각 다른 한 가지 조건만 바꾼 결과입니다. glucose 제한과 무산소 조건을 동시에 적용한 값이 아닙니다. 조건 비교표에는 이처럼 바꾸지 않은 나머지 조건도 함께 기록해야 합니다. 산소가 완전히 없을 때 성장률(0.2117)이 기본 호기성 조건(0.8739)보다 훨씬 낮다는 이 모델 예측은, 산소 호흡(호기성 대사)이 발효보다 훨씬 많은 ATP를 만들어 낸다는 교과서 지식과 같은 방향입니다. 다만 이는 `textbook` 모델의 화학량론과 경계조건이 만들어 낸 계산 결과이지, 이 조건에서의 실험 관찰값은 아닙니다.

{% hint style="warning" %}
**해석상의 주의: "`with model:` 블록을 쓰면 `model` 자체가 새 객체로 복사된다"**

아닙니다. `with model:`은 새 모델을 만들지 않고, **같은 `model` 객체의 변경 이력을 스택에 쌓아 두었다가 블록을 나갈 때 되돌리는** 방식입니다. 따라서 블록 **안에서** `model.slim_optimize()`를 호출해야 그 임시 조건이 반영된 값을 얻습니다. 아래처럼 블록 밖에서 최적화를 호출하면 이미 복원된 원래 조건의 결과를 얻게 되는 흔한 실수가 발생합니다.

```python
# 잘못된 예 — 들여쓰기가 빠져 with 블록 밖에서 최적화를 호출함
with model:
    model.medium = glucose_limited_medium
growth = model.slim_optimize()  # 이미 baseline_medium으로 복원된 뒤의 값!
```
{% endhint %}

{% hint style="warning" %}
**흔한 에러**: `model.medium = glucose_limited_medium`에서 `glucose_limited_medium`에 존재하지 않는 exchange 반응 ID를 키로 넣으면 COBRApy가 조용히 무시하거나 오류를 낼 수 있습니다(버전에 따라 다름). `set(glucose_limited_medium) - set(baseline_medium)`으로 사전에 키 집합이 같은지 확인하는 습관을 들이면 안전합니다.
{% endhint %}

## 4.2 Biomass 대신 ATP maintenance 최대화

FBA는 "세포가 무엇을 최적화하는가"에 대한 가정을 필요로 합니다. 지금까지는 암묵적으로 biomass 생산을 최대화한다고 가정했지만, 같은 $$\mathbf{S}\mathbf{v}=\mathbf{0}$$과 bound 아래에서도 목적함수를 바꾸면 완전히 다른 질문에 답할 수 있습니다. 여기서는 목적함수를 `ATPM`(ATP maintenance, 성장과 무관하게 세포 유지에 소비되는 ATP 가수분해 반응 — [비성장 관련 유지 에너지(non-growth-associated maintenance, NGAM)](../glossary.md)를 구현한 반응)으로 바꾸어 "이 모델의 대사 네트워크가 만들어 낼 수 있는 최대 ATP 생산 능력은 얼마인가"를 묻습니다.

> **선행 셀 필요: §1.1의 `model`, `results`; §3.1의 `wt_growth`**

```python
with model:
    model.objective = "ATPM"    # 목적함수를 biomass에서 ATP maintenance 반응으로 교체.
    max_atp_maintenance = model.slim_optimize(error_value=np.nan)
# with 블록을 벗어나면 목적함수도 자동으로 원래(biomass)로 복원된다.

assert np.isfinite(max_atp_maintenance)
max_atp_maintenance = float(max_atp_maintenance)
restored_growth = float(model.slim_optimize(error_value=np.nan))  # 복원 여부를 재확인.

results["max_ATPM"] = max_atp_maintenance

print("max ATPM:", f"{max_atp_maintenance:.6f}")
print("restored biomass objective:", f"{restored_growth:.9f}")

assert abs(max_atp_maintenance - 175.0) < 1e-6
assert abs(restored_growth - wt_growth) < 1e-9
```

```text
# 기대 출력:
max ATPM: 175.000000
restored biomass objective: 0.873921507
```

가능 영역(feasible region)이 같아도 목적함수가 달라지면 선택되는 해가 달라집니다. "모델 예측"을 보고할 때는 모델 이름만이 아니라 **목적함수와 방향(max/min)**을 함께 적어야 하는 이유입니다.

{% hint style="info" %}
`ATPM`은 [비성장 관련 유지 에너지(NGAM)](../glossary.md)를 나타내는 별도 반응으로, `textbook` 모델에서는 기본적으로 하한 `8.39`($$\mathrm{mmol\,ATP\,gDW^{-1}\,h^{-1}}$$)로 고정된 ATP 가수분해 반응입니다. 이 하한은 성장이 0일 때도 흘러야 하는 기저 ATP 소비를 강제합니다 — §4.1에서 glucose 섭취 용량을 반(10→5)으로 줄였을 때 성장률이 정확히 절반보다 더 낮아진(0.8739→0.4156) 것도 이 고정 유지 비용 때문입니다. 반면 **성장 관련 유지 에너지(GAM)**는 별도 반응이 아니라 biomass 반응 안의 ATP 가수분해 항으로 접혀 성장률에 비례하므로, `ATPM` 하나를 GAM과 NGAM의 합으로 해석하지 않습니다. 이 절은 그 하한을 NGAM 제약으로 두는 대신 목적함수로 바꿔, 반응이 도달할 수 있는 **최댓값**을 진단용으로 구한 경우입니다. GAM·NGAM의 정의·단위·추정 절차는 [Chapter 3 §6](../chapter-3/06.md)에서 다룹니다.
{% endhint %} 목적함수에 따라 선택되는 해가 달라진다는 이 원리는, 유전자 결손 뒤 세포가 새로운 목적을 향해 재조정된다고 가정하는 [Chapter 8](../chapter-8/README.md)의 MOMA·ROOM([대표 논문 가이드](../landmark-papers.md))과도 맞닿아 있습니다 — MOMA와 ROOM은 "biomass 최대화"가 아니라 "wild-type flux와의 거리 최소화"라는 전혀 다른 목적함수를 세운 경우입니다. 이 장에서는 [§7](08.md)에서 직접 비교합니다.

{% hint style="warning" %}
**흔한 에러**: `model.objective = "ATPM"`에서 `"ATPM"`이 실제 반응 ID와 다르면(`textbook`이 아닌 다른 모델을 쓰는 경우) `KeyError`가 납니다. 자신의 모델에서 ATP maintenance 반응 ID가 다를 수 있으므로, `[r.id for r in model.reactions if "ATP" in r.id]`처럼 후보를 먼저 나열해 정확한 ID를 찾으십시오.
{% endhint %}

{% hint style="success" %}
**잠깐, 생각해보기**: `with model:` 블록을 중첩해서 `with model: model.medium = ...; with model: model.objective = ...`처럼 쓰면 어떻게 될까요?

정답: 문제없이 동작합니다. COBRApy의 context manager는 스택(stack) 구조로 변경 이력을 쌓기 때문에, 안쪽 블록을 벗어나면 안쪽에서 바꾼 것만 되돌아가고 바깥쪽 블록의 변경(medium)은 유지됩니다. 바깥쪽 블록까지 완전히 벗어나야 medium도 원래대로 복원됩니다. 이 성질 덕분에 "배지도 바꾸고 목적함수도 바꾼" 조건을 안전하게 조합할 수 있습니다.
{% endhint %}
