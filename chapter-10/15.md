# 14. 미니 프로젝트: 한 조건을 완결된 분석으로 만들기

## 14.0 지금까지 배운 조각을 하나의 완결된 실험으로

§1부터 §13까지 우리는 preflight 고정, FBA와 질량보존 검산, context manager, pFBA/FVA, 유전자 결손, MOMA/ROOM, MILP, gap-filling, production envelope, 대화형 탐색, SBML 왕복, provenance 기록이라는 조각들을 하나씩 따로 실행해 보았습니다. 이 마지막 절은 그 조각들을 **자신이 직접 고른 조건 하나**에 대해 처음부터 끝까지 이어 붙여, "튜토리얼을 따라 했다"에서 "나만의 재현 가능한 계산 실험을 완성했다"로 넘어가는 자리입니다.

다음 순서로 탄소원 하나 또는 유전자 하나를 선택해 이 장의 흐름을 반복해 보십시오. 예를 들어 "glucose 대신 acetate를 유일한 탄소원으로 주면 어떻게 되는가"(탄소원 스레드) 또는 "essential로 분류된 7개 유전자 중 하나를 골라 FBA·MOMA·ROOM으로 비교하면 어떻게 다른가"(유전자 스레드) 중 하나를 선택할 수 있습니다.

1. 조건을 자연어와 수치 bound로 함께 적습니다. (예: "glucose 섭취를 완전히 막고 EX_ac_e의 lower bound를 -10으로 연다" — 이렇게 자연어와 숫자를 나란히 적어야 나중에 코드 없이도 조건을 복원할 수 있습니다.)
2. context manager 안에서 조건을 적용하고 `status`, 목적함수, $$\max|\mathbf{S}\mathbf{v}|$$를 검사합니다. ([§3~4](04.md), [§4](05.md) 참고)
3. pFBA와 관심 반응 3개의 FVA를 계산합니다. ([§5](06.md) 참고)
4. 유전자 perturbation이라면 FBA와 선형 MOMA/ROOM의 가정을 구분해 결과를 비교합니다. ([§7](08.md), [Chapter 8](../chapter-8/README.md) 참고)
5. Plotly 그림에는 축 단위와 조건을 적고, GitBook용 정적 그림도 보존합니다. ([§10](11.md) 참고)
6. 모델 SBML, SHA-256, 환경·결과 JSON을 함께 저장합니다. ([§12~13](13.md) 참고)

최종 보고서에는 "성장률이 얼마였다"만 적지 말고, **어떤 모델의 어떤 버전에서, 어떤 배지와 목적함수로, 어떤 solver 상태를 얻었으며, 질량보존과 불확실성을 어떻게 확인했는지**를 한 문단으로 설명하십시오. 그것이 실행 가능한 코드를 재현 가능한 계산 실험으로 바꾸는 마지막 단계입니다.

💡 **점검용 체크리스트**

- [ ] `cobra.__version__`과 solver 이름을 기록했는가?
- [ ] 조건을 `with model:` 블록 안에서만 적용했고, 블록 밖에서 `model.medium`이 원래대로인지 확인했는가?
- [ ] `solution.status == "optimal"`을 확인했는가?

- [ ] 95개(또는 자신의 모델의) flux 전체를 하나의 고정 벡터와 비교하는 대신, 목적값·핵심 제약·FVA 범위로 검증했는가?
- [ ] 결과를 JSON이나 표로 저장해, 코드를 다시 실행하지 않고도 숫자를 확인할 수 있는가?

## 이 장, 그리고 이 책을 마치며

이 미니 프로젝트를 완성했다면, 이 책의 본문 여정 전체 — [Chapter 1](../chapter-1/README.md)의 대사 네트워크 개념부터 [Chapter 9](../chapter-9/README.md)의 AI 결합까지 — 를 하나의 실행 가능한 코드 흐름으로 검산한 것입니다. 이 장에서 반복해서 강조한 네 질문(무엇을 바꿨는가, solver 상태는 optimal인가, 질량보존과 유한성을 확인했는가, 재현 기록을 남겼는가)은 `e_coli_core`를 벗어나 [BiGG](http://bigg.ucsd.edu/)나 다른 저장소에 공개된 더 큰 모델, 다른 생물종, 실제 연구 문제로 옮겨 가더라도 그대로 유효합니다.

다음으로 읽을 만한 자료는 부록의 [대표 논문으로 읽는 GEM 방법론](../landmark-papers.md)입니다. 이 책 전체에서 다룬 FBA, pFBA, MOMA, ROOM, gap-filling 같은 방법들이 원래 어떤 논문에서 어떤 질문에 답하기 위해 제안되었는지, 그리고 그 논문을 인용할 때 흔히 발생하는 오독은 무엇인지 정리되어 있습니다. 용어를 다시 확인하고 싶다면 [핵심 용어집](../glossary.md)을, 이 책의 내용이 실제 강의의 어느 주차에 대응하는지 보고 싶다면 [강의 PPT–전자책 대응표](../lecture-coverage-map.md)를 참고하십시오.
