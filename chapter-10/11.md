# 10. Production envelope와 대화형 Plotly

## 10.0 "성장이냐 생산이냐"라는 트레이드오프를 눈으로 보기

세포공장을 설계할 때 자주 부딪히는 질문은 "이 균주가 목표 물질을 더 많이 만들도록 강제하면, 성장은 얼마나 희생해야 하는가"입니다. FBA 한 번으로는 "최대 성장일 때의 생산량 하나"밖에 알 수 없습니다. Production envelope는 생산물 flux를 여러 값에 고정하고, 각 지점에서 가능한 성장률의 최소·최대를 계산해, 한 번의 FBA 곡선이 아니라 생산-생장 feasible region의 경계 전체를 보여 줍니다. 이 개념과 phenotype phase plane(PhPP)의 이론적 배경, 균주 설계에서의 활용은 [Chapter 8](../chapter-8/README.md)에서 자세히 다룹니다.

## 10.1 Acetate 생산 포락선 계산

> 🔗 **선행 셀 필요: §1.1의 `model`, `configuration`, `results`**

```python
from cobra.flux_analysis import production_envelope

configuration.processes = 1
envelope = production_envelope(
    model,
    reactions=["EX_ac_e"],           # 생산물로 볼 반응: acetate exchange.
    carbon_sources="EX_glc__D_e",    # 탄소원 기준으로 수율을 정규화.
    points=21,                        # acetate flux를 0~최대까지 21개 지점으로 나눔.
)

peak_row = envelope.loc[envelope["flux_maximum"].idxmax()]  # 성장률이 가장 높은 지점.
peak_growth = float(peak_row["flux_maximum"])
acetate_at_peak_growth = float(peak_row["EX_ac_e"])

results["envelope_peak_growth"] = peak_growth
results["envelope_acetate_at_peak_growth"] = acetate_at_peak_growth
results["envelope_max_acetate"] = float(envelope["EX_ac_e"].max())

print(envelope[["EX_ac_e", "flux_minimum", "flux_maximum"]]
      .head().round(6).to_string(index=False))
print("peak growth / acetate at peak:", peak_growth, acetate_at_peak_growth)

assert abs(peak_growth - 0.873921507) < 1e-6
assert abs(acetate_at_peak_growth) < 1e-9
```

기본 `e_coli_core` 조건에서 최대 성장 지점의 acetate secretion은 0이고, acetate를 더 많이 강제할수록 가능한 최대 성장이 감소합니다(표의 `flux_maximum` 열이 `EX_ac_e`가 커질수록 단조 감소하는 것을 볼 수 있습니다). 이것만으로 생산과 생장이 성장 결합(growth-coupled)되었다고 말할 수는 없습니다. 성장 결합이라면 양의 생장 구간에서 생산물의 **최소 flux**도 양수인지 확인해야 합니다 — 위 표의 `flux_minimum` 열이 모두 0인 것은, 이 조건에서 세포가 acetate를 "전혀 분비하지 않고도" 같은 성장률을 낼 수 있는 대안 경로가 있다는 뜻이며, 곧 **강제로 분비시키지 않는 한 성장과 생산이 결합되어 있지 않다**는 신호입니다.

{% hint style="warning" %}
❓ **흔한 오해: "production envelope가 아래로 기울어 있으면 항상 성장 결합(growth-coupled) 균주다"**

아닙니다. 이 절의 envelope는 acetate flux를 늘릴수록 최대 성장이 줄어드는 "트레이드오프"만 보여 줄 뿐입니다. 진짜 성장 결합 여부를 판단하려면 `flux_minimum`(그 성장률에서 강제로라도 나오는 최소 생산량)이 0보다 큰 구간이 있는지를 봐야 합니다. 이 예제의 `flux_minimum`은 모든 지점에서 0이므로, "acetate를 안 만들고도 그 성장률을 낼 수 있는" 대안 경로가 항상 존재한다는 뜻이고 이는 성장 결합의 반대 신호에 가깝습니다.
{% endhint %}

{% hint style="warning" %}
❓ **흔한 에러**: `carbon_sources` 인자에 실제로 uptake bound가 열려 있지 않은 반응 ID를 주면(예: 이미 `medium`에서 막아 둔 탄소원), `production_envelope`가 모든 지점에서 성장률 0을 반환할 수 있습니다. `model.medium`을 먼저 확인해 지정한 탄소원의 섭취 용량이 0보다 큰지 점검하십시오.
{% endhint %}

## 10.2 Plotly로 hover·zoom·범례 토글하기

> 🔗 **선행 셀 필요: §10.1의 `envelope`; 선택 사항(`plotly`)**

```python
import plotly.graph_objects as go

fig = go.Figure()
fig.add_trace(
    go.Scatter(
        x=envelope["EX_ac_e"],
        y=envelope["flux_minimum"],
        mode="lines+markers",
        name="minimum growth",
        line=dict(color="#9ecae1"),
        hovertemplate="acetate=%{x:.2f}<br>min growth=%{y:.3f}<extra></extra>",
    )
)
fig.add_trace(
    go.Scatter(
        x=envelope["EX_ac_e"],
        y=envelope["flux_maximum"],
        mode="lines+markers",
        name="maximum growth",
        line=dict(color="#08519c"),
        fill="tonexty",
        fillcolor="rgba(49, 130, 189, 0.20)",
        hovertemplate="acetate=%{x:.2f}<br>max growth=%{y:.3f}<extra></extra>",
    )
)
fig.update_layout(
    title="Acetate production envelope",
    xaxis_title="Acetate secretion flux (mmol gDW⁻¹ h⁻¹)",
    yaxis_title="Growth rate (h⁻¹)",
    template="plotly_white",
    hovermode="x unified",
)
fig.show()
```

Jupyter에서 점 위에 마우스를 올리면 정확한 좌표를 읽고, 드래그로 확대하고, 범례를 눌러 경계를 숨길 수 있습니다. GitBook의 Markdown 페이지는 Jupyter kernel과 widget state를 실행하지 않으므로, 위 코드는 복사해 노트북에서 실행하고 공개 문서에는 그림 10.3 같은 정적 PNG를 함께 둡니다.
