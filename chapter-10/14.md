# 13. 실행 결과와 provenance를 JSON으로 남기기

## 13.0 "재현 가능하다"는 주장을 뒷받침하는 파일

이 장 전체에 걸쳐 우리는 §1의 preflight부터 §12의 SBML 왕복 검증까지 매 단계에서 "지금 어떤 조건에서 이 숫자가 나왔는가"를 확인해 왔습니다. 그런데 이 확인들이 노트북 출력창에만 남아 있다면, 노트북을 닫는 순간 사라집니다. Provenance(출처 기록)란 "이 결과가 정확히 어떤 소프트웨어 버전·모델·조건에서 나왔는지"를 추적 가능하게 남기는 메타데이터입니다. 이 절은 이 장에서 쌓아 온 `results` 딕셔너리와 환경 정보를 하나의 JSON 파일로 응고시켜, 다른 사람(혹은 6개월 뒤의 자신)이 같은 결과를 재현하거나 검증할 수 있게 합니다.

마지막으로 핵심 숫자, 패키지 버전, solver, 배지, 목적함수를 한 파일에 기록합니다. timestamp는 결과를 만든 시점이지 모델 release 시점이 아닙니다.

> **선행 셀 필요: §1~12의 `model`, `results`**

```python
import json
import platform
from datetime import datetime, timezone
from importlib.metadata import PackageNotFoundError, version
from pathlib import Path

def installed_version(distribution_name):
    try:
        return version(distribution_name)
    except PackageNotFoundError:
        return None   # 패키지가 설치되지 않았다면 조용히 실패하지 않고 명시적으로 None을 남긴다.

run_record = {
    "created_at_utc": datetime.now(timezone.utc).isoformat(),  # 결과 "생성 시점"(모델 release 시점 아님).
    "python": platform.python_version(),
    "packages": {
        name: installed_version(name)
        for name in ["cobra", "optlang", "swiglpk", "numpy", "pandas"]
    },
    "solver_interface": model.solver.interface.__name__,
    "model": {
        "source": "cobra.io.load_model('textbook')",
        "id": model.id,
        "reactions": len(model.reactions),
        "metabolites": len(model.metabolites),
        "genes": len(model.genes),
        "objective_direction": model.objective.direction,
        "objective_coefficients": objective_coefficients(model),
        "medium_uptake_capacities": {
            reaction_id: float(capacity)
            for reaction_id, capacity in sorted(model.medium.items())
        },
    },
    "benchmarks": results,   # §1~12에서 쌓아 온 모든 검증된 숫자.
    "interpretation_policy": {
        "essential_ratio_below": 0.01,
        "growth_reduced_ratio_below": 0.90,
        "nonfinite_deletion_growth": "classified as zero only after status review",
    },
}

record_path = Path("cobrapy_tutorial_results.json")
record_path.write_text(
    json.dumps(run_record, ensure_ascii=False, indent=2),
    encoding="utf-8",
)

print("saved:", record_path.resolve())
print(json.dumps(run_record["benchmarks"], ensure_ascii=False, indent=2))
```

```text
# 기대 출력(발췌 — 절대 경로·timestamp·소수점 마지막 몇 자리는 실행 환경마다 다를 수 있음):
saved: /path/to/cwd/cobrapy_tutorial_results.json
{
  "wt_growth": 0.873921507...,
  "max_abs_Sv_residual": ...,
  "glucose_5_growth": 0.415597775...,
  "anaerobic_growth": 0.211662950...,
  "max_ATPM": 175.0,
  ...
}
```

버전 기록만으로는 충분하지 않습니다. 실제 연구에서는 Git commit, 입력 데이터 DOI/checksum, 모델 release, solver tolerance, 사용한 스크립트와 seed도 함께 기록합니다. `run_record["interpretation_policy"]`처럼 **숫자만이 아니라 그 숫자를 해석하는 규칙(임계값, 정책)까지 함께 저장**하는 습관은 몇 달 뒤 같은 파일을 다시 열어 보는 사람(원 작성자 포함)이 "[§6](07.md)에서 나온 7/24/106이라는 분류가 어떤 기준에서 나왔는지"를 코드를 다시 읽지 않고도 알 수 있게 해 줍니다.

{% hint style="warning" %}
**흔한 에러**: `record_path = Path("cobrapy_tutorial_results.json")`는 상대 경로이므로, 노트북을 실행하는 현재 작업 디렉터리에 저장됩니다. 여러 조건을 반복 실행하면서 매번 같은 파일명으로 덮어쓰면 이전 실행 기록이 사라집니다. 실무에서는 `f"results_{run_record['created_at_utc']}.json"`처럼 타임스탬프를 파일명에 넣거나, 결과 전용 디렉터리(`results/`)를 만들어 누적 보관하는 것이 안전합니다.
{% endhint %}
