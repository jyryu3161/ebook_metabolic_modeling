# 5. 하나의 최적해를 넘어: pFBA와 FVA

## 5.0 왜 "최적해 하나"로는 이야기가 끝나지 않는가

[Chapter 4](../chapter-4/README.md)에서 이미 배웠듯, 선형계획법(LP)은 목적함수 값을 유일하게 정하지만 **그 값을 만드는 flux 벡터는 여러 개일 수 있습니다.** 비유하면, "이번 학기 평균 학점 4.0"이라는 결과는 하나여도, 그 4.0을 만드는 과목별 성적 조합은 여러 가지일 수 있는 것과 같습니다. §3에서 구한 `wt_growth = 0.873921507`도 마찬가지로 유일한 최적 flux 분포가 아니라 "그 값을 내는 여러 대안 중 solver가 우연히 고른 하나"일 수 있습니다.

pFBA와 FVA는 이 문제에 서로 다른 방식으로 접근합니다. pFBA는 "여러 대안 중 하나를 더 그럴듯한 기준으로 골라내는" 방법이고, FVA는 "그 여러 대안들이 반응별로 얼마나 넓게 퍼져 있는지 범위로 보여 주는" 방법입니다. 즉 pFBA는 답을 하나로 좁히고, FVA는 답의 불확실성 폭을 드러냅니다.

## 5.1 pFBA: 최대 성장 뒤 총 flux 최소화

pFBA(parsimonious FBA; [Lewis et al., 2010](https://doi.org/10.1038/msb.2010.47))는 먼저 최대 성장을 구한 뒤, 그 성장을 유지하면서 총 절대 flux $$\sum_j |v_j|$$를 최소화합니다. 이는 두 단계로 이루어진 사전식(lexicographic) 최적화입니다.

$$
\text{1단계: } \max \; \mu \quad \Rightarrow \quad \mu^{*}
$$

$$
\text{2단계: } \min \sum_j |v_j| \quad \text{subject to} \quad \mu = \mu^{*}
$$

즉 1단계에서 구한 최대 성장률 $$\mu^{*}$$을 고정 제약으로 못박은 뒤, 그 조건 안에서 다시 한 번 "총 효소 사용량에 비례한다고 가정하는 총 flux"를 최소화합니다. 이 사전식 목적은 "같은 성장을 낼 수 있다면 효소 사용이 덜 필요한 경로를 선호한다"는 가정을 추가합니다 — 세포가 불필요하게 많은 반응을 동시에 높은 flux로 돌리기보다, 목표를 달성하는 가장 "경제적인" 경로를 쓴다는 직관을 수식화한 것입니다.

> 🔗 **선행 셀 필요: §1.1의 `model`, `results`; §3.1의 `wt_growth`**

```python
from cobra.flux_analysis import pfba

pfba_solution = pfba(model)   # 내부적으로 1단계(biomass 최대화) + 2단계(총 |v| 최소화)를 자동 수행.
pfba_growth = float(pfba_solution.fluxes["Biomass_Ecoli_core"])
pfba_total_absolute_flux = float(pfba_solution.fluxes.abs().sum())

results["pfba_growth"] = pfba_growth
results["pfba_total_absolute_flux"] = pfba_total_absolute_flux

print("pFBA biomass flux:", f"{pfba_growth:.9f}")
print("pFBA sum |v|:", f"{pfba_total_absolute_flux:.6f}")

assert pfba_solution.status == "optimal"
assert abs(pfba_growth - wt_growth) < 1e-6
```

```text
# 기대 출력:
pFBA biomass flux: 0.873921507
pFBA sum |v|: 518.422086
```

COBRApy 0.30.0의 `pfba_solution.objective_value`는 이 경우 biomass가 아니라 **2단계 목적함수인 총 flux** 약 `518.422086`입니다. 생장률은 반드시 `pfba_solution.fluxes["Biomass_Ecoli_core"]`에서 읽어야 합니다.

{% hint style="warning" %}
❓ **흔한 오해: "`pfba_solution.objective_value`가 생장률일 것이다"**

pFBA는 2단계 최적화이므로 `objective_value`는 **마지막 단계(2단계)의 목적함수 값**, 즉 총 절대 flux 합입니다. 이 값(약 518)을 생장률(약 0.87)로 착각하고 보고하면 두 자릿수 이상 틀린 숫자를 발표하는 사고로 이어질 수 있습니다. 항상 어떤 반응의 flux를 읽고 있는지 변수 이름으로 명확히 구분하십시오.
{% endhint %}

{% hint style="warning" %}
❓ **흔한 에러**: `pfba(model)` 호출 전에 `with model:` 없이 이미 다른 목적함수나 배지로 바꿔 놓은 상태라면, pFBA의 1단계가 biomass가 아닌 다른 목적을 최대화하게 됩니다. pFBA를 호출하기 직전 `model.objective`를 출력해 원하는 목적함수인지 확인하는 습관을 들이십시오.
{% endhint %}

## 5.2 FVA: 최적에 가까운 해들의 반응별 범위

FVA(Flux Variability Analysis; [Mahadevan & Schilling, 2003](https://doi.org/10.1016/j.ymben.2003.09.002))는 관심 있는 각 반응 $$j$$에 대해 다음 두 LP를 추가로 풉니다.

$$
v_j^{\min} = \min v_j, \qquad v_j^{\max} = \max v_j
\qquad \text{subject to} \qquad
\mathbf{S}\mathbf{v} = \mathbf{0}, \;\; \mathbf{c}^{\top}\mathbf{v} \ge f \cdot Z^{*}
$$

여기서 $$Z^{*}$$는 원래 목적함수(예: biomass)의 최적값이고, $$f$$는 `fraction_of_optimum`(예: `0.90`)입니다. 즉 "최적값의 90% 이상을 유지한다"는 제약을 추가로 걸어 두고, 그 제약 안에서 반응 $$j$$가 가질 수 있는 최소·최대 flux를 각각 찾습니다. `fraction_of_optimum=1.0`으로 두면 "최적해들만의" 범위를, 값을 낮추면 "차선책까지 포함한 더 넓은" 범위를 얻습니다.

> 🔗 **선행 셀 필요: §1.1의 `model`, `results`**

```python
from cobra.flux_analysis import flux_variability_analysis

fva = flux_variability_analysis(
    model,
    reaction_list=["PGI", "PFK", "TPI"],   # 관심 반응만 지정하면 전체 95개보다 훨씬 빠르다.
    fraction_of_optimum=0.90,               # 최적 성장의 90% 이상을 유지하는 해들만 고려.
    processes=1,
)
fva["width"] = fva["maximum"] - fva["minimum"]   # 범위 폭 = 대안 최적해들의 다양성 지표.

results["fva_PGI_min"] = float(fva.loc["PGI", "minimum"])
results["fva_PGI_max"] = float(fva.loc["PGI", "maximum"])

print(fva.round(6).to_string())
```

```text
# 기대 출력:
       minimum    maximum      width
PGI -14.299039   9.838761  24.137801
PFK   1.171706  25.290644  24.118938
TPI   1.171706   9.217639   8.045934
```

기준 실행에서 `PGI` 범위는 약 `[-14.299039, 9.838761]`입니다. 부호가 양쪽에 걸친다는 것은 최대 성장의 90% 이상을 유지하는 해들 중 PGI를 정방향으로 쓰는 해와 역방향으로 쓰는 해가 모두 있다는 뜻입니다. 이는 `PGI` 반응이 이 조건에서 필수적이지 않고, 다른 경로(예: pentose phosphate pathway 우회)로도 비슷한 성장을 낼 수 있음을 시사합니다. `PFK`와 `TPI`는 둘 다 양수 구간(각각 `[1.171706, 25.290644]`, `[1.171706, 9.217639]`)에만 걸쳐 있어, 최대 성장의 90% 이상을 유지하는 모든 해에서 항상 정방향으로만 흐른다는 것을 보여 줍니다.

FVA의 각 행은 해당 반응만 최소화·최대화한 별도 LP의 결과입니다. 따라서 모든 반응의 minimum을 한꺼번에 모은 벡터가 하나의 실현 가능한 flux distribution이라고 해석해서는 안 됩니다. `PGI`의 minimum과 `PFK`의 minimum을 동시에 만족하는 단일 flux 벡터가 존재한다는 보장은 없습니다 — 각 행은 "그 반응만" 극단으로 밀어붙였을 때의 결과이기 때문입니다.

{% hint style="warning" %}
❓ **흔한 오해: "FVA의 width가 넓은 반응일수록 세포가 그 반응을 잘 안 쓴다는 뜻이다"**

아닙니다. width가 넓다는 것은 "그 반응의 flux가 대안 최적해들 사이에서 크게 흔들려도 성장률이 거의 그대로 유지된다"는 뜻이지, 실제 세포가 어떤 값을 선택하는지를 말하지 않습니다. 오히려 width가 좁거나 0인 반응(예: `TPI`처럼 좁은 범위)이 그 조건에서 값이 거의 고정되어 있다는, 즉 대사적으로 더 "경직된" 반응일 수 있습니다.
{% endhint %}

{% hint style="warning" %}
❓ **흔한 에러**: `fraction_of_optimum=1.0`으로 두었더니 일부 반응에서 `minimum > maximum`처럼 보이는 부동소수점 오차, 또는 예상보다 좁은 범위만 나오는 경우 — 수치 solver의 허용오차(tolerance) 근방에서 발생하는 정상적인 현상입니다. 엄밀하게 0인지 확인하려면 `abs(width) < 1e-6`처럼 작은 허용오차를 두고 비교하십시오.
{% endhint %}
