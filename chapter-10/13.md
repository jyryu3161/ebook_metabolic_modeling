# 12. SBML 임시 왕복 검증과 SHA-256

## 12.0 "저장했다"와 "그 저장이 온전하다"는 다른 주장이다

[SBML](https://sbml.org/)(Systems Biology Markup Language)은 대사 모델을 저장·교환하는 표준 XML 형식입니다. 형식 자체와 FBC(Flux Balance Constraints) 플러그인이 목적함수·bound·GPR을 어떻게 담는지는 [SBML 실무 보충 자료](../supplements/sbml-practical.md)에서 자세히 다룹니다. 이 절에서 확인하려는 것은 다른 문제입니다 — **파일로 저장한 모델을 다시 읽었을 때, 저장 전과 똑같은 모델이 복원되는가**입니다.

이것이 왜 당연하지 않은지 생각해 봅시다. XML 직렬화(serialization) 과정에서 부동소수점 반올림, 목적함수 계수의 누락, GPR 문자열의 인코딩 오류 등으로 미묘한 정보 손실이 생길 수 있습니다. "SBML로 저장했다"는 사실만으로 재현성이 확보되지는 않습니다. 다시 읽었을 때 반응·대사물 수, 목적함수, 최적화 결과가 유지되는지 직접 확인하고, 정확히 어느 바이트의 파일을 사용했는지 SHA-256 checksum(파일 내용이 한 비트라도 다르면 완전히 다른 값이 나오는 지문)으로 식별하는 것이 "왕복 검증(round-trip validation)"입니다.

## 12.1 저장 → 재로딩 → 4중 검증

> 🔗 **선행 셀 필요: §1.1의 `model`, `results`; §3.1의 `wt_growth`**

```python
import hashlib
from pathlib import Path
from tempfile import TemporaryDirectory

from cobra.io import read_sbml_model, write_sbml_model
from cobra.util.solver import linear_reaction_coefficients

def objective_coefficients(a_model):
    # 목적함수에 실제로 기여하는(계수가 0이 아닌) 반응만 추려 비교용 딕셔너리로 만든다.
    return {
        reaction.id: float(coefficient)
        for reaction, coefficient in linear_reaction_coefficients(a_model).items()
        if abs(float(coefficient)) > 1e-12
    }

with TemporaryDirectory() as temporary_directory:
    sbml_path = Path(temporary_directory) / "textbook_snapshot.xml"
    write_sbml_model(model, sbml_path)          # 메모리의 model을 XML 파일로 직렬화.

    sbml_sha256 = hashlib.sha256(sbml_path.read_bytes()).hexdigest()  # 파일 바이트의 지문.
    restored_model = read_sbml_model(sbml_path)  # 같은 파일을 다시 파싱해 새 Model 객체로.
    restored_solution = restored_model.optimize()

roundtrip_growth = float(restored_solution.objective_value)
results["sbml_sha256"] = sbml_sha256
results["sbml_roundtrip_growth"] = roundtrip_growth

print("SHA-256:", sbml_sha256)
print("roundtrip growth:", f"{roundtrip_growth:.9f}")

# 네 가지 독립적인 검증: solver 상태, 반응 수, 대사물 수, 목적함수, 최적값.
assert restored_solution.status == "optimal"
assert len(restored_model.reactions) == len(model.reactions)
assert len(restored_model.metabolites) == len(model.metabolites)
assert objective_coefficients(restored_model) == objective_coefficients(model)
assert abs(roundtrip_growth - wt_growth) < 1e-6
```

## 12.2 SHA-256과 "생물학적으로 같은 모델"은 다른 질문이다

임시 디렉터리는 셀이 끝난 뒤 삭제되므로 이 예제의 checksum은 검증 절차를 보여 주기 위한 것입니다. 실제 프로젝트에서는 snapshot SBML과 checksum을 결과 디렉터리에 함께 보존하십시오. 같은 생물학적 모델이라도 serializer나 라이브러리 버전이 바뀌면 XML 바이트 배열과 checksum이 달라질 수 있으므로, checksum 자체를 이 장의 고정 기준값으로 assert하지 않습니다 — 오직 "저장 전후 반응 수·대사물 수·목적함수·최적값이 같다"는 생물학적으로 의미 있는 사실만 assert 합니다.

{% hint style="warning" %}
❓ **흔한 오해: "SHA-256이 다르면 모델이 바뀐 것이다"**

아닙니다. SHA-256은 XML **바이트 배열**의 지문이지 모델의 생물학적 내용의 지문이 아닙니다. 같은 반응·대사물·bound를 담고 있어도 XML 저장 순서, 들여쓰기, 타임스탬프 메타데이터, 또는 `cobra`/`libsbml` 버전 차이만으로 바이트가 달라져 완전히 다른 SHA-256이 나올 수 있습니다. 따라서 "생물학적으로 같은 모델인가"는 SHA-256이 아니라 위 코드처럼 반응 수·대사물 수·목적함수·최적값을 직접 비교해야 합니다. SHA-256은 "지금 내가 손에 든 이 파일이 실험에 쓴 그 파일과 바이트 단위로 완전히 같은가"를 확인하는 용도입니다.
{% endhint %}

{% hint style="warning" %}
❓ **흔한 에러**: `objective_coefficients(restored_model) == objective_coefficients(model)`이 실패한다면, SBML의 FBC 플러그인이 목적함수 계수를 저장하지 못하는 오래된 SBML 레벨/버전을 사용했을 가능성이 있습니다. `write_sbml_model`은 기본적으로 FBC를 포함한 최신 SBML을 쓰지만, 다른 도구(예: 오래된 버전의 다른 소프트웨어)로 만든 SBML을 불러올 때는 이 문제가 흔히 발생합니다.
{% endhint %}
