# 2. 질병 맥락 특이적 대사 모델

암과 정상 조직의 유전자 결손 효과를 비교하려면 두 모델의 차이가 생물학적 맥락에서 비롯되도록 분석 조건을 맞추어야 한다. 서로 다른 기저 reconstruction, 배지 또는 biomass 반응을 사용하면 모델 구성 차이를 암 특이성으로 오인할 수 있다.

## 2.1 기저 인간 reconstruction

[맥락 특이적 모델](../glossary.md)은 인간 대사의 합집합을 나타내는 기저 reconstruction에서 출발한다. 대표 자료의 보고 규모는 다음과 같지만 릴리스와 집계 기준을 함께 기록해야 한다.

| reconstruction | 보고 유전자/ORF | 보고 반응 | 보고 대사물 | 출처 |
|:---|---:|---:|---:|:---|
| [Human1](https://metabolicatlas.org/) | 3,625 | 13,417 | 10,138 compartment-specific species | [Robinson et al. (2020)](https://doi.org/10.1126/scisignal.aaz1482) |
| [Recon3D](https://www.vmh.life/) reconstruction | 3,288 ORF | 13,543 | 4,140 unique metabolites | [Brunk et al. (2018)](https://doi.org/10.1038/nbt.4072) |

*Table 2.1. 두 인간 reconstruction의 논문 보고 규모. Recon3D의 flux-consistent model과 전체 reconstruction은 서로 다른 반응·대사물 수를 갖는다.*

기저 reconstruction을 그대로 사용하면 서로 다른 조직에서만 가능한 경로가 한 네트워크에 공존할 수 있다. 그러나 반대로 발현이 낮은 반응을 모두 제거하면 유지·대사 작업에 필요한 경로와 전사 후 조절되는 효소를 잃을 수 있다. 맥락 특이화는 “발현된 반응만 남기기”가 아니라 오믹스 근거와 필수 기능을 함께 만족시키는 모델 선택 문제이다.

## 2.2 맥락 특이화 알고리즘

[GIMME](../glossary.md), [iMAT](../glossary.md), [INIT/tINIT](../glossary.md), [FASTCORE](../landmark-papers.md)와 CORDA는 입력과 목적함수가 서로 다르다.

- GIMME와 iMAT은 발현과 일치하는 flux 상태를 찾되 원래 모델 구조를 반드시 영구 제거하지는 않는다.
- FASTCORE는 신뢰도가 높은 core 반응을 포함하는 작은 flux-consistent subnetwork를 찾는다.
- INIT은 반응 근거 점수와 관찰 대사물 정보를 이용해 조직 모델을 추출한다([Agren et al. 2012](https://doi.org/10.1371/journal.pcbi.1002518)).
- tINIT은 INIT 계열의 task-driven 구현으로 지정한 필수 대사 작업을 만족하도록 모델을 구성한다([Agren et al. 2014](https://doi.org/10.1002/msb.145122)).
- 같은 전사체라도 threshold, GPR mapping, 보호 task와 solver tolerance에 따라 다른 반응 집합이 나온다.

각 알고리즘의 정확한 최적화 정식화와 발현 임계값 민감도는 [Chapter 6](../chapter-6/README.md)에서 다룬다.

Troppo는 여러 맥락 특이화 방법을 공통 인터페이스에서 실행·평가하는 소프트웨어 프레임워크이다. Vieira et al.은 대규모 비교에서 알고리즘 선택이 추출 모델의 구조와 예측 표현형에 영향을 준다는 점을 보였다([DOI 10.1371/journal.pcbi.1009294](https://doi.org/10.1371/journal.pcbi.1009294)). 서로 다른 benchmark의 ROC-AUC와 실행 시간을 출처 없이 한 표에 합쳐 보편적 알고리즘 순위를 만들 수 없다.

모델 구축 시 최소한 다음 정보를 보존한다.

1. 기저 reconstruction의 이름, 릴리스와 checksum
2. 오믹스 단위, 정규화·batch 보정과 표본 포함 기준
3. GPR mapping과 gene identifier 변환 규칙
4. 알고리즘, threshold, protected reaction·task와 무작위 seed
5. 배지, exchange bounds, biomass·maintenance 반응
6. solver, feasibility tolerance와 추출 후 flux-consistency 검사

## 2.3 암-정상 비교 설계

암-정상 선택성을 평가할 때 두 모델은 가능한 한 동일한 계산 기준을 공유해야 한다.

| 비교 요소 | 공정한 비교 조건 |
|:---|:---|
| 기저 구조 | 같은 reconstruction과 annotation 버전 |
| 환경 | 같은 배지 또는 측정치로 근거가 있는 조건별 배지 |
| 목적함수 | 같은 반응과 정규화; 조건별 목적 차이를 쓰면 별도 분석 |
| 유지 에너지 | 같은 정의와 단위; 조건별 값은 근거 제시 |
| 모델 추출 | 같은 알고리즘·threshold·protected tasks |
| 결손 판정 | 같은 상대·절대 성장 임계값과 solver tolerance |

*Table 2.2. 암-정상 모델의 결손 효과를 비교할 때 맞추어야 할 계산 요소.*

정상 대조군의 선택에는 서로 다른 편향이 있다.

- **matched adjacent normal**은 유전적 배경과 일부 기술 batch를 공유하지만, field effect와 종양 미세환경의 영향을 받을 수 있다.
- **[GTEx](https://gtexportal.org/) 같은 독립 건강 조직**은 종양 인접 효과를 피하지만 기증자, 사후 시간, 실험 protocol과 세포 조성이 다를 수 있다.
- **암 세포주와 정상 조직 bulk 자료**의 비교는 배양 적응과 세포 조성 차이를 암-정상 차이로 섞을 수 있다.

따라서 한 정상 기준만으로 “정상 세포에 안전하다”고 결론짓지 않는다. 여러 정상 조직과 세포 유형, 가능한 matched normal을 각각 분석하고 기준선 선택에 대한 민감도를 보고한다.

## 2.4 모델 검증과 데이터 누출

맥락 특이적 모델은 표적 스크리닝 전에 다음 수준에서 검증한다.

1. 원소·전하, blocked reaction, energy-generating cycle과 대사 작업을 점검한다.
2. 측정한 uptake, secretion, 성장률 또는 isotope flux를 가능한 범위에서 재현한다.
3. 추출에 사용하지 않은 유전자 필수성 자료로 예측을 평가한다.
4. 여러 reconstruction·추출법·threshold에서 유지되는 후보를 구분한다.

[DepMap](https://depmap.org/portal/) 필수성 자료를 모델의 threshold 선택과 최종 성능 평가에 모두 사용하면 데이터 누출이 발생한다. 조정 집합과 평가 집합을 세포주 또는 암종 수준에서 분리하고, 최종 후보는 독립 실험으로 검증해야 한다.
