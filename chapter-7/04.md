# 4. 합성 치사(Synthetic Lethality) 기반 표적 발굴

## 4.1 SL의 개념과 치료적 의의

**왜 하나의 유전자만으로는 부족할까요?** §3.5의 끝에서 확인했듯, 필수성만으로 좋은 표적을 찾기는 생각보다 어렵습니다 — 대사 네트워크의 중복성(redundancy) 때문에 웬만한 유전자 하나쯤은 없어도 우회 경로로 버틸 수 있기 때문입니다(§1.2의 글루타민 우회 경로가 대표적 예). 그렇다면 질문을 바꿔봅시다 — "하나를 꺼도 안 죽는다면, **두 개를 동시에** 꺼야만 죽는 조합은 없을까?" 이 질문에 답하는 개념이 바로 **합성 치사(synthetic lethality, SL)**입니다.

> **핵심 개념·용어(English):** 합성 치사(Synthetic Lethality, SL) — 두 유전자가 **동시에** 교란되었을 때만 세포 사멸이 일어나고, 각각 단독으로 교란되었을 때는 세포가 생존하는 유전적 상호작용.

```
경로 A: Gene_A ──→ 필수 대사산물 X ──→ 세포 생존
경로 B: Gene_B ──→ 필수 대사산물 X ──→ 세포 생존

Gene_A 단독 KO: 생존 (경로 B가 보상)
Gene_B 단독 KO: 생존 (경로 A가 보상)
Gene_A + Gene_B 이중 KO: 사멸 (두 경로 모두 차단)
```

SL 기반 치료의 이점: (1) 정상 세포는 온전한 두 경로를 모두 가지므로 손상이 최소화되고, (2) 두 번째 유전자의 억제제는 첫 번째 유전자가 이미 손상된 환자에게만 필요하므로 안전하며, (3) 단일 표적 치료 대비 내성 발생이 어렵고, (4) 종양억제유전자의 손실이라는 "치료 불가능해 보이던" 특징을 치료 가능한 취약점으로 전환할 수 있습니다.

{% hint style="info" %}
💡 **왜 "정상 세포는 안전하다"가 성립할까요?** 위 순환도를 다시 보면, 정상 세포는 Gene_A와 Gene_B 모두 온전한 상태(둘 다 야생형)입니다. 이 상태에서 약물이 Gene_B의 산물만 억제한다고 합시다. 정상 세포에서는 경로 A가 여전히 정상 작동하므로 대사산물 X 생산에 문제가 없어 생존합니다. 반면 암세포는 (예컨대 종양억제유전자 손실로) 이미 Gene_A 경로가 망가진 상태이므로, 같은 약물이 Gene_B마저 막으면 대사산물 X를 만들 방법이 완전히 사라져 사멸합니다. 즉 **같은 약물, 같은 용량**이 유전적 배경에 따라 정상 세포는 무사히 지나가고 암세포만 골라 죽이는 "자연적으로 선택적인" 결과를 만들어 냅니다. 이것이 §3.5에서 배운 "암-정상 선택성"을 얻기 위해 억지로 두 모델을 비교하지 않아도 되는, SL 고유의 강점입니다.
{% endhint %}

SL은 손상 정도에 따라 네 가지 유형(완전 치사·부분 치사·용량 치사·조건부 상보적 치사)으로 세분화할 수 있으며, 그 상세 정의와 이중/삼중 결손 계산 절차는 [Chapter 8 §2.4](../chapter-8/README.md)를 참고하십시오. 이 절에서는 SL을 **약물 표적**으로 활용하는 관점에 집중합니다.

## 4.2 PARP-BRCA: 임상에 성공한 SL 사례

BRCA1/2는 DNA 이중가닥파손(DSB)을 복구하는 **상동재조합복구(HR)**의 핵심 유전자이며, PARP는 단일가닥파손(SSB)을 복구하는 **염기절제복구(BER)**에 관여합니다.

```
BRCA1/2 정상 세포: SSB → PARP 억제 → SSB 축적 → DSB 전환 → HR로 복구 → 생존
BRCA1/2 돌연변이 세포: SSB → PARP 억제 → SSB 축적 → DSB 전환 → HR 불능 → 사멸
```

| 약물 | 대상 암 | FDA 승인 연도 |
|------|--------|-------------|
| Olaparib (Lynparza) | BRCA 돌연변이 난소암·유방암·췌장암·전립선암 | 2014 |
| Rucaparib (Rubraca) | BRCA 돌연변이 난소암 | 2016 |
| Niraparib (Zejula) | 재발성 난소암 | 2017 |
| Talazoparib (Talzenna) | BRCA 돌연변이 유방암 | 2018 |

{% hint style="warning" %}
❓ **잠깐, 생각해보기:** PARP-BRCA 사례는 엄밀히 말해 "대사(metabolism)" SL이 아니라 "DNA 복구(DNA repair)" SL입니다. 그런데 왜 대사 모델링을 다루는 이 장에서 이 사례를 소개할까요?

두 가지 이유가 있습니다. 첫째, PARP-BRCA는 SL이라는 개념 자체가 **가설로만 머물지 않고 실제 FDA 승인 약물로 이어질 수 있음**을 보여주는 가장 강력한 증거이며, 이는 §4.3~§4.4에서 다룰 대사 SL 예측(ENO1/ENO2 등)에도 같은 성공이 재현될 수 있다는 동기를 부여합니다. 둘째, SL을 검출하는 **논리 구조**(두 유전자 모두 단독 KO는 생존, 이중 KO만 치명적) 자체는 DNA 복구든 대사든 동일하며, §4.3의 이중 KO 시뮬레이션·Bliss 독립성 모델은 이 논리 구조를 대사 네트워크에 그대로 적용한 것입니다. 즉 PARP-BRCA는 "SL이라는 개념이 통한다"는 증명이고, 이후 절들은 "그 개념을 대사 네트워크의 계산 도구로 어떻게 구현하는가"를 다룹니다.
{% endhint %}

## 4.3 대사 네트워크 기반 SL 검출: 이중 KO와 Bliss 독립성

대사 네트워크에서 SL을 계산적으로 검출하려면 이중 유전자 KO 시뮬레이션을 수행합니다: 두 단일 KO 모두 비치명적(생존 > 10%)이면서, 이중 KO가 치명적(생존 < threshold)인 유전자 쌍을 찾습니다.

```python
import pandas as pd
from cobra.flux_analysis import single_gene_deletion, double_gene_deletion

def find_synthetic_lethal_pairs(
    model, candidate_genes, method='fba',
    lethal_threshold=0.01, single_viability_threshold=0.10,
):
    """이중 유전자 KO 시뮬레이션으로 합성 치사 쌍을 검출"""
    wt_biomass = model.optimize().objective_value
    single_ko = single_gene_deletion(
        model, gene_list=candidate_genes, method=method, processes=1
    )
    single_dict = {
        next(iter(ids)): growth
        for ids, growth in zip(single_ko['ids'], single_ko['growth'])
        if len(ids) == 1
    }

    double_ko = double_gene_deletion(
        model, gene_list1=candidate_genes, gene_list2=candidate_genes,
        method=method, processes=1,
    )

    sl_pairs = []
    for _, row in double_ko.iterrows():
        genes = list(row['ids'])
        if len(genes) != 2 or pd.isna(row['growth']):
            continue
        g1, g2 = genes
        # LP 허용오차가 -1e-14 같은 작은 음수 성장을 만들 수 있어 0으로 자른다.
        double_growth = max(0.0, float(row['growth']))
        s1 = max(0.0, float(single_dict.get(g1, wt_biomass)))
        s2 = max(0.0, float(single_dict.get(g2, wt_biomass)))
        if (s1 > single_viability_threshold * wt_biomass
                and s2 > single_viability_threshold * wt_biomass
                and double_growth < lethal_threshold * wt_biomass):
            sl_pairs.append({
                'gene_1': g1, 'gene_2': g2,
                'single_growth_g1': s1 / wt_biomass, 'single_growth_g2': s2 / wt_biomass,
                'double_growth': double_growth / wt_biomass,
                'sl_strength': 1 - (double_growth / min(s1, s2))
            })
    columns = ['gene_1', 'gene_2', 'single_growth_g1', 'single_growth_g2',
               'double_growth', 'sl_strength']
    return pd.DataFrame(sl_pairs, columns=columns).sort_values(
        'sl_strength', ascending=False
    )

# 원형 QP MOMA를 쓰려면 QP solver와 같은 WT reference가 필요하다.
# 무료 LP 환경에서는 먼저 method='fba' 또는 'linear moma'로 검산한다.
# sl_pairs = find_synthetic_lethal_pairs(cancer_model, candidate_genes, method='fba')
# 예상 출력 컬럼: gene_1, gene_2, single_growth_g1, single_growth_g2, double_growth, sl_strength
```

두 단일 결손 효과로부터 이중 결손의 독립 기준선을 만드는 한 방법이 **Bliss 독립성 모델**입니다.

$$Bliss\ Score = E_{AB} - (E_A \times E_B)$$

여기서는 생존율을 쓰고 `관측−기대`로 정의했으므로 음수일수록 관측 생존이 multiplicative expectation보다 낮습니다. `-0.1` 같은 값을 보편적 합성치사 경계로 쓰지 말고, 배지·측정오차·반복 실험과 연구에서 사전 정의한 생존 임계값을 함께 사용해야 합니다. 문헌이 inhibition 기준으로 반대 부호를 쓰는지도 확인하십시오.

숫자로 확인해 봅시다. 유전자 A 단독 KO 후 상대 생존율 $$E_A = 0.8$$(80% 생존), 유전자 B 단독 KO 후 $$E_B = 0.7$$(70% 생존)이라 합시다. 두 유전자가 서로 완전히 독립적이라면 이중 KO 생존율은 기대상 $$E_A \times E_B = 0.8 \times 0.7 = 0.56$$이어야 합니다. 그런데 실제로 이중 KO를 시뮬레이션했더니 생존율이 $$E_{AB}=0.4$$로 나왔다면,

$$Bliss\ Score = 0.4 - 0.56 = -0.16$$

−0.16은 관측 생존율이 독립 기대치보다 0.16 낮다는 뜻입니다. 이는 시너지 후보이지만, 이 수치 하나만으로 통계적 유의성이나 합성치사를 확정하지는 않습니다.

{% hint style="success" %}
✏️ **스스로 계산해보기:** 유전자 C 단독 KO 후 생존율이 0.9, 유전자 D 단독 KO 후 생존율이 0.85라 합시다. 두 유전자가 독립적이라면 기대 이중 KO 생존율은 얼마이며, 만약 실제 이중 KO 생존율이 0.77로 측정되었다면 Bliss Score는 얼마이고 어떻게 해석해야 할까요?

$$E_C\times E_D = 0.9\times0.85=0.765,\qquad Bliss\ Score=0.77-0.765=0.005$$

Bliss Score가 0에 매우 가깝다는 것은 관측된 이중 KO 생존율이 "두 유전자가 서로 독립적으로 작동한다"는 가정과 거의 정확히 일치한다는 뜻입니다. 즉 이 두 유전자 사이에는 시너지(합성 치사) 관계도, 길항(하나가 다른 하나의 효과를 상쇄하는) 관계도 뚜렷하게 나타나지 않습니다 — 그냥 "각자 자기 할 일을 하는" 두 개의 독립적인 유전자에 가깝습니다. 이 예시와 본문의 유전자 A·B 사례(Bliss Score = −0.16)를 비교해 보면, 같은 공식이라도 결과가 0에 가까운지 크게 음수인지에 따라 완전히 다른 생물학적 해석(독립 vs. 시너지)으로 이어진다는 것을 알 수 있습니다.
{% endhint %}

## 4.4 ENO1/ENO2 사례와 gMCS

**왜 하필 해당당화효소(enolase)일까요?** 에놀라제(enolase)는 글리콜라이시스의 아홉 번째 단계(2-포스포글리세르산 → 포스포엔올피루브산)를 촉매하는 효소로, 이 반응이 없으면 포도당에서 피루브산까지 가는 경로 자체가 끊깁니다. 즉 에놀라제는 §1.2에서 배운 Warburg 효과(글리콜라이시스 의존)의 "병목 지점" 중 하나이며, 그래서 이 유전자 자체가 없어지는 암(ENO1 결실)이 흥미로운 취약점을 남깁니다.

ENO1(1p36.23)은 다양한 암에서 자주 결실(deletion)되는 유전자로, 상실된 세포는 뉴런 특이적 이소자임(isoenzyme)인 ENO2에 의존해 글리콜라이시스를 유지합니다. 따라서 ENO1 결실 암 세포에서 ENO2를 추가로 억제하면 합성 치사가 유도됩니다. ENO3의 발현 수준이 이 SL의 강도를 완충(buffer)하므로, 완전한 치료 효과를 위해서는 ENO3도 함께 고려해야 합니다.

| 유전자 | 위치 | 발현 패턴 |
|--------|------|----------|
| ENO1 | 1p36.23 | 전신에서 광범위 발현, 다양한 암에서 자주 결실 |
| ENO2 | 12p13.31 | 뉴런 특이적 |
| ENO3 | 17p13.1 | 근육 특이적 |

이 표가 보여주는 흥미로운 지점은 **세 유전자가 서로 다른 조직 특이성을 갖는 이소자임(isoenzyme)**이라는 사실입니다. ENO1은 거의 모든 조직에서 발현되는 "기본형"이고, ENO2는 뉴런에서, ENO3는 근육에서만 주로 발현됩니다. ENO1이 결실된 암세포는 원래 자기 조직에서는 쓰지 않던 "남의 조직 전용 부품"(ENO2)을 빌려 쓰는 셈입니다 — 이는 정상적인 비신경계 조직에서는 ENO2 의존도가 낮으므로, ENO2를 억제하는 약물이 정상 세포에는 비교적 안전하면서 ENO1 결실 암세포만 선택적으로 공격할 수 있다는 뜻이기도 합니다. 이는 §3.5에서 배운 "선택성"이 SL 관계 자체에서 자연스럽게 따라 나오는 또 다른 예시입니다.

```python
def analyze_eno1_eno2_sl(model, eno1_id='ENO1', eno2_id='ENO2'):
    """ENO1 결실 배경에서 ENO2 억제의 SL 효과를 시나리오별로 분석"""
    import numpy as np

    def growth_or_zero(m):
        solution = m.optimize()
        if solution.status != "optimal" or not np.isfinite(solution.objective_value):
            return 0.0
        return max(0.0, float(solution.objective_value))

    results = {'wildtype': growth_or_zero(model)}
    with model:
        model.genes.get_by_id(eno1_id).knock_out()
        results['ENO1_KO'] = growth_or_zero(model)
    with model:
        model.genes.get_by_id(eno2_id).knock_out()
        results['ENO2_KO'] = growth_or_zero(model)
    with model:
        model.genes.get_by_id(eno1_id).knock_out()
        model.genes.get_by_id(eno2_id).knock_out()
        results['ENO1_ENO2_KO'] = growth_or_zero(model)
    return results

# 기대 결과: ENO1_KO/ENO2_KO는 생존(상호 보상), ENO1_ENO2_KO는 사멸(ENO3 부재 시)
```

더 체계적인 접근으로 **gMCS(gene Minimal Cut Sets)**는 생물량 생산을 완전히 차단하는 최소 유전자 집합을 계산하여, 조건(발현 데이터)과 결합해 맥락 특이적 SL을 찾는 gmctool의 핵심 알고리즘입니다. 단순 이중 KO 탐색보다 네트워크 토폴로지를 직접 활용해 체계적으로 취약점을 찾을 수 있다는 장점이 있습니다.

gMCS의 아이디어를 비유로 이해해 봅시다. 전력망에서 어떤 도시로 가는 전력 공급선이 여러 개(주 송전선 + 예비 송전선)라면, 그중 하나만 끊어서는 정전이 나지 않습니다. "정전을 일으키는 최소 절단 집합(minimal cut set)"은 "이 몇 개의 선을 동시에 끊어야 비로소 도시 전체가 정전된다"는 최소 조합을 말합니다. gMCS는 이 개념을 대사 네트워크에 그대로 적용합니다 — 크기 1의 gMCS는 단일 필수 유전자(§3의 KO 스크리닝과 동일)이고, 크기 2의 gMCS는 정확히 이 절에서 다루는 이중 유전자 합성 치사 쌍입니다. 즉 §3의 필수성 분석과 §4의 SL 분석은 사실 "gMCS의 크기가 1이냐 2 이상이냐"라는 하나의 통합된 틀로 볼 수 있습니다. 크기 3 이상의 gMCS(삼중 이상의 조합)까지 계산하면, 이론적으로는 §4.5에서 언급할 다중 표적 조합의 조합론적 폭발 문제를 네트워크 구조를 이용해 훨씬 효율적으로 풀 수 있습니다.

## 4.5 다중 표적 조합 최적화

$$N$$개 유전자 중 $$k$$개 조합을 전수 탐색하는 것은 $$\binom{N}{k}$$의 조합론적 폭발 문제이므로($$N=1000, k=3$$일 때 약 $$1.66\times10^8$$가지), 실무에서는 상위 단일 KO 표적(top-N)만을 대상으로 조합을 탐색하는 휴리스틱 전략을 사용합니다.

이 숫자가 왜 문제가 되는지 감을 잡아 봅시다. $$N=1000$$개 유전자 중 $$k=2$$개 조합(이중 KO)을 전수 탐색하면 $$\binom{1000}{2}=\frac{1000\times999}{2}=499{,}500$$가지이며, 이는 현대 컴퓨터로 충분히 계산 가능한 규모입니다. 그런데 $$k=3$$으로 한 단계만 늘려도 $$\binom{1000}{3}\approx1.66\times10^8$$(약 1억 6천만)가지로 폭증하고, 각 조합마다 하나의 FBA(또는 MOMA)를 풀어야 한다면 전체 계산 시간이 감당하기 어려운 수준이 됩니다. 이것이 "왜 이중 KO까지는 흔히 전수 탐색하면서 삼중 KO부터는 휴리스틱이 필요한가"에 대한 정량적 이유입니다.

{% hint style="info" %}
💡 **실무적 타협책.** 흔히 쓰이는 전략은 (1) §3의 단일 KO 스크리닝에서 상위 N개(예: 상위 50~100개) 후보만 추리고, (2) 그 안에서만 이중·삼중 조합을 탐색하거나, (3) §4.4의 gMCS처럼 네트워크 구조를 활용해 애초에 전수 탐색이 필요 없는 알고리즘을 쓰거나, (4) §4.5 본문에서 소개하는 것처럼 머신러닝으로 유망한 후보를 사전에 좁히는 것입니다. 네 가지 전략 모두 "모든 조합을 다 보지 않고도 좋은 후보를 찾는다"는 같은 목표를 서로 다른 방식으로 달성합니다.
{% endhint %}

삼중음성유방암(TNBC)에서 SDHA와 SL 관계를 갖는 30개 유전자를 식별한 MOMA-RF(MOMA + Random Forest) 연구(Kim et al. 2026)가 이러한 머신러닝 결합 전략의 대표 사례이며, §5.2에서 더 자세히 다룹니다.

{% hint style="info" %}
📌 **§4 미니 요약.** 이 절에서는 "하나만 꺼서는 안 죽지만 둘을 같이 꺼야 죽는" 합성 치사의 논리를 배웠습니다. PARP-BRCA라는 임상 성공 사례로 개념의 힘을 확인했고, 이중 KO 시뮬레이션과 Bliss 독립성 모델로 이를 대사 네트워크에서 계산하는 방법을, gMCS로 이를 네트워크 구조 관점에서 통합하는 방법을 배웠습니다. §3~§4는 모두 "무엇을 끄면 죽는가"라는 같은 질문의 변주(단일 vs. 이중/다중)였습니다. 다음 §5에서는 이 질문들을 실제 암종(간세포암·유방암·범암종 패널)에 적용한 구체적 사례 연구를 살펴보고, §6~§7에서는 질문 자체를 "무엇을 조정하면 되돌아가는가"로 뒤집습니다.
{% endhint %}
