# 8. LLM 기반 GEM 큐레이션과 지식 추출

[GEM 구조](../chapter-3/README.md)의 유전자-단백질-반응 연관(Gene-Protein-Reaction association, GPR) 정비와 [모델 품질 관리(quality control, QC)](../chapter-5/README.md)에는 많은 문헌 검토가 필요하다. **대형 언어 모델(Large Language Model, LLM)**은 대규모 텍스트 자료로 학습되어 언어 입력을 처리·생성하는 트랜스포머(Transformer) 계열 모델이다. 이 절에서는 LLM의 역할을 문헌 기반 후보 선별과 구조화된 지식 추출로 한정하고, 전문가 판정과 GEM 반영 이후의 품질 관리를 별도 단계로 다룬다.

## 8.0 자기 주의는 질의 토큰의 표현을 가중합으로 갱신한다

**자기 주의(Self-Attention)**는 시퀀스의 각 토큰이 다른 토큰의 정보를 가중합해 표현을 갱신하는 메커니즘이다. [Vaswani et al. (2017)](https://proceedings.neurips.cc/paper_files/paper/2017/hash/3f5ee243547dee91fbd053c1c4a845aa-Abstract.html)의 스케일드 내적 주의(scaled dot-product attention)는 다음과 같다.

$$
\text{Attention}(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

$$Q$$(질의, Query)는 갱신할 토큰이 찾는 정보, $$K$$(키, Key)는 각 토큰과의 관련성을 계산하는 표현, $$V$$(값, Value)는 실제 가중합할 정보를 나타낸다. $$d_k$$는 질의와 키 벡터의 차원이며, $$\sqrt{d_k}$$로 나누면 차원이 커질 때 내적의 크기가 과도하게 커져 소프트맥스가 한 항에 치우치는 현상을 완화한다. 질의 토큰 $$i$$와 키 토큰 $$j$$의 점수를 $$s_{ij}=q_i k_j^T/\sqrt{d_k}$$라 하면, 소프트맥스는 키 방향 $$j$$에 대해 $$\alpha_{ij}=\operatorname{softmax}_j(s_{ij})$$를 계산한다. 따라서 갱신되는 것은 질의 토큰 $$i$$의 표현이며, 그 출력은 $$\sum_j\alpha_{ij}v_j$$이다. 점수는 소프트맥스에 입력되는 무차원 수로 해석한다.

**손 계산 예제**: 문장 속 ‘활성’ 토큰을 질의로 두고 ‘기질’, ‘잃었다’, ‘효소’ 토큰의 키에 대한 스케일드 점수($$\sqrt{d_k}$$로 나눈 값)가 각각 $$1.5,\ 2.5,\ 0.5$$라고 가정한다. 첫 가중치는 다음과 같다.

$$
\alpha_{\text{활성},\text{기질}}
=\frac{e^{1.5}}{e^{1.5}+e^{2.5}+e^{0.5}}
=\frac{4.482}{4.482+12.182+1.649}
\approx0.245
$$

같은 방식으로 $$\alpha_{\text{활성},\text{잃었다}}\approx0.665$$, $$\alpha_{\text{활성},\text{효소}}\approx0.090$$이며, 세 값의 합은 반올림 오차 범위에서 1이다. 따라서 ‘활성’의 새 표현은 세 값 벡터를 각각 약 24.5%, 66.5%, 9.0%의 가중치로 결합한다. 이 가중치는 해당 계산에서 정보를 결합한 비율이지, 단어 사이의 생물학적 인과관계를 뜻하지 않는다.

## 8.1 Human2는 후보 선별과 최종 판정을 분리한다

**Human2** 연구는 Human1의 유전자-반응 쌍 26,246개를 UniProt과 Human Protein Atlas의 설명에 대조하는 LLM 보조 선별을 보고했다. 선별 결과는 일치(consistent) 7,774개, 불일치(inconsistent) 2,195개, 판단 불충분(inconclusive) 16,277개였다. 불일치는 논문의 **음성 예측(predicted-negative)** 범주이며, 출처 설명이 불완전하거나 모호한 판단 불충분 16,277개는 후속 분석에서 제외되었다([Luo et al., 2026](https://doi.org/10.1073/pnas.2516511123)).

논문은 음성 예측 2,195개를 사람이 검토하여 1,985개를 진음성(true negative, TN), 기능 연관이 모호한 설명에 가려졌던 210개를 위음성(false negative, FN)으로 분류했다. 논문의 음성 판정 분류 규약을 적용하면 이 부분집합의 음성예측도(Negative Predictive Value, NPV)는 다음과 같다.

$$
\mathrm{NPV}=\frac{TN}{TN+FN}=\frac{1{,}985}{1{,}985+210}=\frac{1{,}985}{2{,}195}\approx0.904=90.4\%
$$

이 값의 분모는 음성으로 예측되어 사람이 검토한 2,195개뿐이다. 따라서 약 90.4%를 양성예측도(Positive Predictive Value, PPV), 전체 26,246쌍의 정확도, 재현율(recall) 또는 자동 큐레이션 성공률로 부를 수 없다. 특히 16,277개가 판단 불충분이었으므로 이 비율만으로 선별되지 않은 항목의 성능을 계산할 수 없다.

Human2에서 보고한 후보 선별과 수동 검토에 GEM 변경에 필요한 승인·검사를 연결하면 다음과 같은 **권고 인간 참여형 검토(human-in-the-loop review)**를 구성할 수 있다. 표 전체가 Human2에 그대로 구현되었다는 뜻은 아니다. 앞의 두 단계와 논문에 명시된 모델 변경 수치는 Human2의 보고 결과이고, 변경 승인 절차와 독립 회귀 검사는 안전한 GEM 큐레이션을 위한 권고 사항이다.

| 권고 단계 | 주체와 입력 | 출력과 승인 경계 |
|:---|:---|:---|
| 후보 선별 | Human2 보고: LLM이 GPR과 출처 설명을 대조 | 일치·음성 예측·판단 불충분 후보이며 최종 생물학적 판정이 아님 |
| 전문가 검토 | Human2 보고: 사람이 음성 예측 후보와 근거를 재검토 | 진음성과 위음성 판정 |
| 공동체 큐레이션(community curation)·모델 변경 | 권고: 전문가와 공동체가 GPR, 반응식, 방향성 및 주석을 함께 검토 | 승인된 변경만 모델에 반영한다. Human2 논문은 Human-GEM 버전 1→2에서 GPR 1,864개와 반응 775개가 변경되었다고 보고한다. |
| 독립 회귀 품질 관리 | 권고: 변경 후 모델 구조, 화학량론 및 대사 과제를 독립적으로 재검사 | 회귀 오류 탐지 결과이며 개별 GPR의 생물학적 정확성 승인이 아님 |

공식 Human-GEM v2.0.0 범용 모델(generic model)은 유전자 기록 2,848개, 반응 기록 12,931개, 구획별 대사물 기록 8,461개를 포함한다. 이는 특정 릴리스(release)의 집계값이며 사람 종 자체의 고정 속성이 아니다.

| 검증 객체 | Human2 논문에서 보고한 평가 종점(endpoint) | 해석 범위 |
|:---|:---:|:---|
| 범용 Human2 | MEMOTE 총점 81% | 형식·주석·화학량론 등을 묶은 모델 시험 점수이며 모든 생물학적 예측의 정확도가 아님 |
| ec-Human2 | 선천성 대사이상 112개 시뮬레이션의 정확도 65.4% | 효소 제약 파생 모델의 질환 과제 평가 종점 |
| NCI-60 효소 제약 세포주 모델 | ec-Human1 약 79% 대비 ec-Human2 약 81%의 플럭스 일관성 | 특정 세포주 파생 모델의 플럭스 일관성 비교 |

장기 특이적(organ-specific), 전신(whole-body), 효소 제약(enzyme-constrained) 및 동적(dynamic) 모델은 범용 Human2에 조직 자료, 기관 연결, 효소 제약 또는 동역학을 추가한 파생 객체이다. 이들의 결과를 범용 Human2의 단일 성능으로 합쳐서는 안 된다. 같은 선별 처리 흐름(pipeline)을 iML1515에 적용해 표시한 1,089건도 추가 분석이 필요한 잠재 불일치이지, 검증된 오류나 자동 수정이 아니다.

## 8.2 D2Cell 사례에는 문헌 추출 결과를 검증하는 기능적 분해를 적용한다

**D2Cell**의 공개 초록은 대사공학 문헌에서 LLM으로 공학 전략을 추출하고 표준화된 세포공장 데이터베이스를 구축했다고 보고한다([Li et al., 2026](https://doi.org/10.1016/j.tibtech.2026.03.026)). 공개 초록에서 직접 확인되는 데이터베이스 규모는 표준화 항목 또는 세포공장 설계 29,006개, 생물체 751개, 생산물 1,210개이다. [저자가 공개한 데이터셋 기록](https://doi.org/10.5281/zenodo.18240770)은 해당 세포공장 데이터 파일을 식별하지만, 문서 말뭉치 수, 구현 모델과 응용 프로그래밍 인터페이스(Application Programming Interface, API), 단계의 정확한 순서, 모델별 벤치마크 수치까지 확정하지 않는다.

따라서 이 절에서는 D2Cell의 실제 구현 순서를 단정하지 않고, 문헌 추출 결과를 검증하기 위한 기능적 틀로 개체명 인식(Named Entity Recognition, NER), 관계 추출(Relation Extraction, RE), 개체 정규화(Entity Resolution, ER)를 구분한다.

1. NER은 텍스트에서 유기체, 유전자, 생산물과 같은 정해진 유형의 표현을 식별한다.
2. RE는 식별된 개체 사이의 과발현, 생산 또는 연관 관계를 구조화한다.
3. ER은 동의어와 표현 변이를 공식 데이터베이스 식별자에 연결한다.

이 기능적 분해는 각 출력의 출처 추적성과 검토 지점을 구분하는 데 유용하지만 오류를 제거하지는 않는다. NER에서 개체를 놓치면 뒤 기능이 그 관계를 복구하지 못할 수 있고, RE의 잘못된 연결이나 ER의 오정규화는 최종 표에 전파될 수 있다. 공개 초록과 데이터셋 기록에는 모델별 정밀도(precision)·재현율이나 과제 분해(task decomposition)의 개선 폭이 제시되지 않으므로 관련 벤치마크 결론은 제시하지 않는다.

| 비교 항목 | Human2 | D2Cell |
|:---|:---|:---|
| 입력 | 기존 GPR과 단백질 기능 설명 | 대사공학 문헌 |
| 자동 출력 | GPR 일치성 선별 범주 | 공학 전략의 표준화된 지식 후보 |
| 자동화 범위 | 전문가가 검토할 음성 예측 후보를 좁힘 | 문헌 정보를 구조화된 항목으로 추출함 |
| 사람의 확인 | 후보의 기능 연관 판정과 모델 변경 승인 | 근거 구절, 관계, 식별자와 추출 단위 확인 |
| 대표 미확정·실패 | 판단 불충분, 모호한 설명에 따른 위음성 | 누락 개체, 잘못된 관계, 오정규화와 기능 간 오류 전파 |
| GEM 반영 | 승인된 큐레이션과 독립 품질 관리를 거친 별도 단계 | 추출 항목 자체는 GPR·반응의 승인 또는 GEM 변경이 아님 |

## 8.3 출처 추적성과 독립 품질 관리가 GEM 반영의 조건이다

**환각(Hallucination)**은 LLM이 출처에 근거하지 않은 반응, 식별자, 경로 또는 설명을 그럴듯하게 생성하는 오류이다. 환각뿐 아니라 출처 누락, 관계 방향 오류, 동의어 오정규화도 GEM의 GPR과 반응을 잘못 바꿀 수 있다. 따라서 LLM 출력, 인간 승인, 자동 회귀 검사와 실제 GEM 반영을 같은 사건으로 기록해서는 안 된다.

최소 출처 추적성(provenance)·품질 관리 체크리스트는 다음과 같다.

- **입력과 출처 추적성**: LLM 이름·정확한 버전, 프롬프트, 입력 스냅샷(snapshot), 출처 식별자와 근거 구절 위치, 데이터베이스·모델 릴리스, 실행일을 기록한다.
- **판정과 승인**: 후보·판단 불충분·기각 상태, 근거를 검토한 사람과 승인 결과, 반영 커밋(commit) 또는 릴리스를 기록한다.
- **독립 품질 관리**: GPR 논리와 식별자, 반응의 원소·전하 수지와 방향성, 경계조건, 대사 과제, 중복 반응·막다른끝 대사물 검사와 MEMOTE 결과를 확인한다.
- **변경 후 회귀 검사**: 기존 대사 과제와 필수 기능이 유지되는지 검사하고 실패 항목을 자동 수정이 아니라 재검토 대상으로 돌린다.

### 학습 적용 미니 사례는 반영 보류 근거를 식별하게 한다

가상의 LLM이 기존 모델에 `gA and gB -> R17`이라는 GPR을 추가하자고 제안했다고 가정한다. 기록에는 모델명이 `Model-X`, 데이터베이스가 `latest`, 출처가 디지털 객체 식별자(DOI) 하나로만 적혀 있고, 정확한 모델 버전·전체 프롬프트·근거 구절 위치·데이터베이스와 GEM 릴리스·반응식과 원소·전하 수지 검사·검토자와 승인 결과는 없다.

학습자는 이 후보에 대해 (1) 누락된 출처 추적성 항목을 표시하고, (2) GPR 논리·식별자·화학량론·대사 과제 회귀 검사를 포함한 독립 검증 계획을 작성하며, (3) 인간 승인이 기록되기 전 GEM 반영 허용 여부를 판정한다. 판정에 앞서 위 체크리스트의 네 항목을 기준으로 이 기록이 충족한 것과 비어 있는 것을 각각 표시해 보면 방향을 잡을 수 있다. 판정 결과는 [마무리](summary.md)의 「답안」 절에 있다.

{% hint style="danger" %}
LLM이 생성한 GPR 규칙이나 반응은 전문가 승인과 독립 QC를 거치기 전에 GEM에 반영하지 않는다. [Chapter 5](../chapter-5/README.md)의 모델 품질관리와 화학량론 검사는 문장 근거의 확인을 보완하지만 대신하지 않는다.
{% endhint %}

Human2는 후보 선별과 모델 변경 사이에 전문가 판정을 두는 사례이며, D2Cell은 문헌을 구조화된 후보로 바꾸는 사례이다. 두 처리 흐름 모두 자동 출력의 규모가 생물학적 정확도나 모델 타당성을 직접 뜻하지 않으며, 다음 절의 기반 모델(foundation model)과 가상 세포(virtual cell)도 같은 출처 추적성·검증 경계를 유지해야 한다.
