# 11. 미래 연구 의제: 예측이 아니라 검증 기준으로 읽기

연도·시장 규모·정확도를 단정하는 로드맵은 기술 전망과 과학적 증거를 혼동하기 쉽다. 이 분야의 진전을 평가할 때는 다음 질문이 더 유용하다.

이 절이 왜 필요한지 짚고 넘어가자. §2~9에서 본 모든 방법은 "논문에서 보고한 성능 수치"와 함께 소개되었지만, 이 장 전체가 반복해서 강조했듯(§3.2의 MOMA-RF, §5.2의 MuSHIN, §7.3의 MARL 벤치마크 등) 그 수치는 항상 특정 데이터·특정 분할·특정 평가 방식이라는 맥락 안에서만 의미를 가진다. AI+GEM처럼 빠르게 변하는 분야에서 "올해 최고 성능"이라는 뉴스 헤드라인에 휩쓸리지 않으려면, 숫자 자체보다 "그 숫자가 어떤 조건에서 나왔고, 무엇을 검증했는가"를 먼저 확인하는 습관이 훨씬 오래 쓸모 있다. 아래 표는 이 장에서 다룬 각 방법론이 앞으로 더 성숙해지려면 통과해야 할 구체적인 검증 관문과, 흔히 그 관문을 눈속임으로 통과한 것처럼 보이게 만드는 함정을 정리한 것이다.

| 연구 방향 | 필요한 검증 | 실패하기 쉬운 지점 |
|:---|:---|:---|
| 미분 가능 대사 모델 | 독립 조건의 flux·성장 예측, 정확한 제약 만족 | 빠른 근사가 원 LP/MILP의 해와 달라짐 |
| 실시간 다중 오믹스 통합 | 시간 정렬 자료와 전향적 공정 예측 | 서로 다른 시간척도·배치 효과를 정적 모델에 합침 |
| 종·세포주 간 전이학습 | 학습에 없던 종/계통을 완전히 격리한 외부 검증 | 가까운 계통·공통 반응의 데이터 누출 |
| LLM 보조 큐레이션 | 후보 recall뿐 아니라 전문가 확인 precision, 출처 추적 | inconclusive를 사실로 바꾸거나 잘못된 DB 설명을 반복 |
| 자율 DBTL | 사전 정의된 목표·안전 한계 아래 실제 폐쇄 루프 성능 | 시뮬레이션 제안과 자동 실험 검증을 같은 단계로 서술 |
| 표준 벤치마크 | 모델·배지·solver·데이터 split·평가지표 버전 고정 | 서로 다른 조건의 최고 수치만 비교 |

Human2가 보여 준 현실적인 방향은 “전문가가 사라지는 자동화”가 아니라 **후보 선별 + 출처가 남는 검토 + 자동 회귀검사 + 커뮤니티 승인**이다. AI+GEM 연구도 같은 원칙으로, 성능 숫자보다 외부 검증·불확실성·재현 가능한 버전 기록을 우선해야 한다.

### 새 논문·새 도구를 만났을 때 스스로 던져볼 질문 목록

이 장을 마치고 실제로 새로운 AI+GEM 논문이나 도구를 접하게 되면, 다음 다섯 가지 질문을 순서대로 점검해보길 권한다. 이 목록은 이 장에서 반복해서 사용한 비판적 읽기 방식을 체크리스트로 압축한 것이다.

1. **어떤 데이터로 학습·평가했는가?** ([§3.3](03.md)의 세포주 단위 분할처럼) 훈련과 테스트가 진짜로 독립적인가, 아니면 데이터 유출 가능성이 있는가?
2. **보고된 지표가 무엇인가?** 정확도만 보고했다면 ([§2.6](02.md)에서 봤듯) 클래스 불균형을 감춘 것은 아닌지 MCC·Recall도 확인했는가?
3. **"인공적으로 만든 검증"과 "독립적인 실측 검증"을 구분했는가?** ([§5.2](05.md)의 인공 제거 반응 복원 vs. 실제 미지 반응 발견처럼) 내부 benchmark 성능을 외부 적용 성능으로 착각하지 않았는가?
4. **화학량론·물리적 제약을 실제로 지키는가, 아니면 그럴듯해 보일 뿐인가?** ([§10.2](10.md)) 출력을 $$\mathbf{S}\mathbf{v}=\mathbf{0}$$과 flux bounds로 직접 재확인했는가?
5. **불확실성·실패 사례를 함께 보고하는가?** ([§10.4](10.md)) 성공 사례만 강조하고 실패·한계는 언급하지 않는 논문은 한 번 더 의심해볼 필요가 있다.

이 다섯 질문은 특정 논문을 폄훼하기 위한 것이 아니라, 이 장에서 배운 개념(교차검증, 혼동 행렬, 화학량론적 타당성, 보정)을 실전에서 활용하는 습관을 들이기 위한 것이다. [Chapter 10](../chapter-10/README.md)의 실습에서 같은 습관 — 코드로 직접 재현하고, 기대 출력과 실제 출력을 비교하는 것 — 을 다시 연습하게 된다.
