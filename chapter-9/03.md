# 3. 유전자 필수성 예측: ML과 GEM의 첫 만남

**[유전자 필수성](../glossary.md)(Gene Essentiality)** 예측은 항생제 개발, 암 치료 표적 발굴([Chapter 7](../chapter-7/README.md)), 대사공학([Chapter 8](../chapter-8/README.md))에서 핵심적 역할을 한다. 필수성 자체의 생물학적 응용은 해당 장에서 다루며, 이 절은 **ML이 필수성 예측 정확도를 어떻게 끌어올리는가**에 집중한다.

이 절이 이 장에서 가장 먼저 등장하는 이유는 다음과 같다. [§2](02.md)에서 배운 로지스틱 회귀·Random Forest·혼동 행렬·MCC는 전부 "레이블이 있는 분류 문제"를 다루는 도구였다. 유전자 필수성 예측은 정확히 그 형태(Y=필수/비필수의 이진 레이블)를 띠면서 동시에 [Chapter 8](../chapter-8/README.md)의 FBA 결손 시뮬레이션이라는 익숙한 생물학적 배경을 갖고 있어, ML의 개념을 GEM에 처음 접목해보는 가장 자연스러운 출발점이다.

## 3.1 전통적 FBA 기반 접근의 한계와 DepMap

[Chapter 8](../chapter-8/README.md)에서 본 전통적 *in silico* 유전자 제거는 각 유전자 $$g$$를 제거해 FBA로 최대 생물량 $$Z_{KO}$$를 계산하고, $$Z_{KO} < \alpha \cdot Z_{WT}$$(보통 $$\alpha=0.1$$)이면 필수로 판정한다. 이 접근의 한계는 (1) 목적함수 의존, (2) 클래스 불균형(필수 유전자 10~20%), (3) 조건 의존성, (4) GPR 규칙 부정확성이다.

**Cancer Dependency Map([DepMap](https://depmap.org/portal/))**은 대규모 암 세포주 CRISPR-Cas9 스크린을 제공하는 대표 자료원이다. DepMap의 Chronos gene-effect 원점수는 일반적으로 더 음수일수록 강한 의존성을 뜻한다. 아래 Kim et al. 연구는 이를 별도의 0~1 **GDS/probabilistic lethality** 체계로 가공해 0.5 이상을 essential label로 사용했다. 서로 방향이 반대인 두 점수 체계를 같은 임계값 표에 섞지 말아야 한다.

{% hint style="warning" %}
**해석상의 주의:** "필수성 레이블은 실험으로 측정한 절대적 사실이다." — CRISPR 스크린도, FBA 결손 시뮬레이션도 모두 특정 조건(배지, 세포주, 임계값)에서 얻은 **측정·계산 결과**일 뿐이다. §2.1에서 이미 짚었듯 `e_coli_core`의 "필수 유전자 7개"조차 COBRApy 기본 배지와 1% 성장 임계값이라는 특정 선택의 산물이다. DepMap의 Chronos 점수도 CRISPR 라이브러리·세포주 배양 조건·통계 보정 방법에 따라 달라질 수 있는 정량값이지, 절대 진리표가 아니다.
{% endhint %}

## 3.2 MOMA-RF 통합: 민감도 0.37→0.55 (Kim et al., 2026)

[Chapter 8](../chapter-8/README.md)에서 배운 **MOMA(Minimization of Metabolic Adjustment)**는 유전자 제거 후 WT 상태에 가장 가까운 플럭스 분포를 찾는다("근접성 원칙").

$$\min \sum_i (v_i - v_i^{WT})^2 \quad \text{s.t. } \mathbf{S} \cdot \mathbf{v} = 0, \; \mathbf{v}_{lb} \leq \mathbf{v} \leq \mathbf{v}_{ub}$$

[Kim et al. (2026)](https://doi.org/10.3390/ijms27115059)은 DepMap RNA-seq로 **50개 유방암 세포주**(luminal A 8, luminal B 7, HER2-positive 10, TNBC 25)의 모델을 [tINIT](../landmark-papers.md)([Agren et al., 2014](https://doi.org/10.1002/msb.145122)) + Recon2M.2(Recon2M 계보, [Ryu, Kim & Lee 2017](https://doi.org/10.1073/pnas.1713050114))로 재구성했다. 각 유전자 KO에 대해 MOMA가 만든 **반응 flux 벡터**를 ML 입력 특징으로, DepMap gene-dependency label을 정답으로 사용했다. 즉 실제 MOMA-RF 입력은 이 절의 교육용 degree/betweenness 특징 목록이 아니라, 세포주 특이적 MOMA flux 재분포이다.

| 지표 | MOMA 단독 | MOMA-RF | 변화 |
|:---|:---:|:---:|:---:|
| Sensitivity | 0.37 | 0.55 | **+49%** |
| MCC | 0.27 | 0.33 | **+22%** |
| Accuracy | 0.86 | 0.84 | -2%p |
| Specificity | 0.92 | 0.87 | -5%p |
| Precision | 0.33 | 0.31 | -2%p |

민감도 증가는 **18 percentage points**, 상대값으로 약 49%이다. MCC도 0.27에서 0.33으로 증가했지만 precision은 0.33에서 0.31로 낮아졌다. 따라서 “모든 면에서 정확해졌다”가 아니라, 불균형 자료에서 더 많은 참 의존성을 회수하는 대신 일부 거짓양성을 감수한 결과로 읽어야 한다. 차이는 TNBC·HER2-positive 세포주에서 통계적으로 유의했고 luminal A/B에서는 유의하지 않은 경향이었다.

{% hint style="info" %}
**잠깐, 생각해보기:** Accuracy가 소폭 낮아졌는데(0.86→0.84) 왜 일부 측면의 개선이라 부를까? 민감도와 MCC는 증가했지만 precision은 감소했다. 따라서 단일 숫자로 “우월”하다고 말하지 말고, 표적 누락과 거짓양성의 비용을 함께 보고 용도에 맞는 지표를 선택해야 한다.
{% endhint %}

## 3.3 분류기 선택의 검증 방법

논문은 SVM·logistic regression·random forest(RF)·neural network를 비교하고, 클래스 불균형 처리로 oversampling, random undersampling, SMOTE-ENN을 시험했다. RF와 neural network가 대표 TNBC 모델에서 높은 민감도를 보였고, 25개 TNBC 세포주로 확장했을 때 RF의 세포주 간 분산이 더 작아 최종 분류기로 선택되었다. 정확한 비교값은 특정 학습/검증 분할과 undersampling에 종속되므로, 다른 연구에서 RF가 자동으로 최선이라는 결론은 아니다. 핵심 재현 항목은 **세포주 단위 외부 검증, fold 내부 특징 선택, hold-out test 격리, 불균형 처리와 임계값 기록**이다.

### 데이터 누출(data leakage)이라는 함정: "세포주 단위"로 나눠야 하는 이유

방금 언급한 "세포주 단위 외부 검증"이 특별히 강조되는 이유를 짚어 둔다. 50개 세포주 각각에서 유전자 하나하나가 예제(row) 하나가 된다면, 같은 세포주에서 나온 수천 개의 유전자 예제들은 그 세포주 고유의 배경(성장 속도, 돌연변이 스펙트럼)을 공유한다. 만약 [§2.5](02.md)의 훈련/테스트 분할을 유전자 단위로 무작위로 하면, 같은 세포주의 다른 유전자들이 훈련 세트와 테스트 세트에 동시에 섞여 들어갈 수 있다. 그러면 모델은 "이 세포주는 이런 패턴을 보인다"는 정보를 훈련 중에 이미 살짝 엿본 뒤 같은 세포주의 테스트 예제를 맞히는 셈이 된다. 결국 학습에 쓰지 않은 새로운 세포주에 적용했을 때보다 성능이 부풀려진다. 이를 **데이터 누출(data leakage)**이라 부른다. 올바른 검증은 특정 세포주 전체를 통째로 테스트 세트로 남겨두는 것이다 — 이것이 "세포주 단위(cell-line-level) 분할"의 의미다.

{% hint style="warning" %}
**해석상의 주의:** "Stratified K-Fold를 썼으니 데이터 유출 문제는 해결됐다." — [§2.5](02.md)의 Stratified K-Fold는 클래스 비율(필수/비필수)만 fold마다 맞춰줄 뿐, 같은 세포주의 예제가 훈련·테스트에 함께 섞이는 문제까지 막아주지는 않는다. 이런 상황에서는 `sklearn.model_selection.GroupKFold`처럼 "세포주 ID"를 그룹으로 지정해 같은 그룹이 훈련·테스트에 걸쳐 나뉘지 않도록 하는 별도의 분할 전략이 필요하다.
{% endhint %}

## 3.4 e_coli_core로 특징 행렬과 반응 필수성 레이블 만들기

아래 코드는 실제 연구의 유전자 필수성 예측을 축소 재현한 것이 아니라, 반응을 관측 단위로 삼는 **교육용 [반응 필수성](../glossary.md) 분류** 예제다. 유전자 필수성은 [GPR](../chapter-3/README.md)의 AND/OR 규칙을 평가한 `single_gene_deletion()`으로 별도로 정의해야 하며, 두 레이블을 서로 바꾸어 부르지 않는다.

§2.2에서 만든 그래프 `G`를 이어받아, 이제 실제로 학습 가능한 특징 행렬을 완성한다. 아래 코드의 기대 출력(야생형 성장률, 필수/비필수 반응 수)은 COBRApy 0.30.0의 `textbook` 모델(`e_coli_core`)을 기본 배지와 바이오매스 목적함수로 풀고 야생형 대비 1% 성장을 필수성 임계값으로 둔 조건에서 얻은 값이다. 원 강의자료에는 solver 종류와 허용오차가 기록되지 않았으므로 이 두 항목은 결손이며, 재현할 때는 [Chapter 10 §1](../chapter-10/02.md)의 preflight 절차대로 `cobra.Configuration().solver`를 명시적으로 고정하고 그 값을 함께 기록한다. 성장률은 solver를 바꿔도 소수점 셋째 자리까지는 대체로 유지되지만, 필수/비필수 개수는 임계값 근처 반응에서 solver의 대안 최적해와 허용오차에 따라 갈릴 수 있다.

```python
import numpy as np
import pandas as pd

# 위상 특징: 연결 차수와 매개 중심성 (95개 노드라 전체 계산도 빠르다)
degree = dict(G.degree())
betweenness = nx.betweenness_centrality(G)

# 플럭스 특징: 야생형 FBA (4장에서 배운 것과 동일)
wt = model.optimize()
print(f"야생형 성장률: {wt.objective_value:.3f} h^-1")
# 기대 출력: 야생형 성장률: 0.874 h^-1

rows = []
for rxn in model.reactions:
    rows.append({
        'reaction': rxn.id,
        'degree': degree[rxn.id],
        'betweenness': betweenness[rxn.id],
        'wt_flux': abs(wt.fluxes[rxn.id]),
    })
features = pd.DataFrame(rows).set_index('reaction')

# 특징의 관측 단위가 반응이므로 레이블도 반응 자체를 하나씩 제거해 만든다.
# "연결된 유전자가 모두 필수" 같은 proxy는 isozyme GPR을 잘못 처리할 수 있다.
from cobra.flux_analysis import single_reaction_deletion

reaction_ko = single_reaction_deletion(model, processes=1)
ko_growth = {
    next(iter(ids)): float(growth)
    for ids, growth in zip(reaction_ko["ids"], reaction_ko["growth"])
}
features["essential"] = [
    int(not np.isfinite(ko_growth[rid])
        or ko_growth[rid] < 0.01 * wt.objective_value)
    for rid in features.index
]
print(features['essential'].value_counts())
# 기대 출력: non-essential 77, essential 18
```

```python
# Random Forest로 반응 필수성 예측
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier

X = features.drop(columns='essential')
y = features['essential']

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42, stratify=y
)

rf = RandomForestClassifier(
    n_estimators=200, max_depth=8,
    class_weight='balanced', random_state=42
)
rf.fit(X_train, y_train)

for name, imp in sorted(zip(X.columns, rf.feature_importances_),
                         key=lambda x: x[1], reverse=True):
    print(f"{name}: {imp:.3f}")
```

이 코드는 §2.6에서 배운 ROC·MCC 평가, §2.3의 K-Means 클러스터링과 함께 이 장 끝의 "실습" 절에서 하나의 완결된 파이프라인으로 이어진다.

`rf.feature_importances_`가 출력하는 숫자의 해석 범위는 제한적이다. RF의 특징 중요도는 대체로 "그 특징으로 트리를 분기했을 때 불순도(impurity, 클래스가 얼마나 섞여 있는지)가 얼마나 줄었는가"를 트리 전체에서 평균 낸 값이며, 모든 특징의 중요도 합은 1이 되도록 정규화되는 경우가 많다. 예를 들어 `degree`의 중요도가 0.45라면 "이 모델이 필수성을 가르는 데 사용한 정보의 약 45%가 연결 차수에서 왔다"는 뜻이지, "연결 차수가 높으면 45% 확률로 필수"라는 뜻이 아니다. 또한 이 값은 상관된 특징(예: `degree`와 `betweenness`는 서로 어느 정도 상관된 위상 지표다)이 있으면 중요도가 여러 특징에 나뉘어 낮게 보일 수 있다는 한계도 있다 — 특징 하나를 빼고 다시 학습해 성능 변화를 보는 **순열 중요도(Permutation Importance)**가 더 안정적인 대안으로 쓰인다.

## 3.5 편향-분산 트레이드오프: 작은 데이터셋에서 모델 복잡도 고르기

§3.4에서 만든 `e_coli_core` 반응 데이터셋은 훈련 예제가 겨우 95개(테스트 분할 후에는 그마저 66개 정도)뿐이다. 이렇게 작은 데이터셋에서 RF의 `max_depth`를 어떻게 고르느냐는 단순한 튜닝 이상의 근본적인 문제, 즉 **편향-분산 트레이드오프(Bias-Variance Tradeoff)**와 맞닿아 있다.

- **편향(Bias)**이 크다는 것은 모델이 너무 단순해서 데이터의 실제 패턴조차 잡아내지 못하는 상태다. 예컨대 `max_depth=1`인 트리(그루터기, stump) 하나는 "연결 차수가 3 이상이면 필수"처럼 규칙 하나만 배울 수 있어, 훈련 데이터에서조차 정확도가 낮다 — 이를 **과소적합(Underfitting)**이라 부른다.
- **분산(Variance)**이 크다는 것은 모델이 너무 복잡해서 훈련 데이터의 사소한 잡음(noise)까지 암기해버리는 상태다. `max_depth`에 제한을 두지 않으면 트리는 95개 반응 각각을 구별하는 규칙을 끝없이 만들어내 훈련 정확도는 100%에 가까워지지만, 새로운 반응(테스트 세트)에는 그 암기한 규칙이 통하지 않는다 — 이를 **과적합(Overfitting)**이라 부른다.

두 극단 사이 어딘가에 훈련 오차와 테스트 오차의 합이 최소가 되는 "적당한 복잡도"가 있다. 이를 숫자로 가늠하면 다음과 같은 경향이 나타난다(구체적 수치는 실행 환경마다 다르므로 예시로만 읽는다).

| `max_depth` | 훈련 정확도(경향) | 테스트 정확도(경향) | 해석 |
|:---:|:---:|:---:|:---|
| 1 | 낮음 | 낮음 | 과소적합 — 편향이 큼 |
| 3~5 | 중간~높음 | 이 근처에서 최고 | 편향과 분산의 균형 |
| 제한 없음(예: 20+) | 매우 높음(≈100%) | 오히려 하락 가능 | 과적합 — 분산이 큼 |

{% hint style="info" %}
**잠깐, 생각해보기:** 데이터가 95개뿐인 `e_coli_core` 예제와, 유전자 1,515개인 iML1515로 같은 실습을 한다면 어느 쪽이 깊은 트리(`max_depth`가 큰 트리)를 더 안전하게 쓸 수 있을까? 일반적으로 데이터가 많을수록 분산이 줄어드는 경향이 있어(암기할 잡음 대비 진짜 패턴의 비율이 높아지므로) 더 복잡한 모델을 써도 과적합 위험이 상대적으로 낮아진다. 반대로 `e_coli_core`처럼 표본이 작을 때는 `max_depth`를 낮게, `class_weight='balanced'`를 유지하고, 가능하면 §2.5의 K-Fold 교차검증으로 여러 분할의 평균 성능을 확인하는 것이 안전하다.
{% endhint %}

향후 확장 방향은 다중 오믹스 통합, 세포주 간 전이 학습(Transfer Learning), 데이터가 축적되면 GNN(§4)으로의 전환, 조건 의존 필수성 예측이다.
