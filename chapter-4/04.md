# 4. LP를 푸는 법: 심플렉스법과 내점법, 그리고 COBRApy 솔버

## 4.1 심플렉스 알고리즘(Simplex Algorithm)

1947년 George Dantzig가 개발한 심플렉스법은 3.3절에서 확인한 기본 정리를 그대로 활용한다: 가능 폴리토프의 한 꼭짓점에서 시작해, 목적 함수값을 개선하는 인접 꼭짓점으로 이동하는 것을 반복하다가, 더는 개선할 인접 꼭짓점이 없으면 종료한다. 3.3절의 장난감 예제에 비유하면, 심플렉스법은 예를 들어 $$(0,0)\to(6,0)\to(6,4)\to(2,8)$$ 순으로 인접 꼭짓점을 옮겨 다니며 $$Z$$가 계속 증가하는 것을 확인하다가, $$(2,8)$$에서 더 이상 개선되는 인접 꼭짓점이 없으므로 멈추는 것과 같다.

```
┌──────────────────────────────────────────────────────┐
│              심플렉스 알고리즘의 흐름                  │
│  1. 기본 실행가능해(Basic Feasible Solution) 선택      │
│         ↓                                             │
│  2. 목적 함수 개선 방향 확인                          │
│         ↓                                             │
│  3. 개선되는 인접 꼭짓점이 있는가?                    │
│     Yes → 이동 → 4로                                  │
│     No  → 현재 해가 최적 → 종료                        │
│  4. 새로운 기본 실행가능해로 갱신 → 2로 돌아감         │
└──────────────────────────────────────────────────────┘
```

| 특성 | 설명 |
|:---|:---|
| 최악 시간 복잡도 | 지수 시간 $$O(2^n)$$ (Klee–Minty 예제) |
| 평균 실제 성능 | 다항 시간에 가까움 — genome-scale FBA는 보통 1초 미만 |
| 메모리 효율성 | 기저(Basis) 역행렬만 유지 |
| Warm-start | 이전 해에서 가까운 문제를 매우 빠르게 재해결 가능 |

## 4.2 내점법(Interior-Point Method)

1984년 Narendra Karmarkar가 개발한 내점법은 꼭짓점을 순회하는 대신, 가능 영역의 **내부**를 따라 **중앙 경로(Central Path)**를 추적하며 최적해로 수렴한다. 핵심 아이디어는 경계 접근을 억제하는 **장벽 함수(Barrier Function)**이다.

$$\min_{\mathbf{x}} \; \mathbf{c}^\mathsf{T}\mathbf{x} - \mu \sum_{i} \ln(x_i)$$

장벽 매개변수 $$\mu>0$$을 점차 0으로 줄이며 중앙 경로를 따라 최적점에 접근한다.

| 특성 | 심플렉스법 | 내점법 |
|:---|:---|:---|
| 탐색 방식 | 꼭짓점 간 이동 | 내부 경로 추적 |
| 최악 시간 복잡도 | 지수 | 다항 $$O(n^{3.5}L)$$ |
| 대규모 문제 성능 | 선형~초선형 | 우수 (희소 행렬 연산 최적화) |
| Warm-start | 우수 | 제한적 |
| 병렬화 | 기저 갱신의 순차적 의존성 | 희소 행렬 연산의 병렬화 가능 |

현대 LP 솔버(Gurobi, CPLEX)는 문제 구조를 자동 분석해 두 알고리즘 중 하나를 선택하거나, 둘을 동시에 돌려 먼저 끝난 결과를 반환한다.

## 4.3 COBRApy 솔버 인터페이스

[COBRApy](https://opencobra.github.io/cobrapy/)는 **[optlang](https://github.com/opencobra/optlang)** 라이브러리를 통해 여러 LP 솔버를 동일한 Python 인터페이스로 다룰 수 있게 해준다.

| 솔버 | 종류 | 라이선스 | 권장 사용 상황 |
|:---|:---|:---|:---|
| **[GLPK](https://www.gnu.org/software/glpk/)** | 오픈소스 | GPL | 소규모 모델, 교육 목적. COBRApy 기본 포함 |
| **[Gurobi](https://www.gurobi.com/)** | 상용 | 유료(학술 라이선스 제공) | 대규모 LP/MILP와 QP MOMA |
| **[CPLEX](https://www.ibm.com/products/ilog-cplex-optimization-studio)** | 상용 | 유료(학술 라이선스 제공) | 대규모 LP/MILP와 QP MOMA |
| **[SCIP](https://www.scipopt.org/)** | 오픈소스 | ZIB Academic | MILP(혼합정수계획법) 문제 |

```python
import cobra
from cobra.io import load_model

model = load_model("textbook")   # e_coli_core, Chapter 1~3에서 쓴 것과 동일한 모델

# 현재 솔버 확인 및 변경
print(f"현재 솔버: {model.solver.interface.__name__}")
model.solver = "glpk"           # 또는 "gurobi", "cplex"

# 솔버 세부 설정
model.solver.configuration.timeout = 30       # 30초 타임아웃
model.solver.configuration.presolve = True    # 프리솔브 활성화
```

`e_coli_core`처럼 작은 모델은 GLPK로도 충분히 빠르지만, FVA처럼 반응마다 최소화·최대화를 반복하는 대규모 계산에서는 상용 솔버가 유리할 수 있다. GLPK는 LP/MILP는 풀지만 QP는 지원하지 않으므로 [Chapter 8](../chapter-8/README.md)의 **quadratic [MOMA](../glossary.md)**에는 QP를 지원하는 optlang 솔버가 필요하다. COBRApy의 `moma()` 기본값은 계산이 쉬운 `linear=True`(linear MOMA)라는 점에도 유의한다.

## 4.4 심플렉스를 손으로 돌려보기: 타블로 방법

4.1절에서 "인접 꼭짓점으로 이동한다"고 말로 설명한 과정을, 이번에는 3.1절의 장난감 LP에 2.5절의 슬랙 변수를 적용해 **숫자로 직접** 따라간다. 세 부등식 제약에 슬랙 변수 $$s_1, s_2, s_3 \ge 0$$을 하나씩 붙이면 완전한 등식 표준형이 된다.

$$
\begin{aligned}
\max \quad & Z = 0.5v_1+0.8v_2 \\
\text{s.t.} \quad & v_1+v_2+s_1=10 \\
& v_1+s_2=6 \\
& v_2+s_3=8 \\
& v_1,v_2,s_1,s_2,s_3\ge0
\end{aligned}
$$

**초기 꼭짓점(0,0).** $$v_1=v_2=0$$을 비기저(nonbasic) 변수로 두면, $$s_1=10,\ s_2=6,\ s_3=8$$이 기저(basic) 변수가 되어 원점 $$(0,0)$$에 대응하는 실행가능해가 된다. 이때 $$Z=0$$이다.

**1회전 — 진입 변수 선택.** 목적 함수 $$Z=0.5v_1+0.8v_2$$에서 $$v_2$$의 계수($$0.8$$)가 $$v_1$$의 계수($$0.5$$)보다 크므로, 단위당 개선 폭이 더 큰 $$v_2$$를 진입 변수로 선택한다(가장 가파른 방향을 고르는 **단테치히 규칙, Dantzig's Rule**). 얼마나 늘릴 수 있는지는 $$v_2$$가 등장하는 각 행에서 "비율 검정(ratio test)"으로 정한다.

| 제약행 | $$v_2$$의 계수 | 우변 | 비율(우변/계수) |
|:---|:---:|:---:|:---:|
| $$v_1+v_2+s_1=10$$ | $$1$$ | $$10$$ | $$10$$ |
| $$v_2+s_3=8$$ | $$1$$ | $$8$$ | $$\mathbf{8}$$ ← 최소 |

최소 비율은 $$8$$이므로 $$s_3$$가 이탈(leaving) 변수가 되고, $$v_2$$는 $$8$$까지만 늘어난 뒤 멈춘다 — 더 늘리면 $$s_3<0$$이 되어 실행 불가능해지기 때문이다. 이 결과는 정확히 3.2절 오각형에서 위쪽 변 $$v_2=8$$에 도달했다는 뜻이다. 새 꼭짓점은 $$(v_1,v_2)=(0,8)$$이고, $$Z=0.8(8)=6.4$$로 개선되었다.

**2회전 — 다시 진입 변수 선택.** $$s_3$$를 $$v_2=8-s_3$$로 치환해 다른 식에 대입하면 새로운 목적식을 얻는다.

$$Z = 0.5v_1+0.8(8-s_3)+6.4-6.4 = 6.4+0.5v_1-0.8s_3$$

비기저 변수 $$v_1$$의 계수가 여전히 $$+0.5$$로 양수이므로 개선 여지가 남아 있다. $$v_1$$을 진입 변수로 다시 비율 검정을 한다.

| 제약행(치환 후) | $$v_1$$의 계수 | 우변 | 비율 |
|:---|:---:|:---:|:---:|
| $$v_1+s_1-s_3=2$$ | $$1$$ | $$2$$ | $$\mathbf{2}$$ ← 최소 |
| $$v_1+s_2=6$$ | $$1$$ | $$6$$ | $$6$$ |

최소 비율 $$2$$이므로 $$s_1$$이 이탈하고, $$v_1$$은 $$2$$까지 늘어난다. 새 꼭짓점은 $$(v_1,v_2)=(2,8)$$ — 정확히 3.3절에서 손으로 찾은 그 최적점이다! $$Z=6.4+0.5(2)=7.4$$로 다시 개선되었다.

**3회전 — 종료 판정.** $$v_1=2-s_1+s_3$$을 목적식에 대입해 최종 형태로 정리한다.

$$Z = 7.4 - 0.5\,s_1 - 0.3\,s_3$$

이제 비기저 변수($$s_1, s_3$$)의 계수가 모두 **음수**이다. 즉 지금 값이 0인 두 변수 중 어느 것을 늘려도 $$Z$$는 오히려 줄어든다 — 더 이상 개선할 인접 꼭짓점이 없다는 뜻이므로 심플렉스는 여기서 멈춘다. 최종적으로 $$(v_1^*,v_2^*)=(2,8)$$, $$Z^*=7.4$$를 얻어 3.3절의 결과와 정확히 일치한다.

```
(0,0) --[v2 진입, s3 이탈]--> (0,8) --[v1 진입, s1 이탈]--> (2,8) = 최적
   Z=0                Z=6.4                    Z*=7.4
```

{% hint style="info" %}
**팁:** 4.1절 서두에서는 예시 경로로 $$(0,0)\to(6,0)\to(6,4)\to(2,8)$$을 들었는데, 방금 손으로 돌린 타블로는 $$(0,0)\to(0,8)\to(2,8)$$로 단 두 번만에 도착했다. 모순이 아니다 — 진입 변수를 고르는 규칙(단테치히 규칙 등)에 따라 심플렉스가 지나가는 경로는 달라질 수 있지만, 가능 영역이 볼록이고 유계인 한 **도착하는 최적 꼭짓점은 같다.** 어떤 경로를 택하느냐는 계산 효율성의 문제이지, 최적해 자체를 바꾸지 않는다.
{% endhint %}

**최종 타블로가 말해주는 것 — 5절로 가는 다리.** 방금 얻은 $$Z=7.4-0.5s_1-0.3s_3$$이라는 식은 우연히 나온 것이 아니다. $$s_1$$은 제약 $$v_1+v_2\le10$$의 슬랙이었고, 그 계수 $$0.5$$는 5.2절에서 포도당 공급 한계를 $$10\to11$$로 완화했을 때 손으로 확인한 $$\Delta Z^*=0.5$$와 정확히 같다. $$s_3$$는 제약 $$v_2\le8$$의 슬랙이었고, 그 계수 $$0.3$$은 5.3절에서 경로 2의 상한을 $$8\to9$$로 늘렸을 때 확인한 $$\Delta Z^*=0.3$$과 정확히 같다. 즉 **최종 심플렉스 타블로에서 슬랙 변수의 계수(부호를 바꾼 값)가 바로 그 제약의 쌍대 변수(그림자 가격·환원비용)이다.** 반면 $$v_1=2$$의 상한(6)에 대응하는 슬랙 $$s_2=4$$는 여전히 기저 변수로 남아 값이 0이 아니므로("빠듯하지 않음"), 그 제약의 쌍대 값은 0이다 — 5.1절에서 "빠듯한 제약만 0이 아닌 쌍대 값을 가진다"고 말한 것이 바로 이 현상이다. 심플렉스법이 원 문제를 풀어 최적 꼭짓점을 찾는 바로 그 계산 과정 안에, 쌍대 문제의 답이 이미 함께 들어 있는 것이다.

## 4.5 최적성 조건 미리보기: KKT

방금 확인한 "모든 비기저 변수의 계수가 0 이하가 되면 종료"라는 심플렉스의 정지 조건은, 사실 더 일반적인 최적화 이론의 한 사례이다. 부등식 제약이 있는 최적화 문제의 최적해가 만족해야 하는 조건을 **KKT 조건(Karush–Kuhn–Tucker Conditions)**이라 부른다. FBA의 LP에 대해 KKT 조건을 쓰면 다음 네 가지로 요약된다.

1. **원 실행가능성(Primal Feasibility):** $$\mathbf{S}\mathbf{v}=\mathbf{0}$$, $$\mathbf{v}^{lb}\le\mathbf{v}\le\mathbf{v}^{ub}$$ — 지금까지 다룬 원래 제약이다.
2. **쌍대 실행가능성(Dual Feasibility):** 부등식 제약에 대응하는 쌍대 변수의 부호가 정해진 범위(관례에 따라 $$\ge0$$ 또는 $$\le0$$) 안에 있어야 한다 — 5.1절에서 도입할 $$\boldsymbol\alpha,\boldsymbol\beta$$가 이에 해당한다.
3. **정상성(Stationarity):** 목적 함수의 기울기가 활성 제약들의 기울기의 조합으로 정확히 상쇄되어야 한다 — "더 이상 개선 방향이 없다"는 조건으로, 방금 타블로에서 확인한 "모든 비기저 변수 계수 $$\le0$$"과 같은 내용이다. 정확한 부호 규약은 등식을 어느 방향으로 쓰느냐에 따라 달라지므로 5.6절에서 하나의 구체적 규약을 정해 끝까지 일관되게 유도해 본다.
4. **상보 여유성(Complementary Slackness):** 각 부등식 제약과 그 쌍대 변수의 곱이 0 — 예를 들어 제약이 빠듯하지 않으면($$s_2=4\neq0$$) 그 쌍대 변수는 반드시 0이라는 것으로, 방금 확인한 "$$s_2$$가 기저에 남아 있으면 쌍대 값 0" 현상과 정확히 같다.

이 네 조건의 완전한 유도와 각 기호의 의미는 5.6절에서 쌍대 LP를 처음부터 다시 만들어 확인한다. 여기서는 "심플렉스의 정지 조건 = KKT 조건의 한 사례"라는 사실만 기억해 두면 충분하다.
