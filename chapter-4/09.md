# 9. Flux Variability Analysis (FVA)

## 9.1 FVA의 목적과 정형화

표준 FBA는 솔버가 고른 한 최적 플럭스 벡터를 반환하지만, 3.4절과 8절에서 본 대로 이 해는 유일하지 않을 수 있습니다. \*\*FVA(Flux Variability Analysis)\*\*는 이 비유일성을 정량화합니다: 목적 함수가 최적값의 일정 비율 $$f$$ 이상을 유지한다는 조건 아래, 각 반응 $$j$$의 플럭스가 가질 수 있는 최솟값과 최댓값을 계산합니다.

$$v_j^{\min} = \min_{\mathbf{v}} v_j, \qquad v_j^{\max} = \max_{\mathbf{v}} v_j$$

$$\text{s.t.} \quad \mathbf{S}\mathbf{v}=\mathbf{0}, \quad \mathbf{v}^{lb}\le\mathbf{v}\le\mathbf{v}^{ub}, \quad Z \ge f\cdot Z^*$$

반응마다 최소·최대 두 번의 LP를 풀어야 하므로 총 $$2n$$개의 LP가 필요합니다.

**3절의 장난감 예제로 FVA 확인하기.** 3.4절의 대안 최적해 상황(목적함수 $$0.5v_1+0.5v_2$$, $$f=1.0$$)에서 FVA를 손으로 계산해 봅시다. 최적 면은 $$v_1+v_2=10$$이면서 $$2\le v_1\le6$$(동시에 $$4\le v_2\le8$$)인 구간입니다. 따라서:

$$v_1^{\min}=2,\ v_1^{\max}=6 \qquad v_2^{\min}=4,\ v_2^{\max}=8$$

FVA는 정확히 이 범위 $$[2,6]$$과 $$[4,8]$$을 보고합니다 — "두 경로 사이에서 정확히 어떻게 나뉘는지는 알 수 없지만, 각각 이 범위 안에서 움직일 수 있다"는 정직한 답입니다.

|   $$f$$ 값  | 해석        | 적용 상황              |
| :--------: | --------- | ------------------ |
| 1.0 (100%) | 엄격한 최적 상태 | 이론적 한계 분석          |
| 0.95 (95%) | 거의 최적 상태  | 사용자가 허용한 5% 목적 완화  |
| 0.90 (90%) | 준최적 상태    | 사용자가 허용한 10% 목적 완화 |

**유일한 최적해에서도 f<1이면 범위가 생긴다는 것을 확인하기.** 방금 예제는 목적함수가 대안 최적해를 갖는 특수한 경우($$0.5v_1+0.5v_2$$)였습니다. 그렇다면 3.3절처럼 최적해가 유일한 원래 목적함수($$Z=0.5v_1+0.8v_2$$, $$Z^*=7.4$$)에서는 $$f=1.0$$일 때 당연히 $$v_1,v_2$$ 모두 범위 없이 $$(2,8)$$로 고정됩니다. 그런데 목표를 살짝 낮춰 $$f=0.9$$(즉 $$Z\ge0.9\times7.4=6.66$$)로 완화하면 어떻게 될까요? "완전히 최적일 필요는 없고, 최적값의 90%만 유지해도 된다"는 여유가 생기자 다시 범위가 나타납니다.

$$v_2$$를 최소화하려면 $$v_1$$을 최대한 활용해 $$Z\ge6.66$$을 채워야 하는데, $$v_1+v_2\le10$$과 $$0.5v_1+0.8v_2=6.66$$이 동시에 빠듯한 지점을 풀면 $$v_1=\tfrac{67}{15}\approx4.467$$, $$v_2=\tfrac{83}{15}\approx5.533$$을 얻습니다. 반대로 $$v_2$$를 최대화하는 쪽은 여전히 자신의 상한 $$8$$에서 막히므로(그 상태에서 $$v_1\ge0.52$$만 확보하면 $$Z\ge6.66$$이 이미 만족됩니다), $$v_2^{\max}=8$$입니다. 같은 논리를 $$v_1$$에 적용하면:

$$v_1^{\min}=0.52,\ v_1^{\max}\approx4.467 \qquad v_2^{\min}\approx5.533,\ v_2^{\max}=8$$

$$f=1.0$$에서는 한 점 $$(2,8)$$이었던 것이, $$f=0.9$$로 단 10%만 완화해도 이렇게 넓은 구간으로 벌어집니다 — 실제 genome-scale 모델에서 `fraction_of_optimum` 값을 1.0 대신 0.9\~0.95로 자주 쓰는 이유가 바로 여기에 있습니다. 완벽한 최적성을 살짝 양보하는 대신, 생물학적으로 그럴듯한 "거의 최적인" 대안 경로들까지 폭넓게 진단할 수 있게 됩니다.

![COBRApy textbook 모델에서 최대 성장률 90퍼센트 이상을 유지할 때 선택 반응들의 FVA 최소 최대 구간과 하나의 pFBA 해](../.gitbook/assets/fva-intervals.png)

_그림 4.6: 파란 선은 반응별로 서로 다른 LP를 풀어 얻은 FVA 주변 구간이고, 주황 마름모는 그 안의 한 pFBA 대표 해이다(COBRApy 0.30.0, GLPK, `fraction_of_optimum=0.9`). 모든 파란 끝점을 동시에 갖는 단일 flux 벡터가 존재한다는 뜻이 아니며, 구간 폭은 통계적 신뢰구간도 아니다._

```python
from cobra.flux_analysis import flux_variability_analysis

fva_result = flux_variability_analysis(model, fraction_of_optimum=0.9, processes=1)
print(fva_result.loc[["EX_glc__D_e", "PGI"]])
#              minimum    maximum
# EX_glc__D_e  -10.0000    -9.0466
# PGI           -14.2990     9.8388
# ...
```

## 9.2 반응 분류: 차단됨 / 목표 유지에 필요 / 유연함

| 분류                           | 조건                                    | 생물학적 의미                               | 예시(`e_coli_core`)          |
| ---------------------------- | ------------------------------------- | ------------------------------------- | -------------------------- |
| **차단됨(Blocked)**             | $$v_j^{\min}=v_j^{\max}=0$$           | 현재 모델·조건·목표 제약에서 흐를 수 없음              | `textbook`, $$f=0.9$$에서 8개 |
| **목표 유지에 필요(Required at f)** | 구간이 0을 포함하지 않음                        | 지정한 목표 수준 $$fZ^*$$를 유지하려면 0으로 만들 수 없음 | $$f$$와 배지에 의존              |
| **고정 활성(Fixed active)**      | $$v_j^{\min}\approx v_j^{\max}\neq0$$ | 목표 제약 아래 값이 사실상 고정                    | $$f=0.9$$에서는 0개            |
| **유연함(Flexible)**            | $$v_j^{\max}-v_j^{\min}>\epsilon$$    | 허용되는 최적·준최적 해 사이에서 값이 달라짐             | PGI 등 87개                  |

```python
fva_result['range'] = fva_result['maximum'] - fva_result['minimum']
blocked = fva_result[(fva_result['minimum'].abs() < 1e-9) & (fva_result['maximum'].abs() < 1e-9)]
fixed_active = fva_result[(fva_result['range'].abs() < 1e-6) & (fva_result['maximum'].abs() > 1e-9)]
required = fva_result[~((fva_result['minimum'] <= 1e-9) & (fva_result['maximum'] >= -1e-9))]
flexible = fva_result[fva_result['range'] > 1e-6]

print(f"차단={len(blocked)}, 목표유지에 필요={len(required)}, "
      f"고정활성={len(fixed_active)}, 유연={len(flexible)}")
# textbook, f=0.9: 차단=8, 목표유지에 필요=31, 고정활성=0, 유연=87
```

**고정 활성** 또는 **0을 포함하지 않는 FVA 구간**을 곧바로 "필수 반응"이라고 부르면 안 됩니다. 전자는 특정 목표 하에서 플럭스가 고정되었다는 뜻이고, 후자는 $$fZ^*$$를 유지하는 데 필요하다는 뜻입니다. 엄밀한 **반응 필수성**은 그 반응을 0으로 막고 다시 최대 생장을 계산해 무성장 또는 정한 임계값 이하인지 판정합니다. 차단 반응 역시 현재 배지·목표 때문에 0인 **조건부 비활성**과 네트워크 구조상 어떤 조건에서도 흐를 수 없는 **전역 차단**을 구분해야 합니다.

표준 FVA는 화학량론과 플럭스 범위만 고려하므로, 환경과 물질을 교환하지 않고 ATP나 환원력을 무한 생성하는 열역학적으로 불가능한 순환(**제3형 경로, Type-III Pathway**, [Chapter 2](../chapter-2/) 참고)을 허용할 수 있습니다. `loopless=True` 옵션(**Loopless FVA, ll-FBA**)은 이런 내부 순환을 추가 제약으로 제거합니다.

```python
ll_fva = flux_variability_analysis(
    model, fraction_of_optimum=0.9, loopless=True, processes=1
)
```

{% hint style="success" %}
**꼭 알아야 할 논문 — FVA.** Mahadevan & Schilling (2003), _The effects of alternate optimal solutions in constraint-based genome-scale metabolic models_ (doi: `10.1016/j.ymben.2003.09.002`)는 동일한 최적 성장률을 만드는 내부 플럭스 분포가 크게 다를 수 있음을 체계적으로 보였습니다. 핵심 결론은 솔버가 반환한 한 FBA 해를 유일한 생물학적 답으로 해석하지 말고, 관심 반응의 허용 범위를 FVA로 함께 보고해야 한다는 것입니다.
{% endhint %}

## 9.3 계산 비용을 줄이는 방법 — 웜스타트와 병렬화

9.1절에서 언급했듯 FVA는 반응마다 최소화·최대화를 각각 풀어야 하므로 총 $$2n$$개의 LP를 풉니다. `e_coli_core`처럼 $$n=95$$인 모델도 190번의 LP를 풀어야 하며, iML1515처럼 $$n=2{,}712$$인 모델에서는 5,424번의 LP가 필요합니다. 다행히 이 LP들은 서로 완전히 독립적이지 않습니다 — 매번 등식 제약 $$\mathbf{S}\mathbf{v}=\mathbf{0}$$과 목적값 하한 $$Z\ge fZ^*$$은 동일하고, 목적 함수 계수만 반응마다 바뀝니다.

* **웜스타트(Warm Start)** — 4.1절 표에서 확인했듯 심플렉스법은 이전 최적 기저에서 가까운 문제를 매우 빠르게 재해결할 수 있습니다. 한 반응의 FVA를 풀고 난 기저를 다음 반응의 시작점으로 재사용하면, 매번 처음부터 푸는 것보다 훨씬 빠릅니다.
* **병렬화(Parallelization)** — $$2n$$개의 LP는 서로 독립적이므로 여러 CPU 코어에 나눠 동시에 풀 수 있습니다. COBRApy의 `flux_variability_analysis(model, processes=4)`처럼 `processes` 인자로 병렬 실행을 지정합니다.

```python
import time

start = time.time()
fva_serial = flux_variability_analysis(model, fraction_of_optimum=0.9, processes=1)
print(f"단일 프로세스: {time.time()-start:.2f}초")

start = time.time()
fva_parallel = flux_variability_analysis(model, fraction_of_optimum=0.9, processes=4)
print(f"4개 프로세스: {time.time()-start:.2f}초")
# e_coli_core처럼 작은 모델은 프로세스 생성 오버헤드 때문에 오히려 병렬이 느릴 수 있음 —
# 병렬화는 iML1515처럼 반응 수가 많은 genome-scale 모델에서 이득이 커짐
```

{% hint style="warning" %}
❓ **흔한 오해:** "`processes` 값을 무조건 크게 설정할수록 빠르다"는 항상 참은 아닙니다. `e_coli_core`처럼 반응 하나하나의 LP가 이미 매우 빠른(수 밀리초) 작은 모델에서는, 프로세스를 새로 만들고 결과를 모으는 오버헤드가 오히려 병렬화의 이득보다 클 수 있습니다. 병렬화는 개별 LP 자체가 무거운 대규모 genome-scale 모델에서 진가를 발휘합니다.
{% endhint %}
