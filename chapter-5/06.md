# 6. 품질 관리, 표준 및 재현성

품질 관리는 모델의 형식, 보존 법칙, 수치 실행 가능성, 주석 및 외부 예측력을 서로 분리하여 평가하는 과정이다. 자동 시험은 반복 가능한 결함 탐지에 적합하지만, 모델이 특정 연구 질문에 타당한지를 인증하지는 않는다. 따라서 시험 결과는 모델 release, 조건별 phenotype benchmark 및 변경 이력과 함께 해석한다.

## 6.1 MEMOTE 리포트의 해석

[MEMOTE](../glossary.md)는 대사 모델의 구조·일관성·주석을 표준화된 시험으로 평가하고, 버전 관리와 지속적 통합에 연결할 수 있는 도구이다. 대표 평가 항목은 다음과 같다.

| 평가 영역 | 대표 검사 | 해석 |
|:---|:---|:---|
| Basic | 모델 요소, 경계 반응, 중복 및 기본 속성 | 파일 구조와 모델 구성의 결함 |
| Biomass | biomass precursor, 질량 및 직접 대사산물 | 목적 반응 정의와 생산 가능성 |
| Consistency | [질량·전하 균형](../glossary.md), [화학량론적 일관성](../glossary.md), 에너지 cycle | 보존 법칙과 네트워크 구조 |
| Annotation | reaction·metabolite·gene·SBO 주석 | 외부 지식과의 연결 및 재사용성 |

*Table 5.16: MEMOTE 결과를 해석하기 위한 상위 범주. 실제 시험 목록과 점수 포함 여부는 MEMOTE release 및 설정에서 확인한다. 출처: [Lieven et al. (2020)](https://doi.org/10.1038/s41587-020-0446-y).*

MEMOTE 총점은 점수화된 시험과 weight로부터 계산된다. 고정된 네 범주의 보편적 가중치나 출판 합격선은 없다. 같은 모델이라도 MEMOTE 버전, configuration, skipped test 및 weight가 다르면 총점이 달라질 수 있다. 모델 간 비교에는 최소한 다음을 고정한다.

- 모델 파일 checksum과 objective
- MEMOTE 및 dependency 버전
- solver와 tolerance
- configuration, test selection 및 weight
- 오류·skip 항목과 실행 환경

총점보다 먼저 질량 또는 에너지 생성 결함, 이전 release 대비 회귀, 연구 질문과 직접 관련된 시험 실패를 검토한다. 주석 점수가 높아도 표현형 예측은 틀릴 수 있으며, 표현형이 맞아도 질량 보존을 위반한 모델은 적절한 설명 모델이 아니다.

## 6.2 서로 다른 일관성 개념

### 반응별 질량·전하 균형

분자식과 전하가 알려진 반응 $$j$$에 대해 각 원소와 전하의 합이 보존되어야 한다. 원소 조성 행렬을 $$\mathbf E$$라 하면 이상적인 내부 생화학 반응은

$$
\mathbf E\mathbf S_{\cdot j}=\mathbf 0
$$

을 만족한다. 질량 또는 전하가 맞지 않는 반응은 외부 공급 없이 양성자나 ATP를 만들어 낼 수 있으므로 반드시 균형을 맞춘다. 균형에서 가장 흔히 누락되는 화학종은 양성자(H⁺)와 물(H₂O)이며, KEGG를 비롯한 여러 데이터베이스와 교과서가 이들을 생략한다. 전하와 protonation 상태는 모델이 가정한 pH(흔히 세포질 ~7.2)에서 각 작용기의 pK로 결정하므로, 이 검사는 정확한 formula와 protonation convention에 의존한다. Biomass, exchange, demand, sink 반응과 polymer·조성 불명확 대사산물이 포함된 반응은 의도적으로 불균형이거나 formula가 불완전하므로 별도로 분류한다.

### 화학량론적 일관성

내부 반응 행렬 $$\mathbf S_I$$가 양의 대사산물 질량 벡터 $$\mathbf m>0$$와 양립하는지 검사한다.

$$
\mathbf S_I^T\mathbf m=\mathbf 0,\qquad \mathbf m>0
$$

양의 해가 없으면 내부 네트워크에 질량 생성 또는 소멸을 암시하는 구조가 존재한다. 이 검사는 알려진 분자식을 직접 사용하지 않으므로 반응별 formula balance와 동일하지 않다.

예를 들어 $$A\rightarrow B$$와 $$B\rightarrow C$$만 있으면 $$m_A=m_B=m_C>0$$인 해가 가능하다. 여기에 $$A\rightarrow B+C$$를 추가하면 앞의 두 식과 함께 $$m_A=m_B+m_C$$를 만족해야 하므로 양의 해가 존재하지 않는다.

### Flux consistency와 blocked reaction

반응 $$j$$가 주어진 bound 아래 어떤 정상상태 flux vector에서도 비영 flux를 가질 수 없다면 [blocked reaction](../glossary.md)이다. Flux consistency는 각 반응이 적어도 하나의 feasible state에서 사용 가능한지를 묻는다. 질량을 보존하는 네트워크에도 dead end나 지나치게 엄격한 bound 때문에 blocked reaction이 존재할 수 있다.

### Energy-generating cycle과 stoichiometrically balanced cycle

모든 교환 반응을 닫은(모든 exchange flux $$=0$$) 닫힌계에서도 어떤 내부 반응 집합이 여전히 flux를 나를 수 있으면, 이는 **화학량론적으로 균형 잡힌 순환**(stoichiometrically balanced cycle, SBC; Type III extreme pathway)이다. SBC는 열역학·조절 제약이 부족해 생긴 재구축의 인공산물로, 닫힌계에서 ATP나 redox 운반체를 외부 공급 없이 재생하거나 대사물을 무에서 만들어 낼 수 있다. SBC는 futile cycle과 다르다. futile cycle은 화학량론적으로 **불균형**이며 에너지를 **소비**하는 반면, SBC는 균형 잡힌 순환으로 닫힌계에서 에너지를 **생성**한다.

외부 에너지 공급을 차단한 조건에서 ATP maintenance 또는 에너지 운반체 재생 flux가 양수가 될 수 있다면 참여 반응의 방향성, 수송 또는 proton coupling을 점검한다. 제거는 주로 참여 반응의 방향성과 열역학적 제약을 다시 부여해 수행하며, 각 변경이 성장 예측에 영향을 주지 않는지 확인하고 가설로 문서화한다. 시뮬레이션 단계에서는 loopless FBA나 최소 노름(min-norm) 해로 개별 최적해에서 SBC 순환 flux를 억제할 수 있으나([Chapter 4](../chapter-4/README.md)), 이는 특정 해에서 순환을 감추는 것일 뿐 네트워크에서 순환을 제거하지는 않는다. 또한 단순히 ‘모든 exchange를 0’으로 설정하는 것만으로 모든 모델에서 충분한 시험이 되는 것은 아니므로, 시험에 사용한 demand/maintenance 반응과 닫은 경계를 명시한다.

| 검사 | 필요한 정보 | 검출 대상 |
|:---|:---|:---|
| Mass/charge balance | formula와 charge | 개별 반응의 원소·전하 불균형 |
| Stoichiometric consistency | 내부 $$S$$ | 질량 벡터와 양립하지 않는 구조 |
| Flux consistency | $$S$$와 bounds | 어떠한 feasible state에서도 쓰이지 않는 반응 |
| Energy-cycle test | 경계·방향성·energy demand | 외부 공급 없는 에너지 생성 |

## 6.3 외부 벤치마크와 자료 누수

표현형 검증에서는 예측과 실험의 조건을 동일하게 맞춘다. 다음 예시는 계산 방법을 설명하기 위한 가상 자료이며 특정 iML1515 release의 성능값이 아니다.

| 조건 | 예측 성장률 ($$\mathrm{h}^{-1}$$) | 실측 성장률 ($$\mathrm{h}^{-1}$$) | 절대 상대오차 |
|:---|---:|---:|---:|
| 조건 A | 0.87 | 0.80 | 8.75% |
| 조건 B | 0.42 | 0.45 | 6.67% |

$$
\mathrm{relative\ error}
=\frac{|\widehat\mu-\mu_{obs}|}{|\mu_{obs}|}\times100\%
$$

관측값이 0에 가깝거나 검출 한계 이하인 경우 상대오차는 불안정하므로 absolute error, 허용 구간 또는 분류 지표를 사용한다. 성장률 비교에는 기질 섭취율을 실측값으로 제한했는지, maintenance와 biomass 반응이 동일한지, FBA가 최대 성장 가정을 사용하는지를 명시한다.

[Gap-filling](../glossary.md)에 사용한 성장 조건이나 [유전자 필수성](../glossary.md) 자료를 그대로 최종 시험에 사용하면 성능이 과대평가된다. 가능하면 condition, strain 또는 study 단위로 독립 test set을 두고, 큐레이션 중 열람한 자료까지 provenance에 기록한다.

### 평가 지표는 질문별로 분리한다

대사 모델에는 하나의 보편적인 “정확도”가 없다. 구조 품질, 기능 가능성, 정량 플럭스·성장 예측, 유전자 결손 예측은 서로 다른 자료와 지표를 요구한다.

| 평가 질문 | 예측 단위 | 대표 비교 자료 | 권장 지표와 주의점 |
|:---|:---|:---|:---|
| 모델 파일·구조가 유효한가? | 반응·대사물·주석 | SBML, formula/charge, MEMOTE | balance·consistency·annotation·round-trip. 높은 MEMOTE 점수는 표현형 정확도의 증명이 아니다. |
| 지정 기능을 수행할 수 있는가? | 대사 과제 | task definition, 생리 문헌 | 과제별 feasible/infeasible. 입력·출력·배지·최소 flux를 함께 보고한다. |
| 조건별 성장·섭취·분비를 재현하는가? | 연속값 또는 방향 | 배양 성장률, exchange flux | 절대/상대 오차, 상관, 허용 구간. 단위·정규화·측정 오차를 맞춘다. |
| 유전자 결손 효과를 재현하는가? | 이진 필수/비필수 | 동일 균주·배지의 knockout screen | confusion matrix, sensitivity, specificity, precision, MCC. [GPR](../chapter-3/README.md) 규칙과 필수성 임계값을 고정한다. |
| 질병·조직 선택성을 재현하는가? | 순위 또는 방향 | dependency, biomarker, fluxomics | ROC/PR-AUC, rank correlation, 방향 일치. 정상 대조·동일 전처리·독립 cohort가 필요하다. |

유전자 필수성에서 양성 class를 “실험상 필수”로 두면 $$TP$$는 모델과 실험이 모두 필수로 판정한 유전자다. 대표 지표는

$$
\mathrm{sensitivity}=\frac{TP}{TP+FN},\qquad
\mathrm{specificity}=\frac{TN}{TN+FP},\qquad
\mathrm{precision}=\frac{TP}{TP+FP}
$$

이며, 심하게 불균형한 결손 자료에서는 정확도(accuracy)만으로 비교하지 않고 매튜스 상관계수(Matthews correlation coefficient, MCC)나 precision–recall 지표를 함께 제시한다. 유전자 필수성은 GPR을 평가한 `single_gene_deletion()` 결과이고, 반응을 직접 차단한 [반응 필수성](../glossary.md)과 같은 레이블이 아니다. 성장 임계값, 배지, 목적함수, solver status와 non-finite 해의 처리 규칙을 사전에 정하고 보고한다.

## 6.4 SBML Level 3 package와 주석

SBML Level 3는 Core와 선택적 package로 구성된다. 제약 기반 모델에는 **[FBC(Flux Balance Constraints) Version 2](../glossary.md)**가 주로 사용된다. FBC는 flux bound, objective, GPR association, 화학식 및 전하 등의 표현을 제공한다. 반응을 pathway 또는 subsystem으로 묶는 기능은 별도의 **Groups** package가 담당한다.

| 정보 | 권장 표현 |
|:---|:---|
| Flux bounds·objective·GPR | SBML Level 3 FBC v2 |
| Pathway·subsystem membership | SBML Level 3 Groups |
| 생물학적 의미 | SBO term |
| 외부 DB 교차 참조 | Identifiers.org URI와 qualifier |
| 모델 설명·큐레이터 | SBML metadata 및 배포 문서 |

사양: [SBML FBC Version 2](https://sbml.org/documents/specifications/level-3/version-1/fbc), [SBML Groups Version 1](https://sbml.org/documents/specifications/level-3/version-1/groups/). FBC와 Groups는 독립 package이므로 소프트웨어가 각각을 읽고 보존하는지 round-trip test로 확인한다. SBML은 모든 식별자·참조·큐레이션 note를 담지 못하므로, 배포 시 전체 스프레드시트(및 BiGG 같은 웹 자원)를 함께 제공해 손실을 보완한다.

MIRIAM은 모델의 최소 보고·주석 원칙을 제시하며, 특정 접두사 문자열 하나를 ‘MIRIAM 식별자’로 선언하는 것만으로 준수되는 것은 아니다. 데이터베이스와 생물학적 qualifier가 명확한 지속 식별자를 사용한다. MetaNetX/MNXref는 여러 biochemical DB 사이의 metabolite·reaction cross-reference를 제공하지만, 자동 매핑은 protonation, compartment 및 reaction direction을 재검토해야 한다.

## 6.5 합의 모델과 pan-model

여러 도구의 반응 집합을 결합할 때 교집합은 보수적이고 합집합은 포괄적이다.

$$
R_{core}=\bigcap_{k=1}^{K}R_k,\qquad
R_{union}=\bigcup_{k=1}^{K}R_k
$$

교집합은 false positive를 줄일 수 있지만 실제 tool-specific 반응도 제거하여 false negative를 늘릴 수 있다. 합집합은 누락 회수 가능성을 높이지만 근거가 약한 반응도 포함한다. 도구들이 같은 annotation 또는 reaction DB를 공유하면 ‘지지 도구 수’는 독립 증거 수가 아니다. 따라서 합의도는 확률이 아니라 큐레이션 우선순위로 사용한다.

균주별 모델에서 core와 union을 계산하려면 객체 identity가 아니라 정규화된 반응 ID 또는 화학량론 key를 비교한다.

```python
def reaction_id_sets(strain_models):
    """ID 체계가 사전에 정규화되었다고 가정한다."""
    sets = [{r.id for r in model.reactions} for model in strain_models]
    if not sets:
        raise ValueError("strain_models must not be empty")

    core = set.intersection(*sets)
    union = set.union(*sets)
    prevalence = {
        rid: sum(rid in ids for ids in sets) / len(sets)
        for rid in union
    }
    return core, union, prevalence
```

서로 다른 도구의 모델은 같은 ID가 다른 반응을 뜻하거나 다른 ID가 같은 반응을 뜻할 수 있다. 계산 전에 compartment, protonation, 방향성 및 화학량론을 정규화하고 매핑 불확실성을 기록한다. ‘core/shell/cloud’ 경계도 분석자가 정한 prevalence threshold에 의존하므로 보편 범주로 간주하지 않는다.

## 6.6 Release 체크리스트

출판 또는 공유 모델에는 다음을 포함한다.

- SBML 파일과 checksum, semantic version 또는 release tag
- genome assembly·annotation·reaction DB의 accession과 release
- 배지, exchange bounds, objective, maintenance 및 flux 단위
- gap-filled·수동 추가·삭제 반응의 근거와 confidence
- MEMOTE report, 실행 환경, 실패·skip 시험
- 독립 phenotype benchmark와 원자료 처리 코드
- license, citation, curator 및 변경 이력

이 정보가 있어야 다른 연구자가 모델 파일을 단순 실행하는 수준을 넘어, 재구축의 근거와 예측 범위를 재현할 수 있다.
