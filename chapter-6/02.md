# 2. 발현 데이터를 반응 활성 점수(RAS)로 변환하기

## 2.1 GPR의 부울 논리와 연속 점수 휴리스틱

**[유전자-단백질-반응 연관](../chapter-3/README.md)(Gene-Protein-Reaction, GPR)**은 어떤 반응에 필요한 유전자 산물을 부울 AND/OR로 나타낸다. 이 논리는 유전자 하나가 결손됐을 때 반응이 멈추는지를 표현한다. 복합체라면 필수 소단위가 하나라도 빠지면 기능을 잃고, 아이소자임이라면 여러 유전자 중 하나만 남아도 반응이 진행된다. 그러나 GPR 자체는 효소량이나 반응속도를 정량화하지 않는다. 연속 발현값에 최솟값·최댓값을 적용하는 [반응 활성 점수](../glossary.md)(Reaction Activity Score, RAS)는 여러 통합법에서 쓰는 **휴리스틱**일 뿐, 유일한 생물물리학적 변환식이 아니다.

- **복합체(protein complex) 관계 — AND**: 기능성 복합체에 모든 필수 소단위가 필요하다. 연속 RAS에서 최솟값을 사용하면 가장 낮은 발현을 병목 증거로 취급한다.

$$w_{\text{complex}} = \min(w_{g_1}, w_{g_2}, \ldots, w_{g_n})$$

- **아이소자임(isozyme) 관계 — OR**: 여러 유전자 산물 가운데 하나가 반응을 촉매할 수 있다. 최댓값 휴리스틱은 가장 강한 단일 증거를 사용하지만, 여러 아이소자임의 용량이 더해질 수 있는 경우에는 합 또는 효소별 용량 제약이 더 적절할 수 있다.

$$w_{\text{isozyme}} = \max(w_{g_1}, w_{g_2}, \ldots, w_{g_n})$$

- **복합 논리**: `(Gene_A and Gene_B) or Gene_C`에 min/max 휴리스틱을 적용하면 $$\max(\min(w_A,w_B),w_C)$$이다. 이 값은 발현 증거 점수이지 flux 또는 효소활성의 측정값이 아니다.

복합 GPR은 중첩 구조와 특수문자를 포함한 유전자 ID를 가질 수 있으므로 문자열을 직접 `eval`하지 않아야 한다. 실습에서는 [COBRApy](https://opencobra.github.io/cobrapy/)가 이미 파싱한 `Reaction.gpr` 표현을 순회한다.

### 2.1.1 결측 유전자와 GPR이 없는 반응의 처리

다음 두 경우는 발현 부재와 정보 부재를 구분해서 처리해야 한다.

- **결측 유전자(missing gene)**: GPR에 등장하는 유전자가 발현 데이터셋에 아예 없는 경우(예: 시퀀싱 깊이 부족, ID 매핑 실패)이다. 이 유전자를 0으로 취급하면 AND 관계에서는 복합체 전체가 자동으로 "증거 없음"이 되어 버리고, OR 관계에서는 그 유전자만 무시되고 나머지로 판단된다. 결측과 "발현 안 됨"을 구분하지 않고 무조건 0을 대입하면, 특히 AND로 연결된 복합체에서 실제로는 발현되고 있는 반응을 잘못 제거할 위험이 있다. 이 때문에 실무 파이프라인은 결측 유전자를 0이 아닌 "정보 없음"으로 표시해 두고, 후속 단계(예: 대사 작업 검증)에서 별도로 재확인하는 방식을 권장한다.
- **GPR이 없는 반응**: 교환·demand·확산·자발 반응처럼 본래 gene-independent인 경우와 annotation이 누락된 효소 반응을 구분해야 한다. 전자는 배지 또는 기능 작업에 따라 설정하고, 후자는 무조건 열어두지 말고 큐레이션 불확실성으로 기록해야 한다. `textbook` 모델에서는 95개 반응 중 69개에 GPR 문자열이 있지만, 나머지 26개가 모두 같은 생물학적 범주인 것은 아니다.

## 2.2 손으로 직접 풀어보기: 발현 벡터 → 이진화 → 반응 on/off

다음은 해당과정 초반 네 반응을 단순화한 가상 예시이다. 지정한 GPR 집계 규칙과 임계값으로 반응 라벨을 계산한다.

```
포도당(외부) --[R1 GLCt]--> 포도당(세포질) --[R2 HEX1]--> G6P --[R3 PGI]--> F6P --[R4 PFK]--> F1,6BP
```

**1단계 — 유전자 발현값 확인.** 각 유전자의 발현 점수(0~10 스케일의 임의 단위, 예를 들어 정규화된 log2(TPM+1)로 볼 수 있다)가 다음과 같이 주어졌다고 가정한다.

| 유전자 | 발현 점수 | 임계값 $$\theta=3.0$$ 적용 후 이진 라벨 |
|---|---:|:---:|
| GLUT1 | 7.0 | 1 (활성) |
| HK1 | 2.0 | 0 (비활성) |
| HK2 | 8.0 | 1 (활성) |
| PGI | 5.0 | 1 (활성) |
| PFKL | 1.0 | 0 (비활성) |
| PFKM | 6.0 | 1 (활성) |

**2단계 — 고정 임계값(fixed threshold)으로 이진화.** $$\theta=3.0$$보다 크면 1(활성), 아니면 0(비활성)으로 표시한다(위 표의 마지막 열).

**3단계 — GPR로 반응 수준까지 끌어올리기.** 이진 라벨(0 또는 1)에서는 **최솟값(min)이 논리곱(AND)과, 최댓값(max)이 논리합(OR)과 정확히 일치**한다. 이 사실을 이용해 네 반응의 상태를 결정한다.

| 반응 | GPR 규칙 | 연산 | 연속 RAS (0~10 스케일 그대로 min/max) | 이진 결과 (0/1로 AND/OR) | $$\theta=3.0$$ 기준 온/오프 |
|---|---|---|---:|:---:|:---:|
| R1 GLCt (포도당 수송) | `GLUT1` | 단일 유전자 | 7.0 | 1 | **ON** |
| R2 HEX1 (헥소키나아제) | `HK1 or HK2` | OR = max | max(2, 8) = 8.0 | max(0, 1) = 1 | **ON** |
| R3 PGI (포스포글루코스 이성질화효소) | `PGI` | 단일 유전자 | 5.0 | 1 | **ON** |
| R4 PFK (포스포프럭토키나아제) | `PFKL and PFKM` | AND = min | min(1, 6) = 1.0 | min(0, 1) = 0 | **OFF** |

**결과 해석.** 지정한 min/max와 임계값 아래에서 R2는 ON, R4는 낮은 발현 증거를 갖는 OFF 후보로 분류된다. 그러나 이 분류만으로 R4를 즉시 제거해서는 안 된다. 하류 대사 작업에 R4가 필요하면 GIMME나 tINIT 같은 추출법은 낮은 발현에 벌점을 주면서도 반응을 유지할 수 있다. 즉 RAS 라벨과 최종 모델 포함 여부는 서로 다른 계산 단계이다.

{% hint style="warning" %}
**해석상의 주의:** 고정 임계값과 min/max처럼 순서를 보존하는 연산에서는 유전자 이진화와 RAS 이진화 순서가 같은 결과를 낼 수 있다. 백분위수 또는 분포 적합 임계값은 유전자 집합과 반응 집합의 분포가 다르므로 어느 단계에서 적합했는지를 기록해야 한다.
{% endhint %}

아래 코드로 위 손 계산을 그대로 재현해 검증할 수 있다.

```python
# 장난감 GPR 트리를 재귀적으로 평가 — and는 min, or는 max
def evaluate_gpr(gpr_tree, expr_dict):
    if isinstance(gpr_tree, str):          # leaf: 유전자 이름
        return expr_dict[gpr_tree]          # 결측이면 0으로 대체하지 않고 오류로 표시
    op, *children = gpr_tree
    values = [evaluate_gpr(child, expr_dict) for child in children]
    return min(values) if op == 'and' else max(values)

expr = {'GLUT1': 7.0, 'HK1': 2.0, 'HK2': 8.0, 'PGI': 5.0, 'PFKL': 1.0, 'PFKM': 6.0}
reactions = {
    'R1_GLCt': 'GLUT1',
    'R2_HEX1': ('or', 'HK1', 'HK2'),
    'R3_PGI':  'PGI',
    'R4_PFK':  ('and', 'PFKL', 'PFKM'),
}
theta = 3.0
for rid, gpr in reactions.items():
    ras = evaluate_gpr(gpr, expr)
    print(f"{rid:10s} RAS={ras:4.1f}  ->  {'ON' if ras > theta else 'OFF'}")

# 기대 출력:
# R1_GLCt    RAS= 7.0  ->  ON
# R2_HEX1    RAS= 8.0  ->  ON
# R3_PGI     RAS= 5.0  ->  ON
# R4_PFK     RAS= 1.0  ->  OFF
```

실제 GEM에서는 유전자 137개(`e_coli_core`)에서 수천 개(Human1/Human2)까지, 반응 95개에서 1만 개 이상까지 규모만 커질 뿐, **GPR을 반응 증거로 옮기는 기본 원리**는 같다. 다만 실제 알고리즘은 결측 유전자, 가역 반응, 복합체 정량화와 임계값 불확실성을 추가로 처리해야 한다.

### 2.2.1 조금 더 깊은 중첩: 3단 GPR 손 계산

앞의 예제는 AND와 OR이 각각 한 번씩만 등장하는 비교적 단순한 경우였다. 실제 GEM에는 `(GeneA and GeneB) or (GeneC and GeneD) or GeneE`처럼 여러 항이 병렬로 OR에 묶여 있는 경우가 흔하다. 아래 값으로 손 계산을 한 단계 더 연습한다.

| 유전자 | 발현 점수 |
|---|---:|
| GeneA | 9.0 |
| GeneB | 2.0 |
| GeneC | 6.0 |
| GeneD | 7.0 |
| GeneE | 1.5 |

계산 순서는 항상 **가장 안쪽 괄호부터**이다.

1. 첫 번째 복합체: $$\min(w_A, w_B) = \min(9.0, 2.0) = 2.0$$
2. 두 번째 복합체: $$\min(w_C, w_D) = \min(6.0, 7.0) = 6.0$$
3. 단독 아이소자임: $$w_E = 1.5$$
4. 세 경로를 OR로 묶어 최댓값: $$\max(2.0,\ 6.0,\ 1.5) = 6.0$$

따라서 이 반응의 RAS는 6.0이고 임계값 $$\theta=3.0$$에서 ON이다. GeneA의 발현값이 9.0이어도 GeneB와 이루는 AND 가지의 점수는 2.0이다. GPR 구조를 무시하고 가장 큰 단일 유전자 값만 사용하면 복합체 요구를 반영하지 못한다.

## 2.3 임계값(threshold) 설정 전략 비교

GPR로 계산된 RAS는 연속값이다. 많은 알고리즘([GIMME](../landmark-papers.md)의 저발현 집합, [iMAT](../landmark-papers.md)의 high/moderate/low 분류, [FASTCORE](../landmark-papers.md)의 core set)은 이 연속값을 "발현/비발현" 또는 "high/moderate/low"의 이산 범주로 나눈다. [임계값](../glossary.md)은 알고리즘·배지·기능 제약과 함께 결과 모델을 크게 바꾸는 주요 설정 가운데 하나이다(Robaina Estevez & Nikoloski, 2014).

| 방법 | 절차 | 사용 알고리즘 | 특징 |
|---|---|---|---|
| **고정 임계값(fixed threshold)** | $$x > \theta$$ → 활성 | GIMME penalty·confidence bin의 전처리 | 단순하지만 scale·platform에 의존 |
| **백분위수(percentile)** | 상위 N%를 활성으로 간주 | FASTCORE/SwiftCore의 core 집합 구성에 사용할 수 있음 | 표본마다 선택 비율이 고정됨 |
| **분위수(quantile) 분류** | 분위수로 high/moderate/low 구성 | iMAT 입력 분류에 사용할 수 있음 | 세 집합의 경계에 민감 |
| **일반 z-score** | 전체 값의 평균·표준편차로 표준화 | 교육용·탐색적 비교 | 분포가 비정규·이봉이면 해석에 주의 |
| **zFPKM / 분포 적합** | 발현 주봉의 오른쪽에 half-Gaussian을 적합해 표준화 | rFASTCORMICS | 일반 z-score와 다른 sample-specific 변환 |
| **K-means 클러스터링** | 비지도 학습으로 2~3개 발현 그룹 구성 | iMAT 등의 선택적 전처리 | 초기화·군집 분리도에 민감 |

고정 임계값은 구현이 단순하지만 scale과 플랫폼에 의존한다. 백분위수는 항상 정해진 비율을 선택하므로 대부분 유전자가 발현되는 표본과 그렇지 않은 표본을 동일하게 취급한다. 분포 적합법은 표본별 주봉과 저발현 어깨를 이용하지만, 적합할 주봉이 분명하지 않거나 데이터가 희소하면 실패할 수 있다. 어떤 방법도 임계값 불확실성을 제거하지 않으므로 여러 설정에 대한 민감도 분석이 필요하다.

**전역(global) 임계값 vs. 지역(local) 임계값**: 모든 유전자에 같은 기준을 적용하는 전역 방식은 구현이 단순하지만 유전자별 기저 발현 차이를 반영하지 않는다. 유전자별 분포 또는 샘플 내 순위를 사용하는 지역 기준은 이를 완화할 수 있으나, 추가 표본과 규칙이 필요하고 결과가 자동으로 더 정확해지는 것은 아니다.

모델 간 Jaccard index와 유전자 필수성 정확도는 template model, 임계값 후보, 배지, 보호 작업과 평가 데이터에 따라 달라진다. 특정 연구에서 얻은 0.27~1.0 같은 범위나 "상위 10%가 더 정확하다"는 결과를 보편 규칙으로 사용할 수 없다. 임계값별 모델 크기, task 통과, flux consistency와 독립 필수성 성능을 함께 보고해야 한다.

### rFASTCORMICS의 통계적 이진화: zFPKM 방식

rFASTCORMICS는 각 표본의 $$\log_2(\mathrm{FPKM})$$ 밀도에서 주 발현 봉우리의 오른쪽 절반에 Gaussian 곡선을 적합하고 이를 대칭화한다. 적합된 **발현 분포**의 평균 $$\mu_E$$와 표준편차 $$\sigma_E$$로

$$z\mathrm{FPKM}_i=\frac{\log_2(\mathrm{FPKM}_i)-\mu_E}{\sigma_E}$$

를 계산한다. 여기서 $$\mu_E,\sigma_E$$는 전체 유전자 값의 표본평균·표본표준편차가 아니다. 발현 Gaussian을 전체 밀도에서 뺀 잔차에는 비발현 분포를 추가로 적합한다. 원 workflow에서는 $$z\mathrm{FPKM}\ge0$$을 발현 증거로 두고, 비발현 분포의 zFPKM 평균 $$\mu_I$$와 $$-3$$ 중 큰 값인 $$\theta_{inactive}=\max(\mu_I,-3)$$ 이하를 비발현 증거로 분류한다. 그 사이 값은 불확실 영역이다([Hart et al., 2013](https://doi.org/10.1186/1471-2164-14-778); [Pacheco et al., 2019](https://doi.org/10.1016/j.ebiom.2019.04.046)). 따라서 `scipy.stats.zscore`에 $$-3$$을 적용하는 코드는 rFASTCORMICS를 재현하지 않는다.

### 2.3.1 적합된 발현 분포로 zFPKM 계산하기

half-Gaussian 적합 결과가 $$\mu_E=5.8$$, $$\sigma_E=1.0$$이고 비발현 분포 평균이 $$\mu_I=-2.5$$라고 가정한다. 그러면 $$\theta_{inactive}=\max(-2.5,-3)=-2.5$$이다. $$\log_2(\mathrm{FPKM})$$가 6.2, 4.5, 2.0인 세 유전자의 zFPKM은 각각 0.4, -1.3, -3.8이다.

| $$\log_2(\mathrm{FPKM})$$ | zFPKM | 원 workflow의 해석 |
|---:|---:|---|
| 6.2 | 0.4 | 발현 증거($$z\ge0$$) |
| 4.5 | -1.3 | 불확실 영역 |
| 2.0 | -3.8 | 비발현 증거($$z\le\theta_{inactive}=-2.5$$) |

이 계산은 적합 뒤의 표준화만 보여 준다. 실제 분석에서는 kernel density와 half-Gaussian 적합의 품질, 0 FPKM 처리, 사용한 gene set과 비발현 분포 cutoff를 함께 기록해야 한다.

**실습 연결:** 실습에서는 일반 z-score가 rFASTCORMICS의 zFPKM과 다르다는 점을 코드로 구분한다.
{% endhint %}
