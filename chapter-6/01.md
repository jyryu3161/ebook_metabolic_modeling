# 1. 범용 모델에서 맥락 특이적 모델로

## 1.1 왜 하나의 모델을 조직마다 잘라내야 하나

가장 쉬운 질문부터 시작합시다 — 간세포와 뉴런은 같은 유전체를 갖고 있는데, 왜 대사 모델링에서는 굳이 둘을 다르게 취급해야 할까요? 답은 의외로 일상적인 경험에서 이미 알고 있습니다. 같은 아파트 단지에 사는 두 가정이 똑같은 슈퍼마켓(같은 유전체가 제공하는 모든 재료)을 이용할 수 있다고 해서, 두 가정이 매일 같은 메뉴를 먹지는 않습니다. 한 가정은 매운 음식을 즐기고, 다른 가정은 채식 위주로 먹습니다. 슈퍼마켓 진열대(유전체)는 똑같아도, 실제 장바구니(그 세포가 지금 쓰는 대사 경로)는 완전히 다를 수 있다는 것이 이 장 전체를 관통하는 직관입니다.

"이 장을 시작하며"에서 본 요리책 비유 — 범용 GEM은 요리책 전체, 맥락 특이적 모델은 오늘의 식탁 — 를 이제 구체적인 숫자로 확인해 봅시다. [GEM 구조](../chapter-3/README.md)에서 다룬 Human1 같은 인체 GEM은 인체 게놈 전체에 암호화된 대사 능력을 총망라합니다 — 2020년 Human1 원 발표판은 13,417개 반응과 3,625개 유전자를 포함합니다(버전별 수치와 Human2 계보는 [Chapter 5](../chapter-5/README.md) 참고). 이 13,417개 반응 중 어떤 조직도 전부를 동시에 사용하지는 않습니다 — 뉴런은 지방산 베타 산화 능력이 극히 제한적이고, 간이 아닌 조직 대부분은 요소 회로(urea cycle) 전체를 갖추고 있지 않습니다. 특정 조직 모델의 크기는 추출법·임계값·보호 작업에 따라 크게 달라지므로, 반응 수 자체를 품질 점수처럼 해석해서는 안 됩니다 — "반응이 많을수록 좋은 모델"이 아니라 "그 조직의 실제 생리를 정확히 반영하는 반응 집합"이 좋은 모델입니다.

Generic 모델을 그대로 [FBA](../chapter-4/README.md)나 유전자 필수성 예측에 사용하면 다음 세 가지 문제가 생깁니다.

1. **생물학적으로 무관한 예측**: 이 조직에서 전혀 발현되지 않는 유전자의 반응까지 네트워크에 포함되어, flux 분포나 유전자 넉아웃 결과가 비현실적으로 나옵니다. 뉴런에서 있을 리 없는 요소 회로(urea cycle) 반응이 뉴런 모델에 그대로 남아있다면, 그 반응을 없애는 유전자 결손을 "치명적"이라고 잘못 예측할 수 있습니다.
2. **과도한 유연성(flexibility)**: 실제로는 쓰지 않는 경로까지 열려 있으니 "이 반응이 필수인지 대체 가능한지"를 계산하는 [FVA](../chapter-4/README.md)의 범위가 불필요하게 넓어지고, 예측의 특이성이 떨어집니다. 비유하자면, 실제로는 문이 잠겨 있는 비상구까지 "탈출 경로"로 계산에 포함시키는 셈입니다 — 지도상으로는 출구가 여러 개 있지만, 실제로 열리는 문은 그중 하나뿐일 수 있습니다.
3. **표현형 예측 정확도 저하**: 조직·세포주별 유전자 필수성(gene essentiality)을 예측할 때, 맥락 특이적 모델이 generic 모델보다 유의미하게 높은 정확도를 보인다는 것이 여러 벤치마크 연구에서 반복적으로 확인되었습니다.

이 문제를 푸는 것이 바로 Omics 데이터 통합입니다. Omics 통합은 "이 반응이 이론적으로 가능한가"라는 질문을 "이 반응이 **지금, 이 세포에서** 실제로 일어나고 있다는 증거가 있는가"라는 질문으로 바꿔놓습니다.

> **핵심 개념 · 용어(English):** **맥락 특이적 모델(Context-Specific Model)** — Generic GEM으로부터 특정 조직·세포 유형·세포주·질병 상태의 오믹스 데이터(주로 발현 데이터)를 이용해 추출한 서브네트워크(submodel). 발현 증거가 뒷받침하는 반응은 남기고, 증거가 없거나 반대되는 증거가 있는 반응은 제거하거나 억제합니다.

{% hint style="warning" %}
❓ **잠깐, 생각해보기:** 만약 뉴런 모델에 (실제로는 뉴런에서 거의 발현되지 않는) 요소 회로 반응이 그대로 남아 있다면, FBA로 계산한 최적 성장률과 유전자 결손 예측은 어떻게 달라질까요? — 요소 회로는 암모니아를 처리하는 "보너스 경로"처럼 작동해 질소 균형을 더 쉽게 맞추게 해주므로, 실제로는 없는 대안 경로 덕분에 특정 유전자 결손의 영향이 과소평가될 수 있습니다. 즉 "없는 요리법"이 메뉴에 남아있으면, 모델은 실제 세포보다 더 유연하고 더 튼튼하다고 착각하게 됩니다.
{% endhint %}

### 1.1.1 숫자로 보는 "일부만 사용한다"는 말의 의미

"조직마다 실제로 쓰는 반응은 전체의 일부일 뿐"이라는 말을 조금 더 구체적으로 느껴봅시다. 아주 단순화된 장난감 예로, 어떤 범용 모델에 10개의 반응이 있고, 그중 어떤 조직에서 뒷받침되는 발현 증거가 있는 반응이 6개뿐이라고 가정해 봅시다.

| 반응 | 범용 모델 포함 여부 | 이 조직의 발현 증거 | 맥락 특이적 모델 포함 여부 |
|---|:---:|:---:|:---:|
| R1 | 포함 | 있음 (RAS 높음) | 포함 |
| R2 | 포함 | 있음 (RAS 높음) | 포함 |
| R3 | 포함 | 없음 (RAS 낮음) | 제외 |
| R4 | 포함 | 있음 (RAS 중간) | 포함 |
| R5 | 포함 | 없음 (RAS 낮음) | 제외 |
| R6 | 포함 | 있음 (RAS 높음) | 포함 |
| R7 | 포함 | 없음 (RAS 낮음) | 제외 |
| R8 | 포함 | 있음 (RAS 중간) | 포함 |
| R9 | 포함 | 있음 (RAS 높음) | 포함 |
| R10 | 포함 | 없음 (RAS 낮음) | 제외 |

이 장난감 예에서 맥락 특이적 모델은 범용 모델의 60%(6/10)만 남깁니다. 실제 인체 조직 모델에서도 이와 비슷하게 상당수의 반응이 걸러지는 경향이 보고되지만, 정확한 비율은 추출 알고리즘·임계값·조직 종류에 따라 크게 달라지므로 "60%"라는 특정 숫자를 보편적 사실로 암기해서는 안 됩니다. 중요한 것은 비율 자체가 아니라, **"포함 여부를 결정하는 근거가 발현 증거라는 사실"**이며, 이 근거를 어떻게 계산하는지가 2절의 주제입니다.

## 1.2 통합의 파이프라인: 증거 → 가중치 → 서브네트워크

방법론마다 세부 구현은 다르지만, 거의 모든 발현-제약 통합(expression-constrained integration) 방법론은 다음의 공통 파이프라인을 따릅니다.

```mermaid
flowchart TD
    A["1. Omics 데이터 획득<br/>RNA-seq · proteomics · metabolomics"] --> B["2. 전처리 · 정규화<br/>TPM 계산, 로그 변환, 품질 관리"]
    B --> C["3. 유전자 발현 → 반응 활성 점수(RAS)<br/>GPR 규칙 적용"]
    C --> D["4. RAS → 이산/연속 가중치<br/>임계값 설정 또는 연속 스케일링"]
    D --> E["5. 가중치를 LP/MILP 제약으로 통합<br/>GIMME · iMAT · E-Flux · tINIT"]
    E --> F["6. 맥락 특이적 서브네트워크 추출"]
    F --> G["7. 검증<br/>FBA/FVA 재실행, 대사 작업 통과율, gene essentiality"]

    style A fill:#e1f5fe
    style G fill:#c8e6c9
```

이 7단계 파이프라인을 파이프라인이라고 부르는 이유는, 앞 단계의 산출물이 그대로 다음 단계의 입력이 되기 때문입니다 — 마치 정수 처리장에서 원수(1단계)가 침전(2단계)·여과(3~4단계)·소독(5단계)을 차례로 거쳐 수돗물(6단계)이 되고, 마지막으로 수질 검사(7단계)를 통과해야 비로소 "먹는 물"로 인정받는 것과 같습니다. 어느 한 단계에서 오염(오류)이 생기면 그 오류는 뒤 단계로 그대로 전파됩니다 — 예를 들어 2단계의 유전자 ID 매핑이 틀리면, 3단계의 GPR 평가가 엉뚱한 유전자의 발현값을 읽어 들이고, 그 결과 6단계의 서브네트워크 전체가 잘못될 수 있습니다. 그래서 7단계 검증은 생략 가능한 "선택 사항"이 아니라 파이프라인의 필수 구성 요소입니다.

이 장은 **3~5단계** — "발현 데이터를 어떻게 반응 수준의 증거로 변환하고, 이를 최적화 제약으로 바꾸는가" — 에 집중합니다. 조직 특이적 모델을 **재구축(reconstruction)**하는 관점 — draft 모델 준비, gap-filling, 품질 관리 체크리스트, tINIT의 6단계 알고리즘과 MILP 유도 과정(6단계) — 은 [Chapter 5](../chapter-5/README.md)에서 이미 다뤘습니다. 추출된 맥락 특이적 모델에 실제로 FBA/FVA를 실행하는 방법(7단계 검증의 일부)은 [Chapter 4](../chapter-4/README.md)를, 암 세포주 맥락 특이적 모델을 이용한 약물 표적 예측 응용은 [Chapter 7](../chapter-7/README.md)을 참고하십시오.

{% hint style="info" %}
💡 **미리 보기:** 3단계(RAS 계산)는 2절, 4단계(임계값)는 2.3절, 5단계(GIMME/iMAT/E-Flux/tINIT)는 3절에서 각각 자세히 다룹니다. 1단계와 2단계(오믹스 데이터 자체의 획득·전처리)는 4절에서, 단백질체·대사체를 이용한 확장(효소 제약·열역학 제약)은 5절에서 다룹니다.
{% endhint %}
