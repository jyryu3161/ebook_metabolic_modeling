# 4. RNA-seq 데이터: 유형과 전처리 파이프라인

## 4.1 Counts, FPKM, TPM

RNA-seq 값은 정규화 수준에 따라 의미가 다릅니다. 서로 다른 정량 단위나 파이프라인에서 생성한 값을 직접 혼합하면 기술적 차이가 RAS와 맥락 모델의 차이로 전파될 수 있습니다.

**Raw counts**: HTSeq, featureCounts, RSEM 등으로 계산된, 각 유전자에 매핑된 리드(read) 수. 정수값이며 라이브러리 크기와 유전자 길이에 의존하므로 **라이브러리 크기와 조성 차이를 보정하지 않은 원시 계수는 샘플 간에 직접 비교해서는 안 됨**합니다. 예를 들어 유전자 A가 샘플 1에서 1,000 counts, 샘플 2에서 2,000 counts라고 해서 샘플 2에서 유전자 A가 두 배 더 발현된 것은 아닙니다 — 샘플 2의 시퀀싱을 애초에 두 배 더 깊게 했을 수도 있기 때문입니다.

**FPKM (Fragments Per Kilobase of transcript per Million mapped reads)**:

$$FPKM = \frac{X \cdot 10^9}{N \cdot L}$$

($$X$$: 유전자에 매핑된 프래그먼트 수, $$N$$: 전체 매핑된 프래그먼트 수, $$L$$: 유전자 길이(bp)) 라이브러리 크기와 유전자 길이를 보정하지만, 샘플마다 총 FPKM 합이 달라 **샘플 간 비교에는 부적절**합니다.

**TPM (Transcripts Per Million)**:

$$TPM_i = \frac{X_i / \tilde{l}_i}{\sum_j (X_j / \tilde{l}_j)} \times 10^6$$

($$\tilde{l}_i$$: 유전자 $$i$$의 유효 길이) 분모의 합산 때문에 각 샘플의 TPM 합은 $$10^6$$입니다. TPM은 한 샘플 안의 상대 조성 또는 같은 정량 파이프라인에서 만든 GEM 가중치 후보로 사용할 수 있습니다. 그러나 합이 고정된 **조성 자료(compositional data)**이므로 한 유전자의 변화가 다른 유전자의 상대값에도 영향을 주고 RNA 총량 차이를 보존하지 않습니다. 차등발현 검정에는 TPM에 t-test를 적용하기보다 raw counts와 DESeq2/edgeR 같은 count 모델을 사용합니다. 표본 간 GEM 통합에서도 배치, 조직 조성과 정량 파이프라인을 맞추고 민감도 분석을 수행해야 합니다. 자세한 통계 배경은 [오믹스 데이터 분석을 위한 기초 통계](../supplements/omics-statistics.md)를 참고하십시오.

| 특성 | Raw Counts | FPKM | TPM |
|---|---|---|---|
| 정규화 | 없음 | Library size + Gene length | Library size + Gene length |
| 샘플 간 비교 | 라이브러리 크기 보정 전에는 불가 | 총합 차이 때문에 주의 | 동일 파이프라인·조성 효과를 점검한 상대 비교에 사용 |
| 합계 | 가변적 | 가변적 | 항상 $$10^6$$ |
| 주요 용도 | DESeq2/edgeR 등 차등발현 검정의 입력 | 기존 자료 호환 | 샘플 내 상대 발현, GEM 가중치의 한 후보 |

### 4.1.1 손으로 직접 계산해보기: 같은 counts, 다른 정규화

FPKM과 TPM의 차이를 가장 확실하게 이해하는 방법은 같은 원시 데이터로 두 값을 모두 계산해보는 것입니다. 세 유전자의 raw counts와 길이, 그리고 전체 매핑된 프래그먼트 수(라이브러리 크기)가 다음과 같다고 합시다.

| 유전자 | Counts $$X_i$$ | 길이 $$L_i$$ (bp) | 전체 매핑 수 $$N$$ |
|---|---:|---:|---:|
| 유전자 X | 500 | 2,000 | 2,000,000 |
| 유전자 Y | 1,000 | 500 | 2,000,000 |
| 유전자 Z | 300 | 1,500 | 2,000,000 |

**FPKM 계산** ($$FPKM_i = X_i \cdot 10^9 / (N \cdot L_i)$$):

$$FPKM_X = \frac{500 \times 10^9}{2{,}000{,}000 \times 2{,}000} = 125.0$$

$$FPKM_Y = \frac{1{,}000 \times 10^9}{2{,}000{,}000 \times 500} = 1{,}000.0$$

$$FPKM_Z = \frac{300 \times 10^9}{2{,}000{,}000 \times 1{,}500} = 100.0$$

FPKM 값의 합은 $$125+1000+100=1225$$로, 다른 샘플에서 계산하면 이 합이 다른 값이 될 수 있습니다 — 바로 이 점이 FPKM을 샘플 간 비교에 부적절하게 만드는 이유입니다.

**TPM 계산** (먼저 길이로 나눈 비율 $$rate_i = X_i/L_i$$를 구한 뒤, 그 비율의 합으로 다시 나누어 정규화):

$$rate_X = 500/2000 = 0.25, \quad rate_Y = 1000/500 = 2.0, \quad rate_Z = 300/1500 = 0.2$$

$$\sum rate = 0.25+2.0+0.2 = 2.45$$

$$TPM_X = \frac{0.25}{2.45}\times 10^6 \approx 102{,}041, \quad TPM_Y = \frac{2.0}{2.45}\times 10^6 \approx 816{,}327, \quad TPM_Z = \frac{0.2}{2.45}\times 10^6 \approx 81{,}633$$

세 값의 합을 확인하면 $$102{,}041+816{,}327+81{,}633 = 1{,}000{,}001 \approx 10^6$$(반올림 오차 제외)로 정확히 100만이 됩니다. **FPKM은 유전자마다 독립적으로 계산되어 합이 데이터마다 달라지지만, TPM은 계산 과정 자체에 "전체 합을 100만으로 맞추는" 정규화 단계가 포함되어 있다**는 점이 두 지표의 근본적 차이입니다. 이 성질 때문에 TPM은 "이 샘플 안에서 유전자 Y가 전체 발현량의 몇 %를 차지하는가"라는 질문에 곧바로 답할 수 있지만(위 예에서 유전자 Y는 전체의 약 81.6%), 서로 다른 두 샘플의 유전자 Y를 직접 비교할 때는 여전히 "그 샘플의 다른 유전자들이 얼마나 발현되었는가"에 따라 상대적으로 값이 흔들릴 수 있다는 조성 자료(compositional data)의 함정을 안고 있습니다.

## 4.2 RNA-seq의 종류: 벌크에서 단일세포까지

앞 절은 세포 집단의 평균 발현을 측정하는 **벌크 RNA-seq**을 가정했습니다. 단일세포·단일핵·공간 전사체는 관측 단위와 희소성이 다르므로 전처리와 모델 해석도 달라집니다.

| 유형 | 측정 단위 | 대사 모델링에서의 함의 |
|---|---|---|
| **벌크(bulk) RNA-seq** | 조직·세포 집단의 평균 발현 | 이 장에서 다루는 GIMME/iMAT/tINIT 등 대부분의 통합 방법이 가정하는 표준 입력. 세포 이질성(heterogeneity)은 평균화되어 사라짐 |
| **단일세포(single-cell, scRNA-seq)** | 세포 하나하나의 발현 | 세포 유형별로 별도의 맥락 특이적 모델을 만들 수 있어 조직 내 이질성(예: 종양 미세환경의 암세포·면역세포·기질세포)을 반영 가능. 다만 유전자당 리드 수가 매우 적어(drop-out) RAS 계산 전에 세포를 유사한 그룹으로 묶는 pseudobulk 집계가 흔히 필요 |
| **단일핵(single-nucleus, snRNA-seq)** | 핵 안의 전사체만 측정 | 냉동 조직·해리가 어려운 조직(뇌, 지방)에 적합. 성숙한 mRNA 비율이 달라 벌크·scRNA-seq와 직접 비교 시 주의 필요 |
| **공간 전사체(spatial transcriptomics)** | 조직 절편 내 위치 정보가 있는 발현 | 조직 내 위치에 따른 대사 구역화(metabolic zonation, 예: 간의 문맥주위/중심정맥주위 대사 차이)를 맥락 특이적 모델에 반영하는 데 활용 시작 단계 |

{% hint style="warning" %}
**단일세포 자료의 주의점:** 낮은 검출 깊이와 sampling zero 때문에 세포별 RAS는 불안정할 수 있습니다. 유사한 세포를 biological replicate와 세포 유형별로 집계한 **pseudobulk** 프로파일은 흔한 완화 방법입니다. 집계 뒤에도 donor 복제를 유지하고, 사용한 정규화가 해당 통합 알고리즘의 입력 가정과 맞는지 확인해야 합니다.
{% endhint %}

## 4.3 데이터 출처 개관

실제 RNA-seq 발현 데이터를 구할 수 있는 대표적인 공개 포털은 다음과 같습니다.

| 포털 | 운영 기관 | 특징 |
|---|---|---|
| **GDC** (Genomic Data Commons) | NCI | TCGA 등 암 연구의 raw·harmonized 파일과 임상 메타데이터; 접근 등급 확인 필요 |
| **cBioPortal** | MSKCC | 공개 연구의 처리된 분자·임상 행렬을 탐색·시각화; 원시 read 저장소는 아님 |
| **recount3** | Johns Hopkins | GTEx·TCGA·SRA 자료를 공통 파이프라인으로 처리한 count·coverage 요약 |
| **GEO** | NCBI | 연구자가 제출한 원시 및 처리 자료; 플랫폼·전처리 메타데이터를 연구별로 확인 |

TCGA(The Cancer Genome Atlas)는 33종 암에 대한 종양·정상 조직의 RNA-seq과 다중 오믹스 자료를 제공하는 대표적 코호트입니다. 대사 모델링에서는 GDC의 harmonized counts 또는 출처와 정량법이 명확한 처리 행렬을 사용하고, sample type·aliquot·임상 메타데이터를 함께 고정해야 합니다. 구체적인 암종 코드와 암 맥락 모델 구축은 [Chapter 7](../chapter-7/README.md)에서 다룹니다.

TCGA와 GTEx처럼 수집 설계가 다른 코호트를 비교할 때 동일한 정렬·정량화 파이프라인을 사용하면 계산 파이프라인 차이는 줄일 수 있습니다. 그러나 recount3의 통일 처리도 조직 수집, library preparation, 병원과 donor 구성의 batch effect를 제거하지는 않습니다. 연구 그룹과 batch가 교란된 설계는 사후 보정만으로 식별할 수 없습니다.

## 4.4 7단계 전처리 파이프라인

RNA-seq 자료를 GEM 입력으로 변환하는 대표적 파이프라인은 다음과 같습니다.

```
Step 1. 데이터 다운로드      (GDC / recount3 / GEO에서 raw counts 또는 TPM 획득)
Step 2. 품질 관리(QC)        (library depth·mapping·RNA 품질·표본 메타데이터를 연구 설계에 맞게 평가)
Step 3. 정규화                (raw counts → TPM, 필요시 log2(TPM+1) 변환)
Step 4. 유전자 필터링         (대부분 샘플에서 미발현인 유전자 제거, 단백질 코딩 유전자만 선택)
Step 5. 배치 효과 평가·보정   (자료 척도와 설계에 맞는 ComBat/ComBat-seq/RUVseq 등을 선택)
Step 6. 유전자 식별자 매핑    (Ensembl ↔ Entrez ↔ HGNC symbol, 모델의 gene ID와 일치)
Step 7. RAS 변환              (GPR 규칙 적용: AND=min, OR=max — 2절에서 손으로 계산한 바로 그 과정)
```

**배치 효과(batch effect)**는 처리 기관, library preparation 또는 sequencing run과 연관된 체계적 변동입니다. 원 ComBat은 보통 적절히 변환한 연속 발현 행렬에 적용하고, raw count에는 음이항 count 구조를 고려하는 ComBat-seq 같은 방법을 검토합니다. 보정 뒤 PCA가 무작위가 되어야 하는 것이 아니라, 관심 생물학적 그룹의 구조를 보존하면서 batch와 주성분의 연관이 감소해야 합니다. batch와 암종 같은 관심 변수가 완전히 일치하면 통계적으로 두 효과를 분리할 수 없습니다.

**전처리 체크리스트**:

| 단계 | 확인 항목 | 판정 원칙 | 문제 발견 시 조치 |
|---|---|---|---|
| QC | library depth, mapping, duplication, RNA 품질 | assay 목적과 포화도·검정력 분석에 따른 기준; 보편적 10M cutoff 없음 | 원자료·library QC와 제외 사유 검토 |
| 정규화 | TPM 합계 | 계산 정의상 $$10^6$$인지 확인하는 구현 검사 | 유효 길이·0 division·정량화 파일 확인 |
| 필터링 | 검출 유전자 수와 low-count 분포 | annotation과 조직·platform을 반영; 15,000~20,000을 합격 기준으로 사용하지 않음 | 필터별 민감도 분석 |
| 배치 효과 | PCA/모델 잔차와 batch·biology의 연관 | batch 신호 감소와 생물학적 구조 보존을 함께 평가 | 설계행렬·척도·보정법 재검토 |
| ID 매핑 | mapped/unmapped ID와 중복 매핑 | 비율뿐 아니라 unmapped 대사 유전자와 일대다 매핑을 조사 | annotation release와 대응표 고정 |

**흔한 오류와 해결**:

| 오류 | 원인 | 해결 |
|---|---|---|
| Gene identifier mismatch | 발현 데이터의 Ensembl ID와 모델의 Entrez ID/symbol 불일치 | `mygene.info`, `biomaRt`로 매핑 |
| 재구축 후 모든 반응 flux = 0 | 유전자 필터링이 지나치게 엄격하거나 GPR 매핑 오류 | 필터 완화, GPR 규칙 수동 확인 |
| 모델이 biomass를 생산하지 못함 | 핵심 반응이 제거되었거나 네트워크에 갭 존재 | [Chapter 5](../chapter-5/README.md)의 gap-filling 참고 |
| Solver timeout | MILP(iMAT/tINIT)가 너무 복잡함 | 상용 솔버(Gurobi/CPLEX) 사용, LP 기반 방법(GIMME/E-Flux/FASTCORE)으로 전환 |

{% hint style="info" %}
**절 연결:** 3단계 정규화는 4.1절, 7단계 GPR/RAS 변환은 2절의 계산을 실제 분석 순서에 배치한 것입니다.
{% endhint %}
