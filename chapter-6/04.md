# 4. RNA-seq 데이터: 유형과 전처리 파이프라인

## 4.1 Counts, FPKM, TPM

RNA-seq 데이터는 정규화 수준에 따라 세 가지 형태로 제공됩니다. 이 세 형태를 구분하는 것이 왜 중요한지 먼저 짚고 넘어갑시다 — 만약 서로 다른 정규화 방식을 섞어서 비교하면(예: 한 샘플은 FPKM, 다른 샘플은 TPM), 실제 생물학적 차이가 아니라 **단순히 계산 방식이 달라서 생기는 가짜 차이**를 발현 차이로 오인할 수 있습니다. 이는 GEM 가중치 계산에서 특히 치명적인데, 잘못된 정규화 차이가 그대로 반응 활성 점수(RAS)의 차이로 번역되어 맥락 특이적 모델 자체가 왜곡되기 때문입니다.

**Raw counts**: HTSeq, featureCounts, RSEM 등으로 계산된, 각 유전자에 매핑된 리드(read) 수. 정수값이며 라이브러리 크기와 유전자 길이에 의존하므로 **샘플 간 직접 비교가 불가능**합니다. 예를 들어 유전자 A가 샘플 1에서 1,000 counts, 샘플 2에서 2,000 counts라고 해서 샘플 2에서 유전자 A가 두 배 더 발현된 것은 아닙니다 — 샘플 2의 시퀀싱을 애초에 두 배 더 깊게 했을 수도 있기 때문입니다.

**FPKM (Fragments Per Kilobase of transcript per Million mapped reads)**:

$$FPKM = \frac{X \cdot 10^9}{N \cdot L}$$

($$X$$: 유전자에 매핑된 프래그먼트 수, $$N$$: 전체 매핑된 프래그먼트 수, $$L$$: 유전자 길이(bp)) 라이브러리 크기와 유전자 길이를 보정하지만, 샘플마다 총 FPKM 합이 달라 **샘플 간 비교에는 부적절**합니다.

**TPM (Transcripts Per Million)**:

$$TPM_i = \frac{X_i / \tilde{l}_i}{\sum_j (X_j / \tilde{l}_j)} \times 10^6$$

($$\tilde{l}_i$$: 유전자 $$i$$의 유효 길이) 분모의 합산 때문에 **각 샘플의 TPM 합은 항상 일정한 값**($$10^6$$)이 됩니다 — 이 성질이 왜 중요한지는 아래 손 계산에서 확인합니다. 따라서 TPM은 한 샘플 안에서 유전자들의 상대적 조성을 비교하거나, 같은 정량 파이프라인으로 처리한 샘플을 발현 기반 GEM 가중치로 변환할 때 편리합니다. 그러나 합이 고정된 **조성 자료(compositional data)**이므로 한 유전자의 TPM 변화가 다른 유전자의 상대값까지 움직일 수 있고, RNA 총량 차이를 보존하지 않습니다. 차등발현 검정에는 TPM에 t-test를 바로 적용하기보다 raw counts와 DESeq2/edgeR 같은 count 모델을 사용하고, 표본 간 GEM 통합에서도 배치·조직 조성·정량 파이프라인을 맞춘 뒤 민감도 분석을 해야 합니다. 자세한 통계 배경은 [오믹스 데이터 분석을 위한 기초 통계](../supplements/omics-statistics.md)를 참고하십시오.

| 특성 | Raw Counts | FPKM | TPM |
|---|---|---|---|
| 정규화 | 없음 | Library size + Gene length | Library size + Gene length |
| 샘플 간 비교 | 라이브러리 크기 보정 전에는 불가 | 총합 차이 때문에 주의 | 동일 파이프라인·조성 효과를 점검한 상대 비교에 사용 |
| 합계 | 가변적 | 가변적 | 항상 $$10^6$$ |
| 주요 용도 | DESeq2/edgeR 등 차등발현 검정의 입력 | 기존 자료 호환 | 샘플 내 상대 발현, GEM 가중치의 한 후보 |

### 4.1.1 손으로 직접 계산해보기: 같은 counts, 다른 정규화

FPKM과 TPM의 차이를 가장 확실하게 이해하는 방법은 같은 원시 데이터로 두 값을 모두 계산해보는 것입니다. 세 유전자의 raw counts와 길이, 그리고 전체 매핑된 프래그먼트 수(라이브러리 크기)가 다음과 같다고 합시다.

| 유전자 | Counts $$X_i$$ | 길이 $$L_i$$ (bp) | 전체 매핑 수 $$N$$ |
|---|---:|---:|---:|
| 유전자 X | 500 | 2,000 | 2,000,000 |
| 유전자 Y | 1,000 | 500 | 2,000,000 |
| 유전자 Z | 300 | 1,500 | 2,000,000 |

**FPKM 계산** ($$FPKM_i = X_i \cdot 10^9 / (N \cdot L_i)$$):

$$FPKM_X = \frac{500 \times 10^9}{2{,}000{,}000 \times 2{,}000} = 125.0$$

$$FPKM_Y = \frac{1{,}000 \times 10^9}{2{,}000{,}000 \times 500} = 1{,}000.0$$

$$FPKM_Z = \frac{300 \times 10^9}{2{,}000{,}000 \times 1{,}500} = 100.0$$

FPKM 값의 합은 $$125+1000+100=1225$$로, 다른 샘플에서 계산하면 이 합이 다른 값이 될 수 있습니다 — 바로 이 점이 FPKM을 샘플 간 비교에 부적절하게 만드는 이유입니다.

**TPM 계산** (먼저 길이로 나눈 비율 $$rate_i = X_i/L_i$$를 구한 뒤, 그 비율의 합으로 다시 나누어 정규화):

$$rate_X = 500/2000 = 0.25, \quad rate_Y = 1000/500 = 2.0, \quad rate_Z = 300/1500 = 0.2$$

$$\sum rate = 0.25+2.0+0.2 = 2.45$$

$$TPM_X = \frac{0.25}{2.45}\times 10^6 \approx 102{,}041, \quad TPM_Y = \frac{2.0}{2.45}\times 10^6 \approx 816{,}327, \quad TPM_Z = \frac{0.2}{2.45}\times 10^6 \approx 81{,}633$$

세 값의 합을 확인하면 $$102{,}041+816{,}327+81{,}633 = 1{,}000{,}001 \approx 10^6$$(반올림 오차 제외)로 정확히 100만이 됩니다. **FPKM은 유전자마다 독립적으로 계산되어 합이 데이터마다 달라지지만, TPM은 계산 과정 자체에 "전체 합을 100만으로 맞추는" 정규화 단계가 포함되어 있다**는 점이 두 지표의 근본적 차이입니다. 이 성질 때문에 TPM은 "이 샘플 안에서 유전자 Y가 전체 발현량의 몇 %를 차지하는가"라는 질문에 곧바로 답할 수 있지만(위 예에서 유전자 Y는 전체의 약 81.6%), 서로 다른 두 샘플의 유전자 Y를 직접 비교할 때는 여전히 "그 샘플의 다른 유전자들이 얼마나 발현되었는가"에 따라 상대적으로 값이 흔들릴 수 있다는 조성 자료(compositional data)의 함정을 안고 있습니다.

## 4.2 RNA-seq의 종류: 벌크에서 단일세포까지

지금까지는 하나의 조직·샘플에서 세포 수백만 개를 갈아 넣어 얻은 평균 발현값 — **벌크 RNA-seq(bulk RNA-seq)** — 을 암묵적으로 가정했습니다. 그러나 실제로는 목적에 따라 여러 종류의 RNA-seq 기술이 존재하며, 각각이 맥락 특이적 모델링에 주는 함의가 다릅니다.

| 유형 | 측정 단위 | 대사 모델링에서의 함의 |
|---|---|---|
| **벌크(bulk) RNA-seq** | 조직·세포 집단의 평균 발현 | 이 장에서 다루는 GIMME/iMAT/tINIT 등 대부분의 통합 방법이 가정하는 표준 입력. 세포 이질성(heterogeneity)은 평균화되어 사라짐 |
| **단일세포(single-cell, scRNA-seq)** | 세포 하나하나의 발현 | 세포 유형별로 별도의 맥락 특이적 모델을 만들 수 있어 조직 내 이질성(예: 종양 미세환경의 암세포·면역세포·기질세포)을 반영 가능. 다만 유전자당 리드 수가 매우 적어(drop-out) RAS 계산 전에 세포를 유사한 그룹으로 묶는 pseudobulk 집계가 흔히 필요 |
| **단일핵(single-nucleus, snRNA-seq)** | 핵 안의 전사체만 측정 | 냉동 조직·해리가 어려운 조직(뇌, 지방)에 적합. 성숙한 mRNA 비율이 달라 벌크·scRNA-seq와 직접 비교 시 주의 필요 |
| **공간 전사체(spatial transcriptomics)** | 조직 절편 내 위치 정보가 있는 발현 | 조직 내 위치에 따른 대사 구역화(metabolic zonation, 예: 간의 문맥주위/중심정맥주위 대사 차이)를 맥락 특이적 모델에 반영하는 데 활용 시작 단계 |

{% hint style="warning" %}
❓ **흔한 오해:** "scRNA-seq 데이터가 있으니 세포 하나하나마다 완전한 맥락 특이적 모델을 따로 만들 수 있겠다." — 이론적으로는 가능하지만, 단일세포 수준의 데이터는 흔히 유전자당 검출된 리드 수가 극히 적어(많은 유전자가 실제로는 발현되지만 우연히 검출되지 않는 "drop-out" 현상) 개별 세포의 RAS가 매우 불안정합니다. 실무에서는 비슷한 세포끼리 클러스터로 묶어 그 클러스터의 발현을 합산한 **pseudobulk** 프로파일을 만든 뒤, 이 장에서 다룬 벌크 RNA-seq 파이프라인과 동일한 절차(TPM 계산 → GPR → RAS → GIMME/iMAT 등)를 적용하는 것이 표준적인 접근입니다.
{% endhint %}

## 4.3 데이터 출처 개관

실제 RNA-seq 발현 데이터를 구할 수 있는 대표적인 공개 포털은 다음과 같습니다.

| 포털 | 운영 기관 | 특징 |
|---|---|---|
| **GDC** (Genomic Data Commons) | NCI | 공식 저장소, Level 1~4 원시/처리 데이터 모두 제공 |
| **cBioPortal** | MSKCC | 시각화·통합 분석(OncoPrint, 발현-생존 상관) |
| **recount3** | Johns Hopkins | 8,000+ 데이터셋을 통일된 파이프라인으로 재처리, TCGA+GTEx+SRA 비교 가능 |
| **GEO** | NCBI | 미리 처리된 발현 매트릭스 (예: GSE62944) |

TCGA(The Cancer Genome Atlas)는 20,000개 이상의 샘플(종양 약 11,000개, 대응 정상 조직 약 1,000개)에 대해 RNA-seq을 포함한 다중 오믹스 데이터를 33종 암 유형에 걸쳐 생산한 대표적 공개 데이터셋으로, 대사 모델링 연구에서는 주로 Level 3(유전자 발현 counts/TPM 등 처리된 데이터)를 사용합니다. TCGA의 33종 암 코드, 임상 메타데이터 활용, 암 맥락 특이적 모델 구축과 약물 표적 예측의 구체적 파이프라인은 [Chapter 7](../chapter-7/README.md)에서 다룹니다. 여기서는 일반적인 RNA-seq 전처리 관점만 다룹니다.

한 가지 실무적으로 중요한 점은, **recount3처럼 통일된 파이프라인으로 재처리된 데이터를 우선 고려해야 한다**는 것입니다. TCGA와 GTEx(정상 조직 대조군)를 각각 다른 시기에 다른 파이프라인으로 처리된 원본 그대로 비교하면, 생물학적 차이가 아니라 파이프라인 차이(정렬 도구 버전, 어노테이션 버전, 정규화 방식)가 결과를 지배할 위험이 있습니다. recount3는 이런 배치 효과를 줄이기 위해 모든 데이터셋을 동일한 정렬·정량화 파이프라인으로 재처리해 제공합니다.

## 4.4 7단계 전처리 파이프라인

RNA-seq 원시 데이터를 GEM 입력(반응 가중치)으로 변환하는 표준 파이프라인은 다음과 같습니다.

```
Step 1. 데이터 다운로드      (GDC / recount3 / GEO에서 raw counts 또는 TPM 획득)
Step 2. 품질 관리(QC)        (시퀀싱 깊이 > 10M reads, 종양 순도 > 0.6, PCA 이상치 제거)
Step 3. 정규화                (raw counts → TPM, 필요시 log2(TPM+1) 변환)
Step 4. 유전자 필터링         (대부분 샘플에서 미발현인 유전자 제거, 단백질 코딩 유전자만 선택)
Step 5. 배치 효과 보정        (ComBat, RUVseq 등으로 처리 센터/시퀀싱 배치 효과 제거)
Step 6. 유전자 식별자 매핑    (Ensembl ↔ Entrez ↔ HGNC symbol, 모델의 gene ID와 일치)
Step 7. RAS 변환              (GPR 규칙 적용: AND=min, OR=max — 2절에서 손으로 계산한 바로 그 과정)
```

이 7단계 중 5단계(배치 효과 보정)를 조금 더 풀어 봅시다. 여러 병원·연구소에서 각각 샘플을 채취해 시퀀싱했다면, 같은 조직이라도 "어느 기관에서 처리했는가"라는 기술적 요인 하나만으로 발현값이 체계적으로 달라질 수 있습니다 — 이를 **배치 효과(batch effect)**라고 합니다. ComBat 같은 방법은 각 배치(기관/시퀀싱 런)의 평균과 분산을 통계적으로 정렬해, 생물학적 신호(조직 차이)는 보존하면서 기술적 신호(배치 차이)만 제거하려고 시도합니다. 다만 배치와 생물학적으로 관심 있는 그룹(예: 암종)이 완전히 뒤섞여 있지 않고 우연히 일치한다면(confounding), ComBat이 생물학적 신호까지 함께 지워버릴 위험이 있으므로 배치 설계 자체를 사전에 점검해야 합니다.

**전처리 체크리스트**:

| 단계 | 확인 항목 | 통과 기준 | 실패 시 조치 |
|---|---|---|---|
| QC | 평균 시퀀싱 깊이 | > 10M reads | 샘플 제거 |
| 정규화 | TPM 합계 | $$= 10^6 \pm 0.1\%$$ | 재계산 |
| 필터링 | 남은 유전자 수 | 15,000~20,000 | 필터 임계값 조정 |
| 배치 보정 | PCA 그룹화 | 무작위 분포 | ComBat 재적용 |
| ID 매핑 | 매핑률 | > 90% | 다른 annotation 사용 |

**흔한 오류와 해결**:

| 오류 | 원인 | 해결 |
|---|---|---|
| Gene identifier mismatch | 발현 데이터의 Ensembl ID와 모델의 Entrez ID/symbol 불일치 | `mygene.info`, `biomaRt`로 매핑 |
| 재구축 후 모든 반응 flux = 0 | 유전자 필터링이 지나치게 엄격하거나 GPR 매핑 오류 | 필터 완화, GPR 규칙 수동 확인 |
| 모델이 biomass를 생산하지 못함 | 핵심 반응이 제거되었거나 네트워크에 갭 존재 | [Chapter 5](../chapter-5/README.md)의 gap-filling 참고 |
| Solver timeout | MILP(iMAT/tINIT)가 너무 복잡함 | 상용 솔버(Gurobi/CPLEX) 사용, LP 기반 방법(GIMME/E-Flux/FASTCORE)으로 전환 |

{% hint style="info" %}
💡 **연결 고리:** 이 파이프라인의 3단계(정규화)와 7단계(RAS 변환)는 각각 이 장 4.1절과 2절에서 손으로 계산해 본 바로 그 과정입니다. 즉 이 7단계는 새로운 개념이 아니라, 지금까지 배운 조각들을 실무 순서대로 나열한 것입니다.
{% endhint %}
