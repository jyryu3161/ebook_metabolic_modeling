# 교재 문체 규약

대상: `게놈 규모 대사 모델링` 한국어 교재(학부 고학년·대학원 입문). 정본 기준은 [`docs/textbook-editorial-standard.md`](../../../docs/textbook-editorial-standard.md) §1–2이며, 이 스킬은 그 기준을 집필 시점에 적용하기 위한 실행 지침이다.

## 1. 종결 어미 — 파일 유형에 따라 이원화

한 파일 안에서 두 어미를 절대 섞지 않는다.

| 파일 | 어미 | 예 |
|---|---|---|
| 장 `README.md`, 번호 절 `01.md`~`NN.md` | **학술 다체 `~한다/이다`** | "FBA는 목적함수를 최적화하는 해를 구한다." |
| `lab.md`, `chapter-10/*`, `chapter-11/*` | **습니다체 `~합니다/입니다`** | "성장률을 계산하고 조건을 바꿔 관찰합니다." |
| `summary.md` | 해당 장 이론 본문과 동일(다체) | |

작업 전 대상 파일 경로를 먼저 확인하고 어미를 결정한다. 기존 파일을 수정할 때는 그 파일의 지배적 어미를 따른다.

**놓치기 쉬운 두 가지:**

- **볼드·이탤릭으로 감싼 어미에도 그대로 적용된다.** `**확인한다**.`는 `**확인합니다**.`로 쓴다. 검출할 때 강조 마커를 먼저 제거하지 않으면 이런 형태를 통째로 놓친다 — 실제로 8개 장의 `lab.md` 학습목표 위반이 이 이유로 오래 남아 있었다.
- **코드 블록 안의 한국어 주석은 명사구로 쓴다**(`# 성장률 계산`). 산문 어미(다체·습니다체)를 주석에 쓰지 않으며, 실습 파일이든 이론 파일이든 동일하다.

검출 방법: 강조 마커(`**`, `*`, `_`)와 코드 블록·인라인 코드를 먼저 제거한 뒤, `다.`로 끝나되 `니다.`가 아닌 것을 찾는다. `봅시다.`·`합시다.`는 습니다체 청유형이므로 위반이 아니다.

## 2. 금지 표현

다음은 발견 즉시 제거한다.

- 독자 호칭·독려: `여러분`, `기억하십시오`, `~해 봅시다`(이론 본문에서)
- 수사적 질문 제목: `왜 …하는가?`, `…할까요?` → 서술형 명제로 교체
  - ✗ `왜 FBA가 필요한가?` → ✓ `FBA는 부정정 문제에 최적성 가정을 더한다`
- 학습 유도 문구: `왜 이걸 배우나요?`, `아주 쉬운 비유`
- 감탄형·이모지·과장: `놀랍게도`, `강력한`, `혁명적인`, 이모지 전부
- `흔한 오해` 블록 → `해석상의 주의`로 교체

## 3. 논증 전개 순서

핵심 주장은 반드시 이 순서로 쓴다.

```
정의 → 가정 → 수학적 표현 또는 실험적 근거 → 해석 범위와 한계
```

절은 학술적 문제·정의·분석 범위 진술로 시작한다. 배경 잡담이나 동기 부여 훅으로 시작하지 않는다.

## 4. 용어 표기

- 한국어 용어를 우선하고, **처음 등장할 때만** 영어 원어와 약어를 병기한다. 이후에는 하나의 표기로 일관되게 쓴다.
  - `플럭스 균형 분석(Flux Balance Analysis, FBA)` → 이후 `FBA`
- 표준형을 지킨다: **대사물**(대사산물 ✗), **정상상태**(정상 상태 ✗ — 단 편집 기준 원문 인용 시 예외), **의사정상상태**
- 용어집이 있는 항목은 [`glossary.md`](../../../glossary.md)로 링크한다: `[화학량론 행렬](../glossary.md)`
- 생화학적 대상과 계산 객체를 구분한다. `경계조건`, `목적함수`, `솔버`는 생물학적 사실이 아니라 모델에 부여한 계산 조건임을 드러낸다.

## 5. 조건부 진술 — 단정 금지

모델 출력은 **명시된 배지·경계조건·목적함수 아래의 조건부 예측**이다.

- ✗ `대장균의 성장률은 0.874 h⁻¹이다`
- ✓ `COBRApy 0.30.0 `textbook` 모델의 기본 배지와 바이오매스 목적함수를 GLPK로 풀면 $$\mu\approx0.874$$ h$$^{-1}$$이다. 이 값은 모델·배지·유지에너지·solver 허용오차에 의존한다.`

지켜야 할 구분:

- **예측 / 실험 관찰 / 저자의 해석**을 문장에서 분리한다.
- 반응·대사물·유전자 **수는 종의 특성이 아니라 특정 모델 릴리스의 속성**이다. 모델명·버전·집계 기준을 함께 쓴다.
- 정상상태, 최적성, 반응 방향성은 조건 의존 진술이므로 보편 사실로 서술하지 않는다.
- 단일 연구·단일 조건 결과를 일반 법칙으로 확장하지 않는다.

## 6. 수식

기호를 제시할 때 **정의·차원(단위)·적용 가정**을 함께 쓴다. 플럭스는 양이 아니라 단위 시간·단위 생물량당 반응 진행률(`mmol gDW⁻¹ h⁻¹`)임을 명시한다.

GitBook 수식은 `$$...$$` 인라인으로 쓴다. `\qquad`, `\left\{`는 이스케이프 규칙을 지켜야 하며 [`scripts/validate_textbook.py`](../../../scripts/validate_textbook.py)가 검사한다.

## 7. 비유 사용 조건

일상 비유는 정확한 정의를 **대체하지 않는 경우에만** 제한적으로 쓰고, 반드시 **성립하지 않는 지점**을 함께 밝힌다.

> 정상상태는 물탱크의 수위가 일정한 상태에 대응한다. 다만 열역학적 평형과 달리 순 플럭스가 0이 아니므로, 이 비유는 유입·유출이 계속된다는 점까지만 유효하다.

## 8. 작업 절차

1. 대상 파일 경로로 어미(다체/습니다체)를 결정한다.
2. 초안을 §3 순서로 작성한다.
3. §2 금지 표현을 검색해 제거한다.
4. 새 용어는 §4에 따라 병기하고 glossary 링크를 건다.
5. 정량 진술마다 조건(모델·버전·배지·목적함수·solver)이 붙었는지 확인한다.
6. 마친 뒤 `python scripts/validate_textbook.py`로 구조 검사를 통과시킨다.

관련: [[ebook-review]]로 전체 검수, [[ebook-citation]]으로 출처 표기, [[ebook-figure]]로 그림·캡션.
